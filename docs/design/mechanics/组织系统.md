# 通用项目管理系统

## 目的

这是整个游戏**最核心的系统**。让玩家用同一套工具，搭建帝国里的一切组织——军队、科学院、工程队、行政体系、商队。每个组织的底层逻辑完全相同——把一群火柴人按照你设计的层级组织起来，向某个目标运作。

## 玩家体验目标

- **一通百通**：学会了搭军队编制，就学会了搭科学院架构——逻辑一样，只是名字和用途不同
- **亲手设计**：每个组织之间的层级关系、每个层级的编制配置，都是你亲手设定的
- **看到效果**：好的架构高效运转，差的架构处处卡壳
- **随时回落**：你已是皇帝，但缩放到某个排长身上去微操绕后——像《戴森球计划》缩放一样自然

---

## 一、五层级框架

帝国里所有的组织都共享同一套五层级深度：

| 层级 | 行政语境 | 军事语境 | 科研语境 | 工程语境 | 商业语境 |
|------|----------|----------|----------|----------|----------|
| **L5** | 帝国 | 帝国最高统帅部 | 科学院 | 帝国工部 | 全国商号总行 |
| **L4** | 区域/行省 | 师 | 研究所 | 区域工程司 | 区域商行 |
| **L3** | 城市/郡县 | 团 | 研究室 | 项目组 | 城市商号 |
| **L2** | 镇/村 | 连 | 课题组 | 施工队 | 镇级商铺 |
| **L1** | 个体/班 | 排/班 | 实验员 | 工匠/班组 | 个体商贩 |

### 1.1 层级弹性

**加层级**：标准层级之间可以额外插入一层。例如：
- 在"团"(L3)和"师"(L4)之间插入"旅"
- 在"连"(L2)和"排"(L1)之间插入"火力组"

**减层级**：可以跳过某一层级。例如：
- 去掉行省(L4)，中央(L5)直接对接全部城市(L3)——中国某些朝代草创和快灭亡时常这样干
- 现实中装甲部队用"旅"取代"团"，直接对接营

**最底层宽松**：第1层可以额外向下塞**两层**。例如：
- 火力组 → 班 → 排，三个子层都挤在第1层内
- 班是最小编制，但在实际指挥中班长发现直接管单个步兵太难——于是手动分了两个火力组

**灵活命名的意义**：火力组不是正式编制，是班长在战斗中为了方便指挥临时分的——游戏里玩家的层级也是这样：正式的编制层级和临时的指挥分组可以共存。

### 1.2 集团军怎么办？

玩家本身是一个**虚拟层级**。当你的帝国足够大，你需要同时指挥多个"师"时——你是 L5+，下辖多个 L5 组织（多个师）。这个虚拟层级不算在标准五层级内，它是"你"。

就像周天子不过几个师——在游戏里，你作为皇帝管的可能就是几个 L4 级别的军事组织，它们并排放在你的名下。

## 二、组织即全能

**每个组织理论上什么都能干。** 军事编制可以施工（工程兵），科研单位可以被拉去打仗（虽然打得不好），行政机构可以经商（腐败风险大但理论上可以）。

### 2.1 标签系统

玩家给每个组织打上一个**标签**，决定了该组织的主界面默认展示什么：

| 标签 | 主界面默认显示 | 底层仍然可以做的事 |
|------|---------------|-------------------|
| 军事 | 进攻、防御、行军、呼叫支援、部署 | 施工、科研、贸易…… |
| 科研 | 分配课题、审核论文、调拨经费、实验 | 打仗、施工、贸易…… |
| 工程 | 开工、验收、材料调度、质量检查 | 打仗、科研、贸易…… |
| 行政 | 征税、户籍、司法、治安 | 打仗、科研、施工…… |
| 商业 | 采购、销售、定价、物流 | 打仗、科研、施工…… |

UI 上——军事标签的面板显示进攻/防御按钮，科研标签的面罩显示课题管理。但点开**"完整功能"展开箭头**，你会发现所有标签下的组织都有一样的底层能力列表。就像 Blender 的 Modeling、Sculpting、Layout 工作区——功能一样，常用工具不同。

### 2.2 特化收益

虽然全能，但**特化有加成**：

- 军事编制的战斗力远高于科研单位被拉去打仗
- 科研编制的产出远高于军事单位做研究
- 工程编制的施工效率远高于商队修路

特化程度取决于：（1）你给这个组织配置的编制模板（2）人员的资质匹配度（3）装备配置。

## 三、编制管理

在每个层级的节点上，玩家配置该节点的**具体编制**——这个班/连/团/师里具体有什么。

### 3.1 可配置项

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **人员组成** | 该层级包含的角色类型和数量 | 1个机枪手 + 4个步枪兵 + 1个法师 + 2个无人机飞手 |
| **装备配置** | 标配武器、护甲、工具 | 标准步枪、皮甲、工兵铲 |
| **指挥官** | 任命或留空 | "铁拳张三" 或 系统自动填补 |
| **自主决策权限** | 该层级可以自行决定什么 | 攻击/防御/撤退/调兵/征用物资 |
| **默认战术** | 无指令时的自动行为 | 主动进攻 / 坚守阵地 / 游击骚扰 / 自主判断 |
| **补给优先级** | 物资分配的权重 | 高 / 中 / 低 |
| **士气阈值** | 什么条件下溃退/罢工 | 伤亡 >40% 或 指挥官阵亡 或 断粮3天 |

### 3.2 玩家自定义自由度

- 人员组成可以**自由拖拽组合**——这班3个法师2个步兵，那班全是步兵
- 装备可以**从帝国库存中选择分配**（如果帝国物资不够，系统会提示缺货）
- 预设几个模板方便快速搭建，但随时可以拆零件重组

### 3.3 命名规则

- **军事/科研/工程组织**：层级名称可以自由修改（"这个师叫'虎贲师'，下面的团叫'第一团'到'第三团'"）
- **行政区划**：部分锁定，不能随便改（"北京"不能改叫"哈基米"）

## 四、AI 行为与自动化

### 4.1 逐层指挥

每一层级的 AI 只能**直接指挥向下的一层**。它看不到下属的下属在干什么——只是下达任务，下属自己分解执行。

```
帝国统帅(L5) → 只告诉师长(L4)："拿下这个区域"
师长 → 只告诉团长(L3)："你的团攻这个方向"
团长 → 只告诉连长(L2)："你的连守住这个高地"
连长 → 只告诉排长(L1)："你的排从右边绕"
排长 → 具体指挥每个火柴人
```

**这意味着**：你想让帝国自动化运转起来——必须精密地设计每一层。层级设计得好，命令顺畅传达。层级设计得烂，上面下的命令传不到下面，或者下面自己乱搞。

### 4.2 信息上报

反向也一样——下层的问题向上汇报，每层自己判断什么需要上报、什么自己处理。自主决策权限越高，往上报的越少。

### 4.3 玩家干预

你随时可以"**附身**"到任意层级的指挥官：

- **操纵模式**：真的控制那个角色在战场上移动/战斗
- **管理面板模式**：打开该层级的管理面板，下达指令、调整编制
- 两种模式随时切换

当你附身到低层级微操时，高层级帝国继续运转（或自动暂停/减速——取决于你的设置）。

## 五、UI 设计

### 5.1 组织管理面板

同一个面板，所有类型的组织都在这里管理。UI 结构：

```
┌─────────────────────────────────────────────┐
│ [组织标签选择：军事 | 科研 | 工程 | 行政 | 商业]  │
├─────────────────────────────────────────────┤
│                                             │
│   [树状/网状层级图，可拖拽、可缩放]            │
│                                             │
│   L5 帝国统帅部                              │
│   ├── L4 第一师 [任命: 王五] [效率: 85%]      │
│   │   ├── L3 第一团                           │
│   │   │   ├── L2 第一连                       │
│   │   │   │   └── L1 第一排 [12人]            │
│   │   │   │       ├── 火力组A [6人]           │
│   │   │   │       └── 火力组B [6人]           │
│   │   │   ├── L2 第二连                       │
│   │   │   └── L2 第三连                       │
│   │   ├── L3 第二团                           │
│   │   └── L3 第三团                           │
│   └── ...                                    │
│                                             │
├─────────────────────────────────────────────┤
│ [选中节点的详情: 编制/人员/装备/指令/状态]      │
│ [快速操作按钮(根据标签变化)]                   │
│ [▼ 完整功能列表]                              │
└─────────────────────────────────────────────┘
```

### 5.2 标签切换 = 工作区切换

切换标签时：
- 树状图不变（组织还在那里）
- 右侧详情面板的默认展开项变化
- 快速操作按钮变化

参考：Blender 的工作区预设（Modeling/Layout/Sculpting/Shading）——底层一样，布局和常用工具不同。

## 六、预设与分享

### 6.1 预设模板

游戏提供多个预设架构模板：

| 模板 | 描述 |
|------|------|
| 标准军事编制 | 师→团→连→排（中间可加旅） |
| 科学院架构 | 科学院→研究所→研究室→课题组 |
| 工程指挥部 | 总指挥部→区域→项目组→施工队 |
| 标准郡县制 | 州→郡→县→乡 |
| 商业网络 | 总行→分行→商号→商铺 |
| **空白 + 标签** | 选择一个标签，从零搭建 |

### 6.2 UGC 分享

- 导出/导入组织架构配置（JSON 格式）
- 社区可以分享"最优指挥链"、"最高效科学院架构"
- 不同文明的最优架构不一样——农耕 vs 游牧 vs 商业 vs 海洋

## 七、不同文明的最优组织结构

| 文明 | 组织特点 |
|------|----------|
| 农耕文明 | 层级深（行政体系完整），军事编制正规，中央集权 |
| 游牧文明 | 层级浅（2-3层足够），军事编制灵活，以万户/千户为单位 |
| 商业文明 | 经济组织发达，军事外包/雇佣兵，行政精简 |
| 海洋文明 | 海军编制特殊，殖民地管理体系，贸易网络复杂 |

不同地形/气候的火柴人部落自然演化出不同的组织架构——这本身就是文明叙事的一部分。

## 八、边缘情况

- **指挥官死亡无替补**：该节点挂起，下级越级请示或自主行动
- **机构重叠**：两个同层组织职责重叠 → 效率下降
- **过宽管理幅度**：一个指挥官管太多下级 → 每个下级获得关注减少
- **推翻重建**：大规模重组 → 过渡期混乱 [PLACEHOLDER] 天
- **组织解散**：所有人员回归待分配池
