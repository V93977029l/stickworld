# UI/UX 设计规范

> UI 设计方向——不涉及像素级布局。

---

## 一、设计哲学

| 原则 | 说明 |
|------|------|
| **信息按需呈现** | 当前层级不需要的信息不显示 |
| **工作区预设** | 类似 Blender 的 Modeling/Layout/Sculpting——同面板，不同布局 |
| **连续缩放** | UI 元素随缩放渐隐渐显，不是硬切换 |
| **火柴人原生 UI** | 石头面板→羊皮纸→魔方投影，风格随层级自然演变 |
| **堆叠满足感** | 类似《博德之门3》技能栏密集堆叠的快感 |

---

## 二、工作区预设

从通用项目管理系统（[`organization.md`](mechanics/organization.md)）衍生：

同一个组织管理面板，切换标签 = 切换工作区布局：

| 标签 | 默认显示的快速操作 | 隐藏但可展开的 |
|------|-------------------|---------------|
| 军事 | 进攻/防御/行军/呼叫支援/部署 | 施工/科研/贸易等完整功能列表 |
| 科研 | 分配课题/审核论文/调拨经费/实验 | 进攻/施工/贸易等 |
| 工程 | 开工/验收/材料调度/质量检查 | 进攻/科研/贸易等 |
| 行政 | 征税/户籍/司法/治安 | 进攻/施工/科研等 |
| 商业 | 采购/销售/定价/物流 | 进攻/施工/科研等 |

**所有标签下的组织都有一样的底层能力**——展开"完整功能"箭头就能看到。只是不同标签默认把常用的摆出来。

参考：Blender 在不同工作区间切换——底层一样，布局和工具不同。

## 三、按层级的界面切换

| 层级 | 可见 UI | 隐藏 |
|------|---------|------|
| L1 个体 | HP、装备、技能栏 | 地图、报表 |
| L2 小队 | 小队状态、指令按钮 | 帝国报表 |
| L3 编制 | 组织面板、部队列表 | 个体装备 |
| L4 区域 | 城市列表、资源总览、外交 | 个体/小队面板 |
| L5 帝国 | 帝国总览、报表、战略地图 | 战术级面板 |

过渡：渐隐渐显——不是硬切。

## 五、组织架构面板

- 树状/网状图，拖拽、缩放
- 节点 = 组织/机构
- 连线 = 隶属关系
- 悬停 = 当前状态预览
- 右键 = 菜单
- 顶部标签栏切换工作区

---

## 六、缩放行为

- 滚轮连续缩放，无档位
- Shift+滚轮 = 快速缩放
- UI 透明度随缩放平滑过渡

---

## 七、时间控制

| 速度 | 用途 | 快捷键 |
|------|------|--------|
| 暂停 | 设计架构、做决策 | 空格 |
| 1x | 正常 | 1 |
| 2x | 加速等待 | 2 |
| 4x | 宏观快进 | 3 |

- 可设置自动暂停条件
- 可设置附身微操时自动减速
