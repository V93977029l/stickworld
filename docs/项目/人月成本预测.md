# 人月成本预测

> 基于 GDD v4.0 完整设计范围 + 现有代码库（2026-07-18 快照）实际成熟度。
> 假设：Godot 4.7 + GDScript，单一全职开发者（或小团队）。
> 本文档不是承诺——是给创始人参考的**量级估算**，用于决策"要不要砍范围"或"什么时候拉人"。

## 五、冷峻现实

### 这个 GDD 不是一个小项目的设计文档。

它是一个**大战略 + 工厂自动化 + 战术 RTS + 模拟经营 + 文明演化**的缝合怪。每个子系统的深度对标独立游戏：

- 战斗对标《小兵步枪》+《英雄连》——这两个随便哪个都是一整个团队的完整项目
- 经济对标《维多利亚3》的价格信号——P 社的旗舰产品，200+ 人开发
- 组织系统对标《异星工厂》的自动化思维——Wube 团队 5-10 年打磨出来的
- 无缝缩放对标《戴森球计划》——5 人团队 3 年

**这些每一个都是"团队+多年"级别的系统。全部合在一起做了。**

## 六、保守版范围建议

如果目标是**一个人 2 年内能放上 Steam EA 卖钱**，建议砍到以下最小可行范围：

| 砍掉 | 保留 | 理由 |
|------|------|------|
| 7 种文明 → **2 种**（平原+火山） | 核心循环不需要 7 种，2 种足以验证差异化 | 每种文明 = 大量美术 + 配置 |
| 法律系统→**后移到 EA** | EA 阶段不需要道德张力 | 1-2 人月的边角料 |
| 外交系统→**最简单版（宣战/和谈）** | AI 文明雏形即可 | 完整外交是 3-4 人月的大活 |
| 物流系统→**手动补给线** | 后期才需要的自动化物流 | 3-4 人月 |
| 多层级 UI 皮肤→**一套干净 UI** | 先保证可用，好看后补 | 2-3 人月 |
| 演出/叙事→**全靠涌现** | 不用写故事，让系统自己生成 | 2-3 人月 |

**砍完后估计**：编程 ~25-35 人月 + 美术 ~6-10 人月 = **单人约 2.5-3.5 年** 可 EA。

## 七、关键变量（影响 ±30%）

| 变量 | 影响 |
|------|------|
| Godot 熟练度 | 低→工期 +30%，高→基准 |
| 是否用 AI 辅助编程（Vibe Coding） | 显著加速 boilerplate 类代码，但对核心算法帮助有限 |
| 是否买素材/用开源资源 | 美术可压缩到 3-5 人月 |
| 要不要做主机版 | ×1.3-1.5 额外工作量 |
| 要不要做多人 | ×2-3 编程工作量（单人几乎不可行） |
| 是否用 Unity/Unreal 替代 Godot | 不考虑——Godot 已定 |
