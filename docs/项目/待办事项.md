## 地图编辑器方案探讨（待设计）

> 触发原因：当前地图用 Godot 场景（.tscn）存储，手动编辑效率低，需要专门工具。

**核心问题：地图用什么格式存储？编辑器如何工作？**

### 待选方案

1. **直接修改场景文件**（当前方式）
   - 优点：Godot 原生支持，所见即所得
   - 缺点：手动编辑繁琐，无法批量生成，易出错

2. **独立编辑器程序**（@tool 脚本 / 独立 Godot 项目）
   - 编辑器输出结构化数据（JSON/自定义格式），游戏运行时转译为场景
   - 优点：可控性强，支持程序化生成参数
   - 缺点：开发成本高，需维护转译层

3. **Godot 编辑器插件**（@tool + EditorPlugin）
   - 在 Godot 编辑器内提供自定义 Dock，辅助放置地形/装饰/建筑点位
   - 输出 .tscn 或 .json
   - 优点：复用 Godot 编辑器能力，无需独立项目
   - 缺点：插件 API 学习成本

### 待决策问题

- 地图数据是否应该从 .tscn 迁移到 JSON/Resource 驱动（与 xlsx 数据驱动一致）？
- 地形/装饰/建筑点位如何参数化存储？
- Chunk 流式加载（阶段 0.8）与编辑器输出格式的兼容性？
- 是否需要支持程序化生成（如噪波地形）与手工编辑混合？
