## 一、游戏概述

### 1.1 类型定位

**管理学游戏 / 组织机构搭建模拟器。**

不是传统RTS，不是工厂游戏，不是肉鸽。核心是：**体验一个组织结构是如何从零形成的，以及管理学概念的演进。**

### 1.2 核心体验

玩家从部落的一个普通村民起步，需要自己组建"团伙"开始建功立业。逐步搭建各种组织机构（科学院、征兵站、财政部等），体验一个个组织结构是怎么形成的。

**前期**：纯手操治理一个村落/王国。

**后期**：通过科技树引入各种现代管理学工具和现代金融学工具。体验到人类几千年科技发展史的重要节点带来的科技发展与社会治理的进步。

传统RTS和工厂游戏都是纯手操的计划开采物质，不存在市场机制去调价来自动组织生产和开采。这个游戏里玩家能体验到市场机制如何取代纯手操计划。

**生产力发展的演示**：早期王权和赋税起源于提供的公共服务作用（大禹治水→治水劳役推举出共同部落联盟领袖→形成华夏早期王权和赋税），展示社会为什么会这么发展的原因。

一把对局可能持续类似《文明》和P社游戏差不多的游玩时长。

### 1.3 爽感来源

- 从弱小村民到庞大帝国治理者和世界征服者的爽快感
- 囤积大量物资和资源、王国逐渐富裕起来的爽快感
- 各个城市逐渐发展出更多宏伟建筑，极大主要城市的繁华和时光变迁的爽快感
- 数值大后期膨胀也是游戏爽感的一部分

### 1.4 美术风格

**平面2D画风**，类似《火柴人战争遗产》或《杰克驴铁匠铺》。

- 不是低多边形3D
- 不是像素风
- 火柴人风格是美术选择，不是成本妥协
- 战场背景尽量简单、资源复用
- 家园建设界面可以多下心思
- 后期从大世界地图切换到具体城市和战场，多个画风和场景的切换要做到**无缝**
- 火柴人异世界，有火柴人世界独有的历史传说史诗和建筑形式
- 接近西幻但完全弱化西方特征——美术上看不到明显的欧洲要素和东方要素，用世界通用的古代刻板印象美术

## 二、核心结构（创始人手写）

### 2.1 八层结构

游戏由八个核心层构成，**经营建设**是中央枢纽，其余各层通过不同的路径回连到经营建设：

| # | 层名 | 职责 | 玩家交互方式 |
|---|------|------|-------------|
| 1 | **经营建设** | 中央枢纽，所有循环的起点和终点 | 主要操作界面 |
| 2 | **科技层** | 研究科技、解锁新能力 | 搭建科学院、分配学者 |
| 3 | **资源层** | 采集、生产、囤积物资 | 建造生产建筑、设定产出目标 |
| 4 | **扩张层** | 获得新地块和人口 | 世界地图操作 |
| 5 | **建设层** | 建造新建筑和奇观 | 城市界面操作 |
| 6 | **组织层** | 设计指挥链、管理帝国机构 | 组织架构界面 |
| 7 | **战斗层** | 实时出兵战斗 | 2D侧视角战场微操 |
| 8 | **运输层** | 调度资源和兵力 | 物流地图操作 |

### 2.2 层间循环关系

**从经营建设出发的四条基础循环**：

```
[经营建设] → [科技层] → 为发展出更庞大的帝国打下基础 → [经营建设]
[经营建设] → [资源层] → 建设更宏伟的建筑和奇观 → [经营建设]
[经营建设] → [扩张层] → 获得更多的人口和地块 → [经营建设]
[经营建设] → [建设层] → 获得更多的人口和地块 → [经营建设]
```

**科技→组织→经营**：

```
[科技层] → [组织层] → 升级科技 → 
  解锁新兵种 / 新建筑 / 组织指挥协调能力 / 法术考古 → 
[组织层] → 协助科技投入使用 → [经营建设]
```

**资源→扩张→科技→组织→经营**：

```
[资源层] → [扩张层] → 整合资源 → 
[科技层] → 升级资源利用科技 → 
[组织层] → 管理资源 → [经营建设]
```

**扩张→战斗→资源**（核心冲突循环）：

```
[扩张层] → [战斗层] → 获得隔壁地块 → 解锁新知识和关键角色 → 
  扩张到整个小地区统一 → 
  与此同时其他强大部落也在整合 → 扩张举步维艰 → [资源层]
```

**战斗→组织→扩张**：

```
[战斗层] → [组织层] → 指挥军队 → 微操精锐单位 → 
  战斗胜利/失败 → 战后结算 → [扩张层]
```

**组织→运输→多目标回流**：

```
[组织层] → 设计指挥链和指令计划 → 
[运输层] → 自动将运输到的资源通过设计好的计划调度来投入战场或家园 → 
[资源层] / [科技层] / [战斗层] / [建设层]
```

**运输→组织**：

```
[运输层] → 管理运输组织部署和计划 → 
  运输兵力和资源 → 运输科技（书本和技术人员）→ [组织层]
```

**建设→组织→经营**：

```
[建设层] → [组织层] → 建设新建筑 → [经营建设]
```

### 2.3 战斗循环

```
[选择出征部队] → [进入战场] → [实时战斗] → [胜利/失败] → [战后结算]
```

**关键设计**：
- **单局时长**：20-30分钟（比《火柴人战争遗产》更长）
- **多线并行**：可以同时开多场战斗，玩家可以不干预其中任何一场
- **看海模式**：随时可以放大地图观察任意战场，像看蚁群打仗一样 —— 这是发现指挥链设计缺陷的核心方式
- **交战地块**：所有不完整获得的地块持续处于战备状态，漫长的国境线意味着需要同时处理多处冲突 —— 这迫使玩家不是微操每一场战斗，而是**设计好的指挥链和运输计划让前线自动运转**
- **防止地块过碎**：不完整获得的地块（交战地块）不会立即纳入领土，避免地块过碎问题

### 2.4 长期循环

```
[开局] → [征服首个邻国] → [地块更大] → 
[获得更强大的资源整理联动和规模效应] → 
[管理难度也升高 —— 防止强到滚雪球失去乐趣] → 
[通过制度和法律的叠屋架仓等累积资产来缓解管理压力，获得新的正反馈] → 
[对抗更强势力] → [统一大地图或被灭] → [可能的新开局/新战役]
```

**防滚雪球机制**：随着帝国扩张，管理复杂度非线性增长。前期靠手操就能运转的部落，到了帝国阶段需要依靠制度、法律、组织架构来维持 —— 这本身既是挑战，也是新层次的正反馈来源。

## 三、世界与设定

### 2.1 火柴人异世界

一个有法术的世界。所有火柴人本质上是一坨**黑色物质**形成的——这是一种珍惜的、需要抢夺的资源。火柴人可以是被法术（消耗黑色物质）生产出来的，也可以正常繁殖。

火柴人通过召唤穿越过来（异世界火柴人），这解决了：
- 火柴人中出现小孩和老者的设定合理性
- 人数产生受限的问题
- 研发中产生没什么心智的巨人火柴人以及异性火柴人的设定

**设定中需要考虑**：[待补充：黑色物质的具体获取方式和限制条件]

### 2.2 世界观基调

火柴人质感多变——可以像黑色油漆胶水一样，有时被丧尸病毒感染变绿、露出白骨。这种"同时具有多种质感的技巧可以反复使用"。

后期可能出现一些近现代内容和科技，但以无厘头/戏剧化的不严肃方式出现，避免前后体验差距体感过大。

### 2.3 改造人

改造人（即便是火柴人）可能产生伦理争议——起码游戏里的火柴人应该觉得这不恰当。这可以作为一个科技树分支，类似《冰汽时代》的法律分支树。需要给玩家带来道德上的选择张力。

## 五、核心系统

### 4.1 经营建设

- 类似部落模拟器，《冰汽时代》那种感觉
- 前中期应有《冰汽时代》那样的平衡张力——不是资源无限增长，而是有限资源下做出艰难决策
- 尽可能产粮食保持人口增长率和足够的人口
- 各种装备都需要实际有产线（不是点击升级，有实际的生产流程）
- 后期引入市场机制去调价来自动组织生产和开采，而非纯手操计划开采

[待补充：具体资源类型、建筑类型、产线逻辑]

### 4.2 科技系统

不是抽象的（不像很多游戏那样点一下"研究"），也不是硬核写实的。是对现实科研机构的**简单映射**。

- 玩家真的要一步一步搭建出帝国的基础教育体系
- 每个人该做什么、向谁交付成果，都需要在游戏里有详细设置
- 召唤法师作为"程序员"——探索和编译科技的限制因素
- 科技需要扩张才能获得（类似《火柴人战争遗产》征服不同地区解锁不同科技）
- 后期引入各种现代管理学工具和现代金融学工具
- 敌人也可能学会这些科技

[待补充：科技树的具体分支和层级结构]

### 4.3 部队与招兵

- 部队的招兵系统需要从头搭建
- 类似《火柴人战争遗产》——征服弓箭地区有弓箭，征服长矛地区有长矛
- 不同地区的居民有不同火柴人种类和科技种类
- 进攻时以小队为单位指挥

[待补充：兵种具体类型、招兵流程]

### 4.4 组织系统

核心卖点之一。一切管理学相关的概念都需要用到，和玩家交互的是一个个切实的火柴人。

- 玩家需要逐步体验一个个组织结构是怎么形成的
- 科技系统需要你去有机地组织科学院协调工作
- 后期管理帝国时引入**远程观察魔法**查看帝国
- 帝国各部门呈交上来的**报表**（宏观管理视角）
- 预设的粗糙组织结构，玩家可在此基础上改造
- 可复制分享组织结构配置给别人（UGC传播）
- 不同文明的最优帝国组织结构不同（农耕 vs 游牧 vs 商业 vs 海洋）

**包装机制**：当控制部落后，解锁"部落长视角"——更方便地传唤周围人、组织成群人发起行动。宏观上看起来在指挥帝国机构，底层仍然是一个一个火柴人在做简单交互。宏观层面是城池地图、填色地图、战线地图。

[待补充：指挥链具体层级、组织面板UI、各类组织机构的详细功能]

### 4.5 世界地图

- 大世界地图，十几个最多几十个地块
- 观看各个国局势和地图范围
- 各地块有各种资源、不同火柴人种类、不同科技种类
- 每个地块是一个静态势力（初期）
- 有好几个可开局地方
- 后续可以扩展地图或把地图某地块作为新的可游玩势力

## 六、战斗系统

### 5.1 基本形式

- **2D侧视角**，类似《火柴人战争遗产》
- 自动战斗，但你选中单位可以微操
- 类似《小兵步枪》——你选中的是一个**排长**，可以指挥手下向某个位置发起进攻

### 5.2 扩张即战斗

扩张是"确实一个个火柴人在战斗"——不是抽象的数值计算。

### 5.3 火柴人单位

**初期**：火柴人尽可能精准到个——哪怕几十上百个。每个火柴人有独有特性、装备和知识。

**中后期**：发展到一个个大城市后不可能全部火柴人都显示出来，会类似制式的复制人。但你切视角到某个运输队、建筑队、前线，点击角色仍能看到角色特性——**角色特性可能是在你点击时现场生成的**。

### 5.4 主观能动性

个个角色有主观能动性——可能迟疑、可能有技巧地设计战术、可能不听指挥。但**不是拒绝服从命令那种对玩家产生负面效果**。类似历史上法国骑士总是不听指挥擅自向敌人冲锋——指挥失控但结果未必是坏的。

## 九、开发策略

- 迭代式开发，先做核心原型
- 类似部落模拟器 + 《冰汽时代》的感觉
- 创始人自己全权负责工期控制
- 室友在"游戏像个游戏了且能明确交付内容"后才考虑加入
- 开发方式：AI辅助编程（Vibe Coding），不需要教程
