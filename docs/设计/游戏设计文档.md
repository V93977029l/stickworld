## 一、游戏概述

### 1.1 一句话

**火柴人版"从部落到大帝国"——亲手设计帝国每一个层级的运作方式，然后看它自己跑起来。**

### 1.2 类型

一个无法被现有类型定义的缝合怪。硬要说：

| 基因 | 体现 |
|------|------|
| P 社大战略 | 长期策略、外交、文明演化、一局 12h+ |
| 工厂自动化 | 你设计的结构自己跑，你在旁边看 |
| 无缝缩放 | 从控制一个火柴人的脸 → 缩放至帝国全图 |
| 战术 RTS | 底层战斗是真实的，不是数值结算 |
| 模拟经营 | 城市建设、资源管理 |

### 1.3 核心体验弧

你从只有几个追随者的流浪火柴人开始。**每一个齿轮都是你亲手设计的**——帝国的军事编制、科研体系、行政架构、物流网络。到后面，帝国自己运转起来了，你更多是在观察和调优。但**你随时可以"附身"到任意层级——哪怕你已是皇帝，也能缩放到一个排长身上去微操绕后搞破坏。**

### 1.4 设计支柱

详见 [`设计支柱.md`](设计支柱.md)，核心 5 条：

1. **从蝼蚁到神明的尺度跃迁** — 无缝缩放，随时回落
2. **亲手设计的自动化帝国** — 层级架构即玩法
3. **每个火柴人都在真实生活** — 微观可观察
4. **文明演化的沉浸式叙事** — 涌现式历史
5. **收藏与掌控的双重满足**

## 二、核心架构：纵切与横切

游戏系统用**两个维度**来组织：

### 2.1 纵切：八层功能域

**八层 = "你管哪个领域"**，是功能模块的分类：

| # | 层名 | 职责 |
|---|------|------|
| 1 | 经营建设 | 中央枢纽，定居点/城市管理 |
| 2 | 科技层 | 教育体系、研究机构、科技产出 |
| 3 | 资源层 | 采集、生产、市场自动调节 |
| 4 | 扩张层 | 世界地图、地块征服/外交合并 |
| 5 | 建设层 | 建筑和奇观 |
| 6 | 组织层 | **通用项目管理层级系统（见 §三）** |
| 7 | 战斗层 | 多尺度实时战斗（组织层的军事应用） |
| 8 | 运输层 | 物流网络 |

### 2.2 横切：五层级管理深度

**五层级 = "你管多深"**，是通用项目管理系统的核心。详见 [`系统/组织系统.md`](系统/组织系统.md)。

| 层级 | 行政语境 | 军事语境 | 科研语境 | 工程语境 |
|------|----------|----------|----------|----------|
| **第5层** | 帝国级 | 帝国最高统帅部 | 科学院 | 帝国工部/路桥司 |
| **第4层** | 区域/行省级 | 师级 | 研究所 | 区域工程处 |
| **第3层** | 城市/郡县级 | 团级 | 研究室 | 项目组 |
| **第2层** | 镇/村级 | 连级 | 课题组 | 施工队 |
| **第1层** | 具体个体/班组 | 排/班级 | 实验小组 | 具体工匠/班组 |

- **层级间可加可减**：标准层级间可以额外插入一层；也可以跳过某一级（比如中央直接对接县，去掉行省）
- **最底层可额外塞两层**：例如火力组→班→排都挤在第1层内
- **玩家本身是虚拟层级**：你在哪个层级操作，你自己就是那个层级的"上级"

**重要**：五层级不是只能用于军事。它平推到一切需要"把小人组织起来干一件事"的场景——建设一座桥、一场全国普查、一个科学院的持续运作，全都用同一套层级系统。只是军事语境下叫"编制/指挥链"，科研语境下叫"院所架构"，工程语境下叫"工程指挥部"。

## 三、通用项目管理系统

这是整个游戏最核心的系统。详见 [`系统/组织系统.md`](系统/组织系统.md)。

### 3.1 核心概念

**玩家在同一个面板上搭建任意类型的组织结构。** 这个组织可以是：

- 一支军队（此时叫"编制"）
- 一个科学院（此时叫"院所架构"）
- 一个工程队（此时叫"工程指挥部"）
- 一个行政体系（此时叫"官府架构"）
- 一个商队（此时叫"商号架构"）

**每个组织理论上是全能的**——军事单位也可以施工（工程兵嘛），科研单位也可以被拉去打仗（虽然打不好）。玩家给每个组织打上"标签"（军事/科研/工程/行政/商业），标签决定了主界面默认展示哪些操作，但**所有功能都可以通过展开面板调用**。

### 3.2 编制管理

在每个层级节点上，玩家配置该节点的具体编制：

- **人员组成**：几个机枪手、几个步兵、几个无人机飞手、几个法师
- **装备配置**：标配武器、护甲、工具
- **指挥官**：任命或留空（AI 自动填补）
- **自主决策权限**：该层级可以自行决定的事项范围
- **默认行为**：无指令时的自动行为
- **补给优先级**：物资分配的权重

详见 [`系统/组织系统.md`](系统/组织系统.md)。

### 3.3 UI：工作区预设

同一个面板，但不同标签的预设展示不同的 UI 布局——类似 Blender 的 Modeling/Layout/Sculpting 工作区切换。

- **军事标签**：面板上显示"进攻"、"防御"、"行军"、"呼叫支援"等按钮
- **科研标签**：显示"分配课题"、"审核论文"、"调拨经费"
- **工程标签**：显示"开工"、"验收"、"材料调度"
- 但**点开完整功能列表**，你会发现所有标签下的组织都有同样的底层能力——科研单位也能被命令"进攻"（只是你通常不会点这个按钮）

## 四、无缝缩放与阶段演进

### 4.1 无缝缩放

玩家滚动滚轮，画面从**看着一个火柴人的脸** → 连续缩小 → 看到整个帝国版图。

- 不同缩放级别自动适配界面和信息密度
- 最微观：2D 侧视角/俯视角，可以看到火柴人的个体动作
- 最宏观：政治地图色块 + 战线 + 报表

地图概念上是同一张，技术上通过无缝加载实现不同层级的地图切换。

### 4.2 阶段演进（不是独立系统）

之前文档中提到的"孢子式 5 阶段"不是一个独立的系统——它是玩家管理范围自然扩大后的**结果描述**：

| 你管多宽 | 你的身份 | 你主要操作在哪个层级 |
|----------|----------|---------------------|
| 几个追随者 | 流浪者 | 无层级，纯手操个体 |
| 一个定居点（村） | 部落首领 | 第1-2层 |
| 几个城市（县→城） | 城主/小王 | 第2-3层 |
| 一个区域（行省） | 国王 | 第3-4层 |
| 整个已知世界 | 上古魔方持有者/皇帝 | 第4-5层 |

**关键**：即便你是皇帝，你也能缩放到第1层去"附身"一个排长微操绕后。阶段不是锁死你的操作层级——只是描述了你的帝国自然到达了哪个规模。

过渡触发不是"升时代"——是当旧层级的管理变得繁琐时，新层级的工具自然解锁。是**推而非拉**。

## 五、世界与设定

详见 [`世界观设定.md`](世界观设定.md)。

- 火柴人由"黑色沥青"构成——稀有战略资源
- 不同地域的火柴人有不同特征（平原/火山/源流/荒漠/海洋/森林/冰原）

## 六、各系统简述

### 6.1 战斗系统

详见 [`系统/战斗系统.md`](系统/战斗系统.md)。

战斗是通用项目管理系统在军事领域的应用。核心特点：

- **底层真实战斗**：不是数值结算，火柴人在 2D 战场上真实交战
- **战术可微调**：掩体、压制、侧翼、撤退、班组战术 AI——都是玩家可配置的
- **附身指挥**：你可以"附身"到任意层级的指挥官，真刀真枪地操控他移动/战斗，也能打开该层级的管理面板做决策
- **看海**：随时可以放大观察任意战场，像看蚁群打仗

**城镇就地变战场**：战斗触发时**不切换场景**——城镇地图上就地挂载战斗实例，建筑继续显示且可被破坏。玩家能体验"家园被袭"的紧张感，战斗结束自动恢复城镇模式。

**三层命令系统**：单位 AI 决策来源分三层（高优先级覆盖低优先级）：
1. **玩家/指挥链指令**（最高）—— 通过预设号令下达
2. **编制默认战术**（中）—— 由 `自主决策权限` 控制
3. **单位本能**（最低）—— 受击反击、找掩体等生存反应

**战场导演系统**：小兵步枪式灵动性由两层实现——
- **行为层**：每个战斗行为内置可配置概率钩子（如擅自冲锋 5%、犹豫 3%）
- **情绪层**：战场导演周期性（2~5s）给单位打情绪标签（HESITANT/EXCITED/PANICKED/STEADY），影响行为选择权重

情绪概率受指挥官能力、部队士气、文化传统、自主决策权限影响。

**指挥链延迟**：命令从 L(n) 下达到 L(n-1) 有延迟，模拟传令时间。玩家附身该层级指挥官时延迟为 0（直接指挥）。

**预设号令**：P0 阶段提供前进、冲刺、坚守、撤退、找掩体、压制射击、侧翼包抄、集结溃兵等号令，玩家可对单排或全队下达。

参考：《小兵步枪》的排长指挥+士兵 AI 灵动性+呼叫支援、《英雄连》的战术深度、《火柴人战争遗产》的 2D 侧视角战斗。

### 6.2 经济系统

详见 [`系统/经济系统.md`](系统/经济系统.md)。

- **价格信号融入所有系统**：你设计的开采单位被设定为自负盈亏，民间商队逐利——经济系统不是独立面板，是自然涌现的
- 从手操分配 → 市场调节 → 宏观参数调整
- 即便游戏初期（你还是一个小个体时），经济系统就已经存在——你是被市场玩弄的小鱼

### 6.3 扩张系统

详见 [`系统/扩张系统.md`](系统/扩张系统.md)。

参考《合战忍者村》和《史前埃及》的扩张模式：

- **初期**：在一个小区域（村子）内扩张，统一后该区域成为你的"基地"
- **中期**：地图扩大到整个地区（日本/埃及），原来的基地缩小为地图上的一个点
- **后期**：地图继续扩大，已统一区域成为你的核心领土，新区域等待征服

**旅行系统**：玩家在已探索区域间有三种移动方式——
- **步行**：村落之间通过 `道路地图` 连接，玩家可亲自走过去。道路本身也是横向卷轴场景，后期可触发路上事件（伏击、商人、流民）
- **快速旅行**：在大世界地图上选择已探索的目标村落/城市，显示"行进中"过场并消耗游戏内时间。**设定里解释为主角走过去了**，只是省略过程。战斗中禁止快速旅行
- **传送**：进入大型建筑（宫殿、地下城）触发，切换到独立的大建筑内部场景

### 6.7 场景与地图系统

详见 [`../技术/架构/场景与战斗架构.md`](../技术/架构/场景与战斗架构.md)。

**地图类型**：游戏世界由六类地图构成——

| 类型 | 用途 | 特征 |
|------|------|------|
| 村落地图 | 定居点/小城邦 | 横向卷轴 + Y 轴自由度，重复地面纹理 |
| 战场地图 | 战斗专用 | 更长更空旷，两端有建筑/阵地 |
| 道路地图 | 村落间过渡 | 横向卷轴，可触发路上事件 |
| 室内地图 | 小型建筑内部 | 挂载在建筑内，墙面透明化 |
| 大建筑内部 | 宫殿/地下城 | 独立完整场景，传送切换 |
| 山坡地图 | 山坡/阶梯地形 | **独立逻辑**：相机可倾斜、ground_y 按 X 变化、台阶处需工程队搭爬梯，不复用村落卷轴算法 |

**Chunk 流式加载**：每张地图由若干固定宽度的 Chunk（约 1~2 屏宽）拼接。玩家接近末端触发器时预加载下一 Chunk，跨入后卸载旧 Chunk。地图末端 Chunk 的出口触发器加载相邻地图。

**山坡地图独立原则**：山坡/阶梯/倾斜地形走独立 `SlopeMap` 类型，不污染通用村落地图架构。普通地图保持简单，不为山坡让步。山坡地图实际开发时再实现。

**建筑复用机制**：建筑**型号**决定场景文件（一种型号一个 `.tscn`），**等级**决定数据配置（同型号不同等级 = 同场景 + 不同 Resource 数据）。各村落的通用建筑共用同一套模板，每个村落另有 1~2 个独有建筑（独有贴图）。

**建筑三层架构**：游戏内建筑分三层和平共存--
- **地形建筑层**：随地图场景打包，不可拆除（矿山、遗迹、大石头）
- **初始建筑层**：关卡设计时放置，首次进入地图写入存档，可拆除（关卡预设的民居）
- **玩家建筑层**：玩家在游戏中建造，存档 JSON 持久化，可拆除

三层共用同一套建筑场景文件，通过 PlacementGrid 统一占位登记，加载时机不同但互不冲突。

**贴图与占地解耦**：建筑贴图尺寸任意，可以比占地大或小。一棵树占地 1×1（32px 树根），贴图可 96×192px（树冠很宽很高），Sprite2D 用 offset 让树冠向上展开。贴图超出占地范围不冲突，PlacementGrid 只校验占地格子。

**BuildMask 不可放建筑区域**：地图设计时标记"地形上不可放建筑"的格子（大石头、山坡阶梯处、装饰物占地），用纯数据 blockage_mask 表示，不污染建筑系统。玩家尝试在此放建筑时提示冲突，与建筑占用同错误信息。

**通行障碍系统**：火柴人移动受两级透明障碍约束（运行时不可见，调试模式可可视化）--
- **地图级 WalkBarrier**：悬崖/高楼边缘/断崖，蓝色半透明显示
- **建筑级 PassageBarrier**：建筑本体不可通行区域，紫色半透明显示

典型用例：雕像上下可绕过（仅基座障碍）、房屋只能从下方过（上半部分障碍）、矿山完全不可穿过（全覆盖）。建筑 PassageBarrier 形状可与 Footprint 不同，设计师在建筑场景中自由绘制。

**调试覆盖层**：F3 切换显示所有运行时不可见的标记（占地格/障碍/地面线/触发器等），启动时屏幕左上角常驻小字号图例说明各颜色含义。是后续开发必备工具。

**占地网格**：建筑采用红警式网格化占地，单元尺寸 32px。每个建筑占据 N×M 个网格单元（小屋 2×2、兵营 3×3、宫殿 8×6）。一块地只能建一个建筑，选址时实时校验网格冲突。

**村落差异化**：通过 `村落数据文件` 配置抹除单调感——城墙高度、地面色相微调、装饰物密度、独有建筑、环境音等参数。七八个村落 = 七八个数据文件，复用同一套建筑模板。

**室内场景（双层透明化）**：小型建筑（铁匠铺、住宅等）采用双层结构——室外层（墙、屋顶）和室内层共存。玩家走进建筑交互区时，前墙半透明化（modulate.a → 0.3），室内层显示，玩家可自然进入。**不是传送，室外继续正常显示**。玩家能看见"师傅在打铁"。

**大建筑传送**：宫殿、地下城等大型建筑走传送——进入触发器后切换到独立的大建筑内部场景。

**环境系统跨场景保持**：天空（极光、星星、太阳/月亮 Shader）、天气（雨/雪/沙尘暴 GPU 粒子）、光照（CanvasModulate 按时间插值）、地面震动等效果挂在常驻主场景上，地图切换时环境继续工作。环境参数受地理位置（生物群落）和游戏时间双重影响。

**主场景架构**：采用 `GameRoot` 常驻容器 + 子场景动态加载。主场景持有环境系统、相机、UI、输入分发、时间管理、场景加载器、世界分块宿主、战斗调度器等常驻节点。地图作为子场景动态加载/卸载。这是实现"城镇就地变战场"、"快速旅行状态延续"、"无缝过渡"的工程基础。

## 七、法律与道德系统（冰汽时代式）

### 7.1 法律树

部分法律选择会开启道德分支树。每条法律有：

- **可量化效果**（+20% 生产力）
- **不可量化的叙事后果**（民众舆论、国际声誉、历史评价）
- 法律**可以废除**，但废除有代价（适应期混乱、利益集团反弹）

### 7.2 改造人伦理分支（可选灵感）

改造火柴人可能产生伦理争议。可作为科技树中的一个法律分支，类似《冰汽时代》的道德选择。详见 [`系统/科技系统.md`](系统/科技系统.md)。

## 八、时间控制

类似 P 社游戏的多级时间流速：

- 暂停 / 1x / 2x / 4x
- 可设置**自动暂停**（战斗开始/重大事件/法律颁布时）
- 可设置**自动减慢**（切到微观操作时）
- 玩家随时手动控制
