# 河流算法需求文档

> 本文档记录用户对河流生成算法的完整诉求。实现前必须逐条确认，任何方法变更必须征求用户同意。

---

## 一、总体目标

在 8192×8192 全量高度场上，生成覆盖整个大陆的树根状河流网络。河流网络由若干主干河及其支流组成，每个主干河拥有自己的流域，流域覆盖满整个大陆，所有河流最终汇入海洋。

---

## 二、分步流程与预览

每一步必须生成独立的预览图供用户检查，确认无误后再进入下一步。

### 步骤1：流域分割（Watershed Partition）

- 将整个大陆划分为 N 个流域（N 条主干河）
- 每个流域覆盖满整个大陆陆地（无遗漏、无重叠）
- 类比：中国华北所有河流汇入黄河，华南所有河流汇入长江/珠江
- 流域分割基于高度场的流向：每格的水最终流向哪个海岸出口，就属于哪个出口的流域
- **预览图**：每个流域用不同颜色着色，显示流域边界和覆盖范围

### 步骤2：降雨噪声图

- 生成降雨量分布图（fBm 噪声，值域 0~1）
- 降雨量影响每格的水量贡献
- **预览图**：灰度图或彩色图显示降雨分布

### 步骤3：流量累积

- 每格的水量 = 该格降雨量
- 沿流向累积：每格流量 = 自身降雨量 + 所有上游格子的流量之和
- 流量累积必须覆盖全量 8192×8192，不降分辨率
- **预览图**：对数映射灰度图，显示每格的累积流量

### 步骤4：河流提取与宽度渲染

- 流量超过阈值的格子才被视为河流
- 最细的河流宽度至少 1 像素
- 河流宽度 = sqrt(径流量) × 缩放系数
- 主干最粗，支流很细
- 颜色深浅表示径流量（主干深蓝，支流浅蓝）
- **预览图**：群系底图 + 河流叠加（宽度+颜色深浅）

---

## 三、河流路径形态要求

### 3.1 自然弯曲

- 河流必须弯曲自然（歪七扭八），不是直线
- **禁止** 8 方向横平竖直
- **禁止** 45 度锯齿
- **禁止** 180 度回折
- **禁止** 自重叠/自交叉

### 3.2 矢量路径 + 栅格化

- 河流路径在连续空间（浮点坐标）中生成
- 栅格化后才到网格上
- 路径用样条曲线或分形中点位移平滑

### 3.3 树根状分叉

- 每个流域内只有一条主干河
- 主干沿途分叉大量支流
- 支流再分叉，递归 2-3 层
- 分叉角度约束在 30-60 度
- 树根状覆盖满整个流域

---

## 四、降雨与径流量计算

### 4.1 降雨噪声

- 用 fBm 噪声生成降雨量分布
- 每格降雨量 ∈ [0, 1]

### 4.2 流量累积

- 每格流量 = 自身降雨量 + 所有上游格子流量之和
- 流向由高度场决定（水往低处流）
- 河流在某一点的水量 = 上游流域面积 × 平均降雨量

### 4.3 河流宽度

- 宽度 = sqrt(径流量) × 缩放系数
- 主干最粗（径流量大），支流很细（径流量小）
- 最细河流宽度 ≥ 1 像素

---

## 五、禁止事项

1. **不要内流河**：所有河流最终流入海洋（深蓝色水域）
2. **不要湖泊**：不生成湖泊
3. **不要降精度**：8192×8192 全量计算，不降分辨率
4. **不要私自换方法**：任何算法变更必须征求用户同意
5. **不要因为计算量大就切换方案**：如果算不过来，先问用户

## 六、技术约束

### 6.1 计算精度

- 8192×8192 全量计算
- 不降分辨率
- 不使用近似算法替代精确算法

### 6.2 性能

- 如果纯 Python 太慢，可以用 C 加速器
- C 加速器必须保证计算精度不变
- 性能优化不能牺牲算法正确性

### 6.3 数据结构

- 河流路径存储为矢量数据（路径点列表）
- 栅格化只用于预览渲染
- 流域分割结果存储为 int32 标签场（每格属于哪个流域）
- 流量累积结果存储为 float32 场
- 降雨量存储为 float32 场

## 七、算法选型参考

用户提到的参考算法和技术路线：

1. **流域分割**：Watershed Partition（基于 D8/D∞ 流向的反向追踪）
2. **流量累积**：拓扑排序（Kahn 算法）或 Priority-Flood
3. **河流路径**：矢量路径 + 栅格化（非网格上的 8 方向追踪）
4. **分形弯曲**：中点位移法（Midpoint Displacement）或 DLA
5. **科学软件参考**：RichDEM、WhiteboxTools、TauDEM

## 十、已尝试的算法与失败原因（教训记录）

### 10.1 D8 流向 + 流量累积

- **方法**：8 方向最陡下降流向 + Kahn 拓扑排序流量累积
- **失败原因**：
  - 只有 8 个离散方向，河流呈横平竖直/45 度锯齿
  - 相邻格流向相同方向，产生"梳子状"平行流
- **结论**：D8 不适合，8 方向网格追踪天然产生不自然的河流形态

### 10.2 D∞ (D-Infinity) 流向

- **方法**：Tarboton 1997，连续角度流向，流量按比例分配到两个方向
- **失败原因**：
  - 在 8192 上纯 Python 向量化太慢（D∞ 流向计算 10+ 分钟未完成）
  - 改用 C 加速器后完成，但视觉效果仍像 D8（用户反馈"还是 D8"）
- **结论**：D∞ 理论上消除锯齿，但实际效果不明显，且流量分散导致河流不清晰

### 10.3 中点位移法（Midpoint Displacement）

- **方法**：在出口和终点之间递归中点位移，生成弯曲矢量路径
- **失败原因**：
  - 偏移量太大（res×0.03=245像素），产生 180 度回折
  - 支流方向 360 度随机，导致自重叠
  - 没有自交叉检测
- **结论**：中点位移法不适合生成分叉网络，只适合单条弯曲曲线

### 10.4 连续空间生长算法

- **方法**：从海岸出口出发，方向 = 梯度(60%) + 惯性(25%) + 随机(15%)
- **失败原因**：
  - Azgaar 高度场由 BFS 传播生成，梯度始终指向"远离海岸"的同一方向，河流走直线
  - `rasterize_all` 按 level 找下一个点时跳过了支流点，导致"飞雷神直线"（远距离跳跃画线）
  - 自交叉检测太严格，河流太短（429 个路径点，平均 36 点/河）
- **结论**：Azgaar 高度场的梯度太规律是根本问题；路径点存储和栅格化逻辑也有 bug

### 10.5 关键教训

1. **Azgaar 高度场梯度太规律**：BFS 传播生成的高度场，梯度始终指向海岸反方向，河流沿梯度走就是直线。需要用降雨噪声/随机扰动打破这种规律性。
2. **纯 Python 在 8192 上太慢**：D8 流向向量化 10+ 分钟，拓扑排序逐格处理更慢。必须用 C 加速器。
3. **subprocess.capture_output 会阻塞**：C 程序的 stderr 输出导致管道缓冲区满。改用 `subprocess.call`（不捕获输出）解决。
4. **tempfile 磁盘慢**：`tempfile.mkdtemp` 创建的临时目录在慢磁盘上，写 256MB .bin 文件很慢。改用固定路径 `.tmp/`。
5. **不填洼导致流量累积卡住**：局部最低格没有流向（flow_dir=-1），拓扑排序可能不完整。但 Azgaar 高度场本身有梯度，局部最低处少。
6. **河流路径点和栅格化必须一一对应**：不能按 level/river_id 跳跃查找下一个点，必须按路径顺序连续存储和连接。

## 十一、下一步实现计划

> 以下为实现顺序，每步完成并出预览图后，必须等用户确认才能进入下一步。

### 步骤1：流域分割

**输入**：`locked_heightmap_8192.npy` + `locked_continent_8192.png.npy`
**输出**：`int32` 流域标签场（每格属于哪个流域 ID）+ 预览图
**方法**：
1. 在 8192 全量高度场上计算 D8 流向（C 加速器）
2. 找海岸格（陆地+海洋边界）
3. 贪心选 N 个均匀分布的海岸出口（最大最小距离）
4. 从所有海岸格反向 BFS：每格归属其水流最终到达的出口
5. 流域覆盖满整个大陆（每格陆地都属于某个流域）
**预览图**：每个流域用不同颜色着色

### 步骤2：降雨噪声图

**输入**：`seed`
**输出**：`float32` 降雨量场 + 预览图
**方法**：fBm 噪声，值域 [0, 1]
**预览图**：灰度图

### 步骤3：流量累积

**输入**：D8 流向 + 降雨量场
**输出**：`float32` 流量累积场 + 预览图
**方法**：
1. 每格初始流量 = 降雨量
2. 拓扑排序（Kahn 算法，C 加速器）：每格流量 += 上游来水
3. 全量 8192×8192
**预览图**：对数映射灰度图

### 步骤4：河流提取与宽度渲染

**输入**：流量累积场 + 流域标签场
**输出**：`float32` 河流径流量场（含宽度信息）+ 预览图
**方法**：
1. 流量超过阈值的格子才被视为河流
2. 河流宽度 = sqrt(径流量) × 缩放系数
3. 最细河流 ≥ 1 像素
4. 颜色深浅 = 径流量（主干深蓝，支流浅蓝）
5. 栅格化时画粗线（圆形画笔，半径 = 宽度/2）
**预览图**：群系底图 + 河流叠加
