## 〇、设计原始要求（创始人输入存档）

> 本节逐条记录创始人对程序化世界生成的原始设计要求，作为后续实现的约束真相源。任何实现偏离本节需经创始人确认。

### 0.1 素材策略（取代"AI 直出大图"）

- **不采用** AI 一次性生成完整世界大地图：太随机、粗糙、美术风格不统一、反复抽卡。
- **改为**：AI 生成可拼接的小素材 → 人工挑选（当质检员）→ 程序当装配工。
- 程序化生成各城镇/城市的**游戏内地图**，以及整个世界用于观看的**世界地图画布**。

### 0.2 三级地图层级

| 层级 | 名称 | 对应现实 | Demo 进度 |
|------|------|----------|-----------|
| L3 | 大世界（World） | 一个大陆 + 周围海洋 | 纯线条/文字可视化，验证算法 |
| L2 | 地区（Region）/ 城市群 | 省级 / 城市群 | 程序化生成 |
| L1 | 地块（Tile） | **地级市**（不是省） | **Demo 优先解决**：一个城市 + 周围乡村 |

- 三个层级**都需要程序化生成**。
- **子地图外边界轮廓必须与母地图该地区的轮廓保持一致**（层级细分一致性）。

### 0.3 地块划分方式

- **自然、不规则**的地块划分（非固定网格）。参考 P 社 / Azgaar 的不规则省份。

### 0.4 地区层级（L2）要求

- 整个地区**基本上是一个气候**，但不排除局部变异（南草北水）--气候变异**天然形成**（温湿噪声局部极值），不强制。
- 地区类似**行政区划**：在大世界地形图上切割为若干地区，**尽量山地归山地、平原归平原**（地形同质化）。
- 但**经济耦合**优先：如水乡难以单独存在、与背后平原紧密联系（彼此经济互赖），即便分属不同政权互相战争，也应**划入同一地区**。
- 地区是游玩时**相对完整的行政区划单位**。
- **地区-政权关系**（见 §5.5、§15.7）：
  - 小政权：占一个地区，或几个小政权**挤在一个地区**（相似度高=同文化=联盟候选）。
  - 大政权（帝国）：横跨多个地区；多数时候拥有完整地区，或地区边界与帝国相符。
  - 若一个地区被**两个大政权**分割 -> 沿政权边界**切分为两个地区**。
  - 几个小政权共占一地区 -> **不切分**（本就是碎的预期状态）。
- 地区界面：在对应地块中心显示 **Q 版小缩略图标**，对应地块的主要建筑/景观（大矿坑、魔力源泉等）。

### 0.5 地块层级（L1）要求

- 一个地块内只有**一个中心城市**，通常在地块中心。
- 中心城市位置例外：若靠山 → 可能是矿山城市（位于地块边缘）；若靠河 → 可能是渔业城市（位于地块边缘）。
- 中心城市**有城墙**。
- 周围有村落，**村落数量不固定**，取决于周围资源禀赋和地块实际大小：可能一两个，也可能大平原上七八个。
- **相当一部分资源禀赋够的地方也没有村落**（有资源但无人定居）。
- 玩家有需求（如军事需求）时**可建造新村落或城市**。
- 村落一般很小，**偶尔出现一个大部落级别的村落**。
- 部分地区过于贫瘠：**一个城市级别的聚落都没有**。
- 部分地区过于富庶：**有两个城市级别的聚落**，但中心城市仍然只有一个（对标辽宁沈阳 vs 大连）。
- 地块对标的是**地级市**而非省：一个地级市内可能有多个县城，部分县城发展程度可能和地级市差不多。

### 0.6 聚落规模与级别

- 常规建筑有 **4 种**，分别对应 **4 种级别的聚落**。
- **帝国首都的建筑另行单独设计**（不走常规 4 级）。
- 生成时先**计算聚落的大概规模**；规模大到一定程度就是城市。
- 不同规模的聚落可能是**数量级级别的差距**。
- **先预设好聚落级别**，再在该级别内部的规模上下限内生成。
- **绝大部分聚落偏向下限**（现实中大人口聚落很少）。

### 0.7 城市级聚落的支柱产业

- 一个城市级别的聚落通常有**一个或多个支柱产业**：采矿、贸易等。
- 甚至可能**单纯作为帝国的前线战争堡垒和要塞**、重要的前哨站、或一个考察点。

### 0.8 地形对聚落的影响

- 部分地形（如高原）可能使**整个地块或地区的整体聚落规模甚至密度都偏小**。

### 0.9 核心设计疑问（待本文档回答）

> "该怎么科学地生成自然的？从资源开始生成层层叠叠上去然后再根据资源禀赋生成城市吗？那由于绕了很大一弯子很难让我前面的约束设计出来；直接按照我的约束生成吗？那可能太不自然了，即便我的约束已经考虑了一部分自然条件，但我很难穷举。虽说是程序化生成，但其实毕竟不是开放世界，对应的随机生成其实更一次性的。"

→ 本文档 §三 给出答案：**分层适宜度模型 + 可注入设计意图**（既不纯资源推、也不纯约束摆）。

### 0.10 大世界层级（L3）Demo 要求

- 大地图 Demo 版**所有元素用纯线条和文字描述可视化**，勉强可视化即可，用于检验生成算法打磨到位。
- **只需把地形、地块群系、河流、海洋生成明白**即可。
- 大世界是一整个大陆，**周围一圈海洋**。
- 大陆地形**最好复杂一点**。
- 最高层级的程序化生成**随机味不要那么重**。
- **先给一个随机程序批量生成大陆轮廓，创始人选定一个大致构型作为后续细化的模板**。
- 参考网上自制架空世界地图的方案和开源程序方案。

### 0.11 边界一致性

- 子地图生成的**外边界轮廓要和母地图该地区的轮廓保持一致**。

### 0.12 聚落间道路

- 聚落间需要生成道路连接。

### 0.13 政权与联盟（神罗式碎国）

- 生成各政权的大致规模和边界。
- 国家规模**略小**；整块大陆要有**多个联盟或单一政权**。
- **联盟** = 同文化的多个小王国集合，外人视为完整单一国家（神罗式）；便于初期扩张。
- 不排除有**庞然大物的单一政权**，但也没多大，**不可能有占小半块大陆的初始国家**。
- **Demo 出生地区内一定要有好几个小政权**，要更碎一点。

### 0.14 后处理与多职能

- 生成完国家后**后处理微调**各聚落的规模和职能。
- 确定聚落在该国家内的**大致重要程度**，影响其他细节生成。
- 部分城市可**多个职能**（如水城 + 河军事重镇）。
- 部分地区**不需要堆军事力量**（如一片难以跨越的地形保护的内部）。
- 贫瘠地块（沙漠等）可能整地块就一两个小村庄级聚落，**这两个小村庄当"首富"**（地块事实中心）。

### 0.15 每局轻微随机（仅规模）

- 位置扰动、控制区边界重划、±1 聚落：**全部作废**。
- 仅聚落**规模**可在 ±15% 范围内扰动（出生村落+中心城市免疫）。
- 地形/政权/聚落数量/位置/级别/控制区边界：开发期抽卡定死后**永不再变**。

### 0.16 出生点规格

- 出生地区：**标准平原**，有河流、小点的山、贸易。
- 出生地块：**偏标准**--一个中心城市 + 周围 6-7 个村落。
- 地块层面实际游玩时，以**行政区划（聚落）为单位**进行操作。

### 0.17 地图结构（已确认）

- 世界地图上聚落是地块内的一个子单元（点/小领地）；进入聚落后抽象为纯横版世界（横向卷轴矩形）。边界一致性作用于 L3->L2->L1 领土多边形，可玩聚落地图为矩形游戏投影。

### 0.18 生成定位（开发期工具，非运行时）

- 程序化生成是**开发期工具**：用程序化方法保证生成更自然，不是为了反复生成。
- 开发者抽卡（不满意可重抽几次），定下来后**永不再改**，随包发布。
- 送到玩家手里的地图已**几乎完全定死**，起码地形固定。
- 玩家运行时只有：聚落规模 ±15% 扰动（§0.15）+ 按需生成可玩聚落地图（从锁定种子，确定性）。

## 一、核心理念

程序化生成的铁律是**确定性**：同一 `world_seed` 必须产出同一世界。一切生成器从根种子派生子种子，形成种子树。

```
WorldSeed (根)
├── continent_seed      -> 大陆轮廓 + 板块 + 地形
├── world_map_seed      -> L3 划分（地区）+ 生物群落 + 河流
│   └── region_seeds[]    -> 每个地区一个子种子 -> L2 划分（地块）
│       └── tile_seeds[]    -> 每个地块一个子种子 -> L1 聚落布局
└── settlement_seeds[]  -> 每个聚落一个子种子 -> 横向卷轴可玩地图
    └── chunk_seeds[]     -> 每个 chunk 一个子种子
```

| 层级 | 几何形态 | 内容 | 生成产物 | 可视化（Demo） |
|------|----------|------|----------|----------------|
| **L3 大世界** | 大陆轮廓 + Voronoi 地区多边形 | 大陆/海洋/山脉/河流/生物群落/地区划分 | `continent_mask.png` + `WorldMapData` | 纯线条+文字 |
| **L2 地区** | 地区多边形内受限 Voronoi 地块多边形 | 地块群系/气候/Q版地标图标/聚落点位 | `RegionData` + 地块多边形 | 纯线条+图标 |
| **L1 地块** | 地块多边形内聚落布局 | 中心城市 + 村落 + 资源 + 地形 | `TileData` + 聚落列表 | 纯线条+图标 |

## 三、生成哲学：分层适宜度模型 + 可注入设计意图

### 3.1 回答创始人的核心疑问

既不"纯资源推聚落"（绕弯子、约束难施加），也不"纯约束直接摆"（不自然、难穷举）。

**答案是：自然层驱动适宜度场，约束作为可注入的偏置/保证叠加其上，聚落从偏置后的场中采样涌现。**

这是 Dwarf Fortress / RimWorld 世界生成 / Azgaar 聚落生成的通用范式：
- 地理（地形/气候/河流/资源）是**纯自然层**，确定性生成，不含任何聚落。
- 聚落位置由**适宜度场**决定，适宜度从自然层派生——所以"自然"。
- 设计约束（"这地区要有前线要塞""这地块中心城市靠山"）作为**偏置**或**保证项**注入，不替代自然，只扭曲概率——所以"约束可施加"。
- 聚落从偏置后的场采样，带间距约束 + 级别分桶 + 下限偏置——所以"涌现且可控"。

### 3.2 适宜度场

每个候选位置计算一个连续 `suitability` 分数：

```
suitability = w1 * water_access        # 近河/湖/海岸
            + w2 * arable_score        # 可耕地（平原+适宜温湿）
            + w3 * resource_score      # 周边资源禀赋（矿/沥青/魔力泉）
            + w4 * defensibility       # 防御性（背山/扼守隘口/高地）
            + w5 * trade_access        # 贸易可达性（河口/道路交汇/沿海）
            - w6 * harshness           # 恶劣度（高原/极寒/荒漠惩罚）
            - w7 * hazard              # 灾害（火山/沼泽/洪水）
```

权重 `w*` 放在 `biome_rules.tres` / `settlement_rules.tres` 数据表，调"手感"只改表。

### 3.3 设计意图注入

两种注入方式，互斥可叠加：

| 注入类型 | 语义 | 实现 | 示例 |
|----------|------|------|------|
| **偏置（Bias）** | "这里更可能出某类聚落" | 给某区域某产业适宜度加权 | 高原区 harshness 加倍 → 聚落稀疏 |
| **保证（Guarantee）** | "这里必须有一个某类聚落" | 强制放置 + 锁定级别/产业 | 边境地块强制 1 个要塞城市 |

保证项优先于采样，偏置项融入采样概率。创始人约束大多可表达为"地区级保证 + 地块级偏置"。

### 3.4 采样与级别分桶

```
1. 保证项先放置（强制聚落，锁定级别/产业/位置约束）
2. 在剩余区域按 Poisson-disk 间距采样候选点（避免聚落簇拥，除非意图如此）
3. 每个候选点采适宜度 -> 连续 population_score（长尾分布，幂律）
4. 分桶：
   score < v1            -> 不定居（有资源也无人）
   v1 ≤ score < v2       -> 村落 T1
   v2 ≤ score < v3       -> 镇/大部落 T2
   v3 ≤ score < v4       -> 城市 T3
   score ≥ v4            -> 中心城市候选 T4
5. 地块内最高分聚落若达 T4 阈值 -> 升为该地块中心城市（仅 1 个）
   未达阈值 -> 该地块无中心城市（贫瘠地块）
6. 帝国首都：覆盖规则，指定聚落升为 T5（单独建筑集）
```

**下限偏置**：每个桶内的 `population_score` 用 `lower + (upper-lower) * u^k`（k>1，如 k=2）映射，使大部分聚落贴近下限，少数逼近上限——符合"大聚落稀少、数量级差距"。

## 四、聚落系统设计

### 4.1 聚落级别

| 级别 | 名称 | 数量级特征 | 出现频率 |
|------|------|-----------|----------|
| T1 | 村落 | 最小，最常见 | 高 |
| T2 | 镇 / 大部落 | 偶尔一个村落长成大部落 | 中低 |
| T3 | 城市 | 有支柱产业，数量级大于村落 | 低 |
| T4 | 中心城市 | 地块首要城市，有城墙，0或1个/地块 | 极低 |
| T5 | 帝国首都 | 特殊，建筑另行设计 | 手动/特殊放置 |

> 注：常规 4 级建筑对应 T1–T4；T5 单独设计。

### 4.2 支柱产业

T3+ 聚落分配 1+ 支柱产业，依据自然层 + 意图：

| 产业 | 触发条件（自然或意图） |
|------|----------------------|
| 采矿 | 周边有矿脉/沥青矿 |
| 渔业 | 靠河/靠海岸 |
| 贸易 | 河口/道路交汇/沿海良港 |
| 农业 | 大平原高可耕地 |
| 魔力 | 邻近魔力源泉 |
| 军事要塞 | 边境/扼守隘口（多为保证项） |
| 前哨站 | 文明边缘（保证项） |
| 考察点 | 特殊地标附近（保证项） |

### 4.3 中心城市位置规则

- 默认地块几何中心。
- 若地块靠山且资源为矿 → 中心城市靠山侧边缘，产业=采矿。
- 若地块靠河/海 → 中心城市靠水侧边缘，产业=渔业/贸易。
- 由 `settlement_rules.tres` 的位置偏置函数实现，不写死。

### 4.4 地块聚落构成规则

- 中心城市 0 或 1 个（取决于最高分是否达 T4）。
- 城市级聚落（T3）可 0~多个；富庶地块可出现第二个 T3（沈阳/大连型），但中心城市仍唯一。
- 村落（T1/T2）数量 = `f(资源禀赋, 地块大小, harshness)`，可能 0~8+。
- 玩家可在适宜度够但无人定居处新建聚落（沿用现有建设系统）。

## 五、大世界层级（L3）生成管线

### 5.1 大陆轮廓批量生成器（模板挑选工具）

**目标**：批量生成大陆轮廓候选，创始人选定一个作为后续细化的锁定模板（满足"随机味不重 + 可挑选"）。

**算法**（距离场 + 多倍频噪声 + 径向海洋衰减）：

```
ContinentSketcher.batch_generate(count, seed):
  for i in count:
    s = derive(seed, i)
    # 1. 基础陆地 blob：椭圆/多焦点引力场
    base = elliptical_continent(a, b, focus_jitter=s)   # 参考 Unciv 椭圆大陆
    # 2. 边缘扰动：多倍频 Simplex 扰动海岸线
    coast = perturb_edge(base, noise_freq, noise_amp)
    # 3. 径向海洋衰减：边缘 5~15% 强制海洋（保证"周围一圈海洋"）
    landmask = coast * radial_ocean_falloff(margin=0.1)
    # 4. 输出纯线条 SVG 轮廓
    render_outline_svg(landmask, "candidate_%d.svg" % i)
  # 创始人挑选 -> locked_continent_mask.png
```

参考：Unciv 的 8 种大陆模式（泛大陆/群岛/两大陆/分形…）、Azgaar 的 heightmap 策略。

### 5.2 大陆地形生成（锁定模板后细化）

**采用板块构造简化模拟**（参考 World-Synth BFS 板块生长 + Experilous + Benedetti 碰撞位移）：

```
1. 板块划分：在陆地 mask 内撒 Voronoi 种子 -> BFS flood-fill 生长板块
   （cost function 控制形状自然）
2. 板块类型：陆壳 / 洋壳
3. 边界分类：汇聚/离散/转换
4. 地形抬升：
   - 汇聚边界（陆-陆）-> 山脉（沿边界脊状抬高，参考 Benedetti 高斯位移）
   - 汇聚边界（洋-陆）-> 海沟 + 火山岛弧
   - 离散边界 -> 大洋中脊
5. 全局高度场 = base + plate_displacement + 多倍频细节噪声
6. 阈值化：陆/海/湖；高度分级：平原/丘陵/山地/高原
```

**为什么用板块**：纯噪声大陆"随机味重"且山脉分布不真实；板块构造让山脉沿边界成带、海岸线自然复杂——符合"地形复杂 + 随机味不重"。

### 5.3 气候与生物群落

参考 Azgaar / Unciv 的温湿矩阵（Whittaker 图表）：

```
temperature = f(latitude) + noise              # 纬度主导 + 扰动
precipitation = simplex_noise(octaves) + ocean_proximity_bonus
biome = whittaker(temperature, precipitation, elevation)
```

生物群落对应 GDD §五：平原/火山/源流/荒漠/海洋/森林/冰原。

### 5.4 河流水文

标准水文模型（Azgaar / Dwarf Fortress 同款）：

```
1. 每格 flow_direction = steepest_descent(高度场)
2. flow_accumulation = 自身降水 + 上游来水累加
3. flow > threshold -> 河流；沿流向追踪至海/湖
4. 河流跨地块边界一致（继承自高度场，天然连续）
```

### 5.5 L3 地区划分（地理优先 + 经济耦合）

地区是**地理-行政单位**，地理优先划分，再由政权后处理调整（§15.7）。

**划分方法**（cost 加权 Voronoi，让边界贴合地形）：

```
1. 目标地区面积 = 大陆面积 / 期望地区数 -> 种子数 N 涌现自大陆大小（非固定）
2. 撒 N 个地区种子 + Lloyd 松弛 2~3 轮 -> 不规则自然多边形
3. cost 加权：跨地形特征（山脊/大河/荒漠边界）切割 cost 高
   -> 边界倾向沿山/河走，山地归山地、平原归平原（地形同质化）
4. 经济耦合覆盖：水乡+背后平原等经济互赖区域强制并入同一地区
   （检测：水体的相邻低地且共享流域/贸易路径）
5. 裁剪到大陆轮廓；邻接图：共享边界弧段的地区互为邻接
```

每个地区继承主导生物群落（§0.4 "基本一个气候"），但允许内部局部变异（南草北水）——**天然形成**（温湿噪声局部极值，不强制）。
- 小高原 -> 小地区；大平原 -> 大地区（地区大小涌现自地理）。
- **地区数不固定**：涌现自大陆面积与目标地区面积之比。
- **地区-政权交互**：地区先按地理定，政权生成后做一次后处理切分（详见 §15.7）。

### 5.6 Demo 纯线条可视化

L3 Demo 渲染器（不改现有 [MapRenderer](../../../stick-world/modules/world_map/scripts/map_renderer.gd)，新建简化版）：

- 大陆轮廓：黑色折线
- 海洋：浅蓝填充或留白 + "海"字
- 生物群落：纯色平涂多边形 + 图例文字
- 山脉：沿边界脊线的折线/三角符号
- 河流：蓝色折线
- 地区边界：灰色虚线 + 地区名文字
- 无贴图、无纹理、无图标——只验证算法

## 八、边界一致性：受限 Voronoi 层级细分

**核心技术**：受限 Voronoi（a.k.a. bounded/restricted Voronoi），Azgaar 同款。

```
L3 大陆轮廓
  └── 地区种子(在大陆内) -> Voronoi 裁剪到大陆轮廓 -> 地区多边形（精确铺满大陆）
        └── 地块种子(在地区内) -> Voronoi 裁剪到地区多边形 -> 地块多边形（精确铺满地区）
```

保证：子层多边形的并集 = 父层多边形，边界逐像素一致。河流/道路跨层继承时按多边形裁剪，天然连续。

### 11.1 关键定位：开发期 Python 工具，非 GDScript 运行时模块

本阶段的程序化生成是**开发期一次性工具**，产出物是**图片/数据文件**（locked_continent.png、world_map_l3.png 等），交付给游戏消费。因此：

仅当某步骤需要**运行时按需生成**（如玩家首次进入聚落 -> 生成可玩地图）时，才考虑 GDScript 运行时实现，并从锁定数据派生。

### 11.4 工程备忘（踩坑记录）

1. **class_name 冲突**：早期 GDScript 版曾定义 `class_name WorldMapData`，与 `modules/world_map/data/world_map_data.gd` 已有的同名类冲突，导致生成器解析失败。迁移 Python 后此问题消失，但 GDScript 侧新增类名前务必全局检索。
2. **沙箱内存上限**：本机沙箱对子进程有内存限制（曾测约 256MB）。numpy 大数组（8192² float32 = 256MB/个）极易超限被静默 OOM-kill（exit code 0xC000013A，无 Python 堆栈）。
3. **分块(slab) + gc.collect()**：landmask/world_map 全部按行分块（slab_h=128）处理，每块后 `del` + `gc.collect()` 强制释放。否则 numpy 临时数组累积不归还 OS，几块后就 OOM。此方案使内存峰值恒定（不随图片尺寸增长），4096 实测稳定。
4. **float32 全程**：所有 numpy 运算用 `np.float32` 字面量（`np.float32(0.5)` 而非 `0.5`），避免 Python float 自动提升为 float64 导致内存翻倍。
5. **分辨率**：4096×4096 已验证可生成。8192×8192 需沙箱内存 >512MB（输出数组 biome+river+mask 共 192MB + 临时量）。沙箱调高后可跑。
6. **河流低分辨率追踪**：河流在 1024 分辨率高程上追踪再最近邻放大到全尺寸，避免全尺寸高程数组占内存。

## 十五、政权与联盟生成（L3）

### 15.1 生成时机

**地形 + 聚落之后**生成政权。顺序：自然层 -> 聚落 -> 文化圈 -> 都城 -> 政权扩张 -> 联盟 -> 后处理。

### 15.2 文化圈（联盟的基础）

```
1. 撒若干文化种子点（数量 > 预期政权数，保证同文化内能碎成多国）
2. 每种子 flood-fill 扩张，cost = 距离 + 地形难度（山/河/荒漠抬高 cost）
   -> 文化边界自然贴合地理，但不完全（人口压力偶尔跨越）
3. 同文化 = 同语言/传承 -> 后续可结成联盟
```

### 15.3 都城选定

从 T3+ 聚落选都城：
- 偏好：高人口、文化圈中心位置、防御性好。
- **都城密度高 -> 国家小**（满足"碎"）。
- 出生地区都城密度再加成 -> 保证多个小政权。

### 15.4 政权扩张（贴山靠河但不完全）

每都城从自身地块 flood-fill 扩张，cost 函数：

```
expansion_cost(tile) = w_dist * 到都城距离
                     + w_terrain * 地形难度（山/河/荒漠高）
                     + w_culture * 文化摩擦（跨文化 cost 高）
                     + w_pop * (1 / 聚落人口)  # 贫瘠地块扩张慢
```

- 停止条件：遇到他国已占地块 / 扩张预算耗尽（预算调小 -> 小国）。
- **贴山靠河但不完全**：靠 cost 权重实现，非硬规则。山/河 cost 高但**有限**（非无限屏障）-> 边界多停在此，偶尔穿过（人口/距离压力足够时跨越）。
- 难以跨越地形（高山脉/大荒漠）cost 极高 -> 常成天然国界，但其内部的聚落仍归某国（§0.14 不堆军事）。

### 15.5 联盟与单一政权

| 实体 | 构成 | 政治地图表现 |
|------|------|-------------|
| **联盟（Alliance）** | 同文化相邻多个小政权 | 一色外框 + 内部细边界（神罗式），外人视为一国 |
| **单一政权** | 独立一国 | 单色实心 |
| **大单一政权** | 偶有，但硬性 cap | 单色，规模受限 |

- 联盟是**元实体**：内部各王国独立运作（便于玩家初期蚕食扩张），外交上可视作整体。
- **规模上限**：硬性 cap 单政权地块数 -> 无半大陆巨无霸。
- 出大陆构成：多联盟 + 多单一政权混合，以联盟为主（碎）。

### 15.6 碎度保证

- 全大陆：都城密度高、扩张预算小 -> 默认碎。
- 出生地区：额外加成都城密度 -> 强制多个小政权。

### 15.7 地区-政权交互（后处理切分）

地区先按地理定（§5.5），政权生成后做一次后处理：

| 情形 | 处理 | 理由 |
|------|------|------|
| 一个地区被**两个大政权**（帝国）分割 | 沿政权边界**切分为两个地区** | 大政权应贴合地区边界，不被地区切碎 |
| 一个地区内含**几个小政权** | **不切分** | 本就是碎的预期状态；小政权同文化 -> 联盟候选 |

**结果**：
- 大政权（帝国）-> 贴合地区边界，横跨多个完整地区。
- 小政权 -> 在单个地区内细分；多个小政权共占一地区。
- 地区作为"游玩时相对完整的行政区划单位"保持稳定，仅在两大帝国交界处才分裂。

**互不矛盾**：大政权的扩张成本让它天然倾向填满整个地区再外扩；小政权预算小、挤在一地区内。后处理切分只处理少数被两大帝国同时分割的边界地区。

## 十六、聚落间道路生成

### 16.1 地块内道路

- 连接地块内所有聚落。
- 算法：**最小生成树**（MST，保证全连通）+ 少量额外贸易路（高重要度聚落对）。
- 路径偏好平地，避山（除非有隘口），沿河谷（共用 §五 高度场，天然协调）。

### 16.2 跨地块道路

- 都城/重镇间的主路。
- 经**山口**（山脉低点）穿越山系，不在山顶硬穿。
- 沿河、沿海的贸易路优先。

### 16.3 道路与政权

- 同政权内道路密度高；跨国道路稀疏（多在边境口岸/山口）。
- 后处理（§十七）按重要度决定哪些路升级为"官道"。

## 十七、后处理：聚落重要度与职能微调

### 17.1 重要度评分

政权生成后，每聚落算 `national_importance`：

| 级别 | 来源 | 影响 |
|------|------|------|
| 都城 | 政权首都 | 最高驻军/规模/道路枢纽 |
| 重镇 | T3+ 高人口 | 高驻军、贸易枢纽候选 |
| 边境要塞 | 接壤他国/扼守隘口 | 军事职能、要塞化 |
| 贸易枢纽 | 路口/河口/沿海 | 贸易职能、道路优先 |
| 内地城镇 | 政权腹地 | 中等，多农业/手工业 |
| 村落 | 一般 | 低，资源采集 |

### 17.2 职能分配（多职能可叠加）

职能是**标签集合**，条件叠加即可：

```
functions = []
if 近矿脉:        functions += [采矿]
if 河口/沿海/路口: functions += [贸易]
if 边境/隘口:     functions += [军事要塞]
if 魔力泉:        functions += [魔力]
if 大平原高耕地:   functions += [农业]
# 多职能：贸易城 + 军事重镇（条件同时满足即叠加）
```

### 17.3 军事力量分配

- **边境/前线**聚落 -> 军事职能、要塞化、高驻军。
- **被难以跨越地形保护的内部安全区** -> 边境压力低 -> 军事职能少（不堆兵，§0.14）。
- 判定：若某政权腹地被高山/大荒漠与其他政权隔离 -> 该腹地聚落 military_weight 衰减。

### 17.4 规模微调

- `national_importance` 微调聚落 `population_score`（都城/重镇上浮，村落基本不动）。
- 仍受级别桶上下限约束，不越级。

### 17.5 贫瘠地块特例

- 仅 1-2 个村落级聚落的地块：其中**最高者作地块事实中心**（"首富"），虽非城市级仍承担该地块首要职能（如该地块的贸易/行政节点）。

## 十八、每局轻微随机（仅规模扰动）

### 18.1 世界模板（开发期生成后定死，随包发布）

- 大陆轮廓、地形/群系/河流、文化圈+政权框架、聚落数量+位置+级别+控制区边界。

### 18.2 每局 run_seed 扰动（玩家运行时唯一变化）

| 扰动项 | 范围 | 免疫 |
|--------|------|------|
| 聚落规模 | 级别内 ±15% | 出生村落、所在地块中心城市 |

- 位置、控制区边界、聚落数量、级别、政权、地形：**全部不变**。
