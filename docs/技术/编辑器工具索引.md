## addons/ — 编辑器插件

### MapEditor（项目自带）

| 字段 | 值 |
|---|---|
| 路径 | [addons/map_editor/map_editor_plugin.gd](file:///f:/VSCode/game-2/stick-world/addons/map_editor/map_editor_plugin.gd) |
| 类型 | EditorPlugin（`@tool`） |
| 作者 | stick-world（项目自带，v0.1.0） |
| 启用 | Project Settings → Plugins → 启用 MapEditor |
| 用途 | 可视化地图编辑工具栏：「建筑放置」按钮 + 下拉选建筑 + X坐标输入 + 「放置!」按钮 + 「吸附全部」按钮 |

| 配套工具 | [tools/dev/map_grid_drawer.gd](file:///f:/VSCode/game-2/stick-world/tools/dev/map_grid_drawer.gd)（场景内绘制网格 + 建筑占地高亮） |
| 关键常量 | `BUILDINGS_DIR`、`CELL_SIZE = 32` |
| 触发条件 | 当前编辑的场景根节点含 `ground_y` 属性时显示工具栏（典型：VillageMap） |

### dev/ — 开发辅助

#### map_grid_drawer.gd

| 字段 | 值 |
|---|---|
| 路径 | [tools/dev/map_grid_drawer.gd](file:///f:/VSCode/game-2/stick-world/tools/dev/map_grid_drawer.gd) |
| 类型 | `@tool Node2D`（`class_name MapGridDrawer`，挂场景） |
| 用途 | 编辑器内绘制 32px 网格竖线 + 地面线 + 建筑占地高亮 |
| 配套 | 被 [addons/map_editor/map_editor_plugin.gd](file:///f:/VSCode/game-2/stick-world/addons/map_editor/map_editor_plugin.gd) 设置 `ghost_active` / `ghost_world_pos` / `ghost_width_cells` 属性来高亮建筑将占用的竖条 |
| 依赖 | 父节点须有 `ground_y` / `ground_bottom` / `map_right` 属性（典型：VillageMap） |

#### code_scanner.gd

| 字段 | 值 |
|---|---|
| 路径 | [tools/dev/code_scanner.gd](file:///f:/VSCode/game-2/stick-world/tools/dev/code_scanner.gd) |
| 类型 | `SceneTree`（CLI，非 `@tool`） |
| 启动 | `godot --headless --path <project_dir> -s res://tools/dev/code_scanner.gd` |
| 用途 | 扫描所有 `.gd` 文件语法和基本质量问题（CI 用） |
| 扫描目录 | `modules`、`core`、`tests`、`tools`（排除 `addons/`） |
| 退出码 | `0` = 无问题，`>0` = 有问题 |

## 维护规则

- **新增工具**：在 `addons/` 或 `tools/` 下新增任何脚本时，**必须同步更新本文档**（添加条目 + 关键词反查表）。
- **常量变更**：`BUILDINGS_DIR`、`CELL_SIZE`、扫描目录等关键常量变更后，同步更新本文档「操作资源」字段。
- **删除工具**：删除脚本时同步删除本文档对应条目。
- **路由入口**：本文档由 [rule.md](file:///f:/VSCode/game-2/.trae/rules/rule.md) 的「项目文档导航」表格引用，AI 在用户提到编辑器工具/插件关键词时应优先读本文档。
