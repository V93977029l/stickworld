## 项目结构速查

> 完整目录设计见 [`架构/场景与战斗架构.md`](架构/场景与战斗架构.md) §十三「目录结构提案」。

```
stick-world/                     # Godot 项目根（res://）
├── project.godot
├── core/                        # 核心系统（修改需批准）
│   ├── autoload/                # 全局单例（event_bus / scene_manager / save_manager / config_manager）
│   ├── ui_framework/            # UI 基础设施
│   ├── services/                # 抽象服务
│   └── utils/                   # 通用工具类
├── modules/                     # 游戏功能模块（开发最频繁的区域）
│   ├── units/                   # 火柴人 + 行为 AI
│   ├── combat/                  # 战斗系统
│   ├── construction/            # 定居点建设
│   ├── organization/            # 组织架构
│   ├── world_map/               # 战略图模块（鸟瞰多边形领土）
│   ├── world/                   # 场景图模块（卷轴地图，含 GameRoot/Map/CameraRig）
│   ├── buildings/               # 建筑运行时
│   ├── environment/             # 环境/天气/光照
│   ├── player_control/          # 玩家输入分发/选择/编队/附身
│   ├── ui/                      # UI 分层（GlobalHUD/ModePanel/Minimap 等）
│   ├── resources/               # 资源系统
│   └── debug/                   # 调试覆盖层（DebugOverlay）
├── scenes/                      # 顶层场景（game_root.tscn + chunk 模板）
├── config/                      # 数据驱动配置（maps / buildings / resources 等 .tres）
├── assets/                      # 全局共享资源（buildings/ 素材等）
├── addons/                      # 第三方插件
├── prototypes/                  # 原型实验区（不被正式逻辑依赖）
├── tools/                       # 自定义工具脚本（@tool / vector_db）
└── tests/                       # 自动化测试
```

> **术语提示**：「战略图」（`modules/world_map`，鸟瞰多边形领土，玩家不在其中）与「场景图」（`modules/world`，卷轴地图，玩家在其中）是两类不同的地图概念，详见 [`架构/世界地图数据流.md`](架构/世界地图数据流.md) §1。

## 常见问题 FAQ

| 问题 | 解决 |
|------|------|
| EventBus 找不到 | 检查 `project.godot` 的 Autoload 配置，确认 `event_bus.gd` 已注册 |
| GdUnit4 报错 | 当前项目未引入 GdUnit4（见架构文档 §十七 P0-9），使用自研 `TestRunner`；若需引入见 [`../CONTRIBUTING.md`](../CONTRIBUTING.md) |
| CI 失败 | 检查 GitHub Actions 中 Godot 路径，用 `godot --headless` 验证编译 |
| Godot headless 验证 | `"F:/SteamLibrary/steamapps/common/Godot Engine/godot.windows.opt.tools.64.exe" --headless --path "f:/VSCode/game-2/stick-world"` |
| PowerShell 不支持 `&&` | 用 `;` 分隔命令，或用 `if ($LASTEXITCODE -eq 0)` 做条件判断 |
| 场景图与战略图混淆 | 见上方「术语提示」，或查 [`架构/世界地图数据流.md`](架构/世界地图数据流.md) |
| 附身功能不可用 | 已知 bug P0-1，详见 [`架构/场景与战斗架构.md`](架构/场景与战斗架构.md) §十七 |
