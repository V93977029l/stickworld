# 技术架构

> **本文档仅保留技术栈选型和工作流，架构设计已迁移至 [`架构/场景与战斗架构.md`](架构/场景与战斗架构.md)。**
> 该文档是 P0 阶段的工程架构基线（v10），覆盖项目结构、场景与视图、数据存储、架构模式、无缝缩放、性能等全部架构主题。

---

## 一、技术栈选型

| 项 | 方案 | 依据 |
|----|------|------|
| 引擎 | Godot 4.7 | 创始人指定，P0 阶段当前版本 |
| 脚本语言 | GDScript | Godot 原生，开发效率高 |
| 渲染管线 | Forward+ | Godot 4.x 默认渲染器，支持 2D + 3D 混合 |
| 2D 渲染 | Node2D + Sprite2D + AnimatedSprite2D | 平面 2D 画风 |
| 骨骼动画 | Skeleton2D + IK 修改器 | 火柴人战斗动画（见 [`架构/场景与战斗架构.md`](架构/场景与战斗架构.md) §7.1） |
| 物理 | CharacterBody2D + Area2D | 火柴人移动与碰撞检测 |
| 测试 | GdUnit4 | Godot 生态标准（当前未引入，见架构文档 §十七 P0-9） |
| CI/CD | GitHub Actions | `godot --headless` |
| AI 开发 | Vibe Coding 模式 | 创始人偏好，详见 [`AI工作流指南.md`](AI工作流指南.md) |
