# 地图制作教程

> 本教程介绍如何为 stick-world 创建新地区地图。
> 关联架构：`docs/技术/架构/场景与战斗架构.md` §3 地图系统

---

## 一、地图结构总览

每张地图是一个 Godot 场景（`.tscn`），挂载 `VillageMap`（或后续的 `BattleMap`/`IndoorMap`）脚本。

```
VillageMap (Node2D, 挂 village_map.gd)
├── PlacementGrid (Node, 挂 placement_grid.gd)   ← 32px 占地网格（含 BuildMask）
├── TerrainLayer (Node2D)                          ← 地形渲染
│   └── GroundPolygon (Polygon2D)                  ← 地面多边形
├── GroundLine (Marker2D)                          ← 地面线标记（调试用）
├── DecorationLayer (Node2D)                       ← 装饰物（树/石头等）
├── BuildingHost (Node2D)                          ← 运行时所有建筑节点容器
├── TerrainBuildings (Node2D)                      ← 🆕 地形建筑（只读，不可拆除，见步骤 8）
├── InitialBuildingsList (Node)                    ← 🆕 初始建筑数据列表（见步骤 8）
├── WalkBarrier (Node2D)                           ← 🆕 地图级透明障碍（悬崖/高楼边缘，见步骤 9）
├── BuildMaskLayer (Node2D)                        ← 🆕 不可放建筑区域（大石头/山坡阶梯处，见步骤 10）
├── ForegroundLayer (Node2D)                       ← 🆕 前景层（z_index=10，室内支柱/天花板）
├── EntityHost (Node2D)                            ← 火柴人容器（运行时挂载）
├── ChunkTriggers (Node2D)                         ← Chunk 触发器（阶段 0.8）
└── BattleAnchor (Node2D)                          ← 战斗实例挂载点
```

**z_index 层级约定**：
- TerrainLayer = 0（地面最底）
- DecorationLayer = 1
- BuildingHost = 2
- EntityHost = 3（火柴人）
- ForegroundLayer = 10（前景支柱最高，火柴人经过被遮挡）

## 二、创建新地图步骤

### 步骤 1：复制模板

复制 `modules/world/scenes/test_village_map.tscn` 为新地图，例如 `forest_map.tscn`。

### 步骤 2：配置地图参数

选中根节点 `VillageMap`，在 Inspector 中设置：

| 参数 | 说明 | 示例值 | 计算公式 |
|------|------|--------|---------|
| `ground_y` | 地面线 Y（世界坐标） | 450 | 自定义 |
| `ground_ratio` | 地面占屏幕高度比例 | 0.4 | 默认 2/5 |
| `map_left` | 地图左边界 X | 0 | 固定 0 |
| `map_right` | 地图右边界 X | 12288 | 卷轴宽度，按需 |
| `ground_bottom` | 地面底部 Y | 882 | = ground_y + 1080 × ground_ratio |

**关键约束**：
- `ground_bottom` 必须匹配屏幕可见地面范围，否则火柴人显示偏下
- `map_right` 决定卷轴水平长度，当前 P0 最大 8192，后续支持更大

### 步骤 3：调整 GroundPolygon 顶点

选中 `TerrainLayer/GroundPolygon`，修改 `polygon` 属性为 4 个顶点：

```
(0, ground_y), (map_right, ground_y), (map_right, ground_bottom), (0, ground_bottom)
```

例如 ground_y=450, map_right=12288, ground_bottom=882：
```
PackedVector2Array(0, 450, 12288, 450, 12288, 882, 0, 882)
```

### 步骤 4：调整 GroundLine 位置

选中 `GroundLine`，设置 `position = Vector2(0, ground_y)`。

### 步骤 5：配置 PlacementGrid

选中 `PlacementGrid`，设置：
- `grid_width` = map_right / 32（每格 32px）
- `grid_height` = (ground_bottom - ground_y) / 32（向上取整）

例如 map_right=12288, ground_y=450, ground_bottom=882：
- grid_width = 12288 / 32 = 384
- grid_height = (882 - 450) / 32 = 13.5 -> 14

### 步骤 6：设计地形与装饰

在 `TerrainLayer` 下添加地形元素：
- **Polygon2D**：地面色块（自动应用草地噪波材质）
- **Sprite2D**：地形纹理
- **TileMap**：复杂地形（后续支持）

在 `DecorationLayer` 下添加装饰：
- **Sprite2D**：树、石头、草丛等
- 注意装饰不要占用 PlacementGrid 的建筑槽位

### 步骤 7：放置建筑（三层架构）

游戏内建筑分三层和平共存，设计地图时主要操作前两层（地形建筑 + 初始建筑），玩家建筑由运行时存档生成。

#### 7.1 三层对比

| 层 | 存储位置 | 可拆除 | 加载时机 | 示例 |
|---|---|---|---|---|
| **地形建筑** | `TerrainBuildings` 节点（随场景打包） | 否 | 随场景实例化 | 矿山、遗迹、大石头 |
| **初始建筑** | `InitialBuildingsList` 列表（def_id + cell 坐标） | 是 | 首次进入地图写入存档 | 关卡设计放的民居 |
| **玩家建筑** | 存档 `.json` | 是 | 存档加载时实例化 | 玩家新建的农场 |

三层共用同一套建筑场景文件（`modules/buildings/scenes/<building_id>.tscn`），通过 PlacementGrid 统一占位登记，加载时机不同但互不冲突。

#### 7.2 放置地形建筑（不可拆除）

地形建筑是关卡固定的"固态地形"，玩家不能拆除。操作步骤：

1. 选中 `TerrainBuildings` 节点
2. 右键 -> `Instantiate Child Scene` -> 选择建筑场景（如 `bld_mine.tscn`）
3. 设置 `position`：X = 32 的倍数，Y = `ground_y`
4. VillageMap._ready() 会自动将这些注册到 PlacementGrid，标记 `is_terrain=true`

#### 7.3 配置初始建筑（可拆除）

初始建筑是关卡设计时放置的可破坏建筑，首次进入地图时写入存档，之后变为动态建筑。操作方式：

在 `InitialBuildingsList` 节点下添加子节点（或用 ExportArray 配置），每个条目包含：
- `def_id`：建筑 ID（如 `bld_house`）
- `cell_x` / `cell_y`：格子坐标（不是世界坐标，cell_x = world_x / 32）

```
InitialBuildingsList (Node)
├── Entry_01 (def_id=bld_house, cell_x=10, cell_y=12)
├── Entry_02 (def_id=bld_house, cell_x=20, cell_y=12)
└── Entry_03 (def_id=bld_farm,  cell_x=30, cell_y=12)
```

**运行时加载逻辑**（VillageMap._ready 自动处理）：
- 首次进入该地图 -> 将 InitialBuildingsList 内容写入存档 JSON 的 buildings 数组（标记 `is_initial=true`）
- 非首次进入 -> 跳过（存档已有，玩家可能已拆除部分）
- 存档加载时统一从 JSON 实例化所有动态建筑（初始 + 玩家）到 BuildingHost

#### 7.4 关键约束

- 所有建筑底部 Y 必须等于 `ground_y`（cell_y = 0 对应 ground_y 行）
- 建筑 X 坐标必须是 32 的倍数，否则 PlacementGrid 占位校验失败
- 建筑间至少留 1 格（32px）间距，避免占地重叠
- 节点名建议用 `<BuildingId>_<序号>` 格式，方便运行时索引
- **贴图尺寸任意**，可以比占地大或小（详见 `docs/技术/教程/建筑系统教程.md` 贴图与占地解耦）

### 步骤 8：放置 WalkBarrier 透明障碍（悬崖/高楼边缘）

WalkBarrier 用于标记"火柴人不可穿越"的矩形区域，适用于悬崖边缘、高楼堡垒边缘、断崖等"角色不能御空而行"的边界。

**操作步骤**：

1. 选中 `WalkBarrier` 节点
2. 添加 `Area2D` 子节点，命名如 `CliffEdge_West`
3. 在 Area2D 下加 `CollisionShape2D`，shape 选 `RectangleShape2D`
4. 调整 RectangleShape2D 的 size 和 position，覆盖悬崖边缘区域
5. 重复以上步骤添加更多障碍

```
WalkBarrier (Node2D)
├── CliffEdge_West (Area2D)
│   └── CollisionShape2D (RectangleShape2D, size=64x200, pos=(-32, 100))
├── CliffEdge_East (Area2D)
│   └── CollisionShape2D (RectangleShape2D, size=64x200, pos=(8192+32, 100))
└── HighBuildingEdge (Area2D)
    └── CollisionShape2D (RectangleShape2D, size=2000x32, pos=(1000, ground_y-32))
```

**关键约束**：
- 障碍区域是**矩形**，不支持复杂形状（复杂边界用多个矩形拼合）
- 运行时不可见，调试模式显示为**蓝色半透明矩形**（详见 §五 调试覆盖层）
- 火柴人进入障碍区域时回退到上一帧位置，velocity 归零
- **不适用于山坡**（山坡走独立 SlopeMap 类型，详见架构 §3.1）

### 步骤 9：放置 BuildMask 不可放建筑区域

BuildMask 标记"地形上不可放建筑"的格子，适用于大石头、山坡阶梯处、装饰物占地等。

**操作步骤**：

1. 选中 `BuildMaskLayer` 节点
2. 添加 `ColorRect` 子节点（红色半透明，仅设计时可见，运行时不渲染）
3. 设置 ColorRect 的 position 和 size，对齐 32px 网格
4. VillageMap._ready() 会读取这些 ColorRect 的位置尺寸，注册到 PlacementGrid.blockage_mask

```
BuildMaskLayer (Node2D)
├── BigRock_Zone (ColorRect, color=rgba(255,0,0,0.3), pos=(640, 450), size=(96, 64))
├── StairZone (ColorRect, color=rgba(255,0,0,0.3), pos=(1280, 450), size=(64, 32))
└── DecorationZone (ColorRect, color=rgba(255,0,0,0.3), pos=(2000, 450), size=(32, 32))
```

**关键约束**：
- ColorRect 位置和尺寸必须是 32 的倍数（对齐网格）
- BuildMask 是纯数据，不污染建筑系统
- 玩家尝试在此放建筑 -> PlacementValidator 返回冲突，提示与"建筑占用"相同的错误信息
- 调试模式显示为**红色半透明格子**

**与"路障建筑"方案对比**：BuildMask 是纯数据，不会误删、不会出现在建筑列表、与 PlacementValidator 自然集成。路障建筑方案已否决。

### 步骤 10：放置 ForegroundLayer 前景层（室内支柱）

ForegroundLayer 用于室内战斗场景的前景遮挡物（支柱、天花板横梁、吊灯等），火柴人经过时被遮挡。

**操作步骤**：

1. 选中 `ForegroundLayer` 节点
2. 添加 `Sprite2D` 或 `Node2D` 子节点（支柱/天花板贴图）
3. 设置 z_index = 10（自动高于 EntityHost 的火柴人）

```
ForegroundLayer (Node2D, z_index=10)
├── Pillar_01 (Sprite2D, texture=pillar.png, pos=(500, 300))
├── Pillar_02 (Sprite2D, texture=pillar.png, pos=(1000, 300))
└── CeilingBeam (Sprite2D, texture=beam.png, pos=(750, 200))
```

**适用场景**：室内战斗、地下城、宫殿内部。普通村落地图通常不需要 ForegroundLayer（留空即可）。

### 步骤 11：注册到 SceneLoader

编辑 `modules/world/scripts/game_root.gd`，在 `_register_default_maps()` 中添加：

```gdscript
func _register_default_maps() -> void:
    if scene_loader == null or not scene_loader.has_method("register_map"):
        return
    scene_loader.register_map(TEST_VILLAGE_MAP_ID, _VILLAGE_MAP_SCENE, WorldAPI.MapType.VILLAGE)
    # 新增地图注册
    scene_loader.register_map("forest_01", preload("res://modules/world/scenes/forest_map.tscn"), WorldAPI.MapType.VILLAGE)
```

### 步骤 12：测试

运行游戏，通过 `scene_loader.load_map("forest_01")` 加载新地图验证。按 **F3** 开启调试覆盖层查看所有不可见标记（占地格/障碍/地面线等）。

## 三、参数详解

### ground_y（地面线 Y）

火柴人可走区域的顶部边界。火柴人脚部 Y 不会低于此值。

### ground_ratio（地面占比）

地面占屏幕高度的比例。0.4 = 屏幕下方 2/5 是地面。

**修改此值后必须同步修改 ground_bottom**：
```
ground_bottom = ground_y + DESIGN_HEIGHT × ground_ratio
             = ground_y + 1080 × ground_ratio
```

### map_right（地图右边界）

卷轴水平长度。建议：
- 测试地图：8192（足够测试左右移动）
- 正式地图：12288~32768（根据地区规模）

## 四、草地噪波材质（自动）

`VillageMap._ready()` 会自动调用 `_apply_grass_texture()`，为 `GroundPolygon` 生成并应用草地噪波材质：
- FastNoiseLite SIMPLEX 噪波
- 深绿->浅绿渐变
- 每 96px 平铺一次

**自定义草地**：修改 `village_map.gd` 的 `_generate_grass_texture()` 中的配色或频率。

## 五、调试覆盖层（F3）

地图设计时会用到很多运行时不可见的标记（占地格/障碍/地面线等），按 **F3** 切换调试覆盖层可全部可视化。

**启动时屏幕左上角常驻小字号图例**（不淡出）：

```
[F3] 调试
绿格=占地  红格=不可建  蓝区=地图障碍  紫区=建筑障碍
黄线=地面线  青线=地面底  白框=建筑  紫框=Chunk触发
```

**显示内容明细**：

| 元素 | 颜色 | 来源 | 说明 |
|------|------|------|------|
| PlacementGrid 占用格 | 绿色半透明 | `PlacementGrid._cells` | 已被建筑占用的格子 |
| BuildMask 不可放建筑格 | 红色半透明 | `PlacementGrid.blockage_mask` | 地形限制不可放建筑（步骤 9） |
| WalkBarrier 地图障碍 | 蓝色半透明 | `MapInstance.WalkBarrier` 下 Area2D | 悬崖/高楼边缘，火柴人不可穿越（步骤 8） |
| PassageBarrier 建筑障碍 | 紫色半透明 | 各建筑 `PassageBarrier` Area2D | 建筑本体不可通行区域（详见建筑系统教程） |
| ground_y 地面线 | 黄色线 | `MapInstance.ground_y` | 火柴人可走区域顶部 |
| ground_bottom 地面底线 | 青色线 | `MapInstance.ground_bottom` | 火柴人可走区域底部 |
| 建筑边界框 | 白色边框 | `BuildingHost` 子节点 | 每个建筑的 Footprint 矩形 |
| Chunk 触发器范围 | 紫色矩形边框 | `MapInstance.ChunkTriggers` | 流式加载触发区域 |
| 火柴人状态文字 | 白色文字 | `EntityHost` 下 StickmanEntity | 速度/动画/朝向/坐标 |
| FPS / 实体数 | 白色文字 | 引擎 | 屏幕左下角 |

详见架构文档 `docs/技术/架构/场景与战斗架构.md` §10.5。

## 六、当前限制（P0 阶段）

1. **无 Chunk 流式加载**：整张地图常驻内存，map_right 过大会影响性能
2. **无地图编辑器**：手动编辑 .tscn，效率低（见待办事项）
3. **无地图过渡**：地图间切换是硬切，无过渡动画
4. **单一地图类型**：仅 VillageMap，战场/室内地图待阶段 0.5+

## 七、后续计划

- **阶段 0.8**：Chunk 流式加载，支持大地图
