# 程序化材质生成（Procedural Material Generation）

> 本文档是本项目第一次尝试**纯程序化生成材质贴图**的设计与算法文档。
>
> **当前实现状态**：当前 [procedural_materials.gd](file:///f:/VSCode/game-2/stick-world/modules/building_gen/scripts/materials/procedural_materials.gd) 的 `make_thatch_layered()` 采用了**简化版油画笔触方案**（斜向茅草束 + 底边垂挂 + 外框描边），而非本文档描述的完整五阶段算法（叶片原型 + posterize + Kuwahara）。
> 本文档描述的是**目标视觉效果的完整算法设计**，当前实现正在逐步逼近该目标。
> 迭代进度和效果数据见 [iteration_log.md](file:///f:/VSCode/game-2/stick-world/modules/building_gen/reference/iteration_log.md)，迭代工作流见 [程序化材质开发工作流.md](file:///f:/VSCode/game-2/docs/技术/教程/程序化材质开发工作流.md)。

> 目标：用 GDScript 代码生成一张「油画感、层叠叶片、半透明参差边缘」的茅草房顶贴图，
> 用于 `modules/building_gen/scenes/smithy_preview.tscn` 中三块屋顶

>
> 本文档定位为后续程序化材质（墙体石块、木纹、金属锈等）复用的算法基线。

## 1. 设计动机

### 1.1 为什么需要程序化

- 本项目是「火柴人大战略 + 工厂自动化」缝合怪，包含大量程序化建筑。
- 建筑形变（如 smithy 的 `width_cells` 参数化拉伸）要求贴图能跟随几何无缝铺贴，
  外部 PNG 贴图在拉伸时会出现失真、接缝、循环暴露等人工痕迹。
- 程序化生成可在运行时按需产出、随种子变化、可调参数丰富、不增加仓库体积。

### 1.2 视觉目标

参照真实茅草棚光照规律：

| 视觉特征                | 对应算法实现                                       |
|-----------------------|------------------------------------------------|
| 一层压一层的叶片堆叠          | 多排叶片实例化，层间 y 偏移 60% 覆盖                 |
| 颜色由上至下从浅到深（仿光照由强到弱） | 上 1/3 循环 1 次颜色组、下 2/3 循环 1 次，5 色由浅至深 |
| 同一片叶子是同一种颜色         | 叶片实例携带单个 base color，内部仅做轻微明暗扰动          |
| 边缘参差不齐，叶片之间有缝隙    | 底排叶片长度按 1D value noise ±35% 扰动，缝隙 alpha=0  |
| 房顶下半部茅草更长、下垂更深      | 下 2/3 区段叶片高度更大、更稀疏                      |
| 油画感、颜料层层堆叠         | 5 级 posterize + 各向异性 Kuwahara 中值笔触           |
| 深色边缘描边              | 叶片先画放大 1px 的 #793c07 轮廓再画主色              |

### 1.3 调色板

颜色从浅到深（模拟光照由强到弱），其中 `ac5632` 与 `794b2c` 属同一层：

| 索引 | HEX       | 出现概率 | 用途                 |
|----|-----------|--------|--------------------|
| 0  | `#e5b275` | 高      | 最上层、最亮叶（迎光主色）  |
| 1  | `#bf7f41` | 高      | 次亮层主色           |
| 2  | `#aa6a3c` | 中      | 中间层主色            |
| 3  | `#ac5632` | ≤15%   | 偏红零星叶，与下层共存    |
| 4  | `#794b2c` | ≤8%    | 最深、最稀的偏红叶       |

边缘描边色：`#793c07`

「同一层两个颜色」的语义：
- 在某一排（层）以 `#aa6a3c` 为主体色
- 该排中**零星叶片**随机替换为 `#ac5632`（≤15%）或更稀的 `#794b2c`（≤8%）
- 允许这两色在**单片叶子内部**做微妙渐变（尖端略偏 BR、根部偏深），但叶片主体仍归一种色
- 这样做既满足"同层两个颜色"，又不会让红线感过强

## 2. 最佳实践调研结论

### 2.1 业界做法综述

| 来源                          | 关键思路                                       | 对本项目的借鉴 |
|----------------------------|-------------------------------------------|---------|
| Substance Designer Tile Sampler | 把 disk/square shape 拉长成 thin blade，散布并按 gradient blend 分层 | 叶片原型 + 分层 tiling 思路 |
| Modo Shingles 程序纹理        | 程序生成带 bevel 的瓦片，可控高度/宽度            | 「一层压一层」的层间覆盖模型 |
| PBR 透明叶片贴图（常见藤蔓/草材质）   | 多层 alpha 边缘 + 法线贴图模拟丝条               | 边缘 alpha 参差思路 |
| Seamless Texture 工作流       | offset-and-paint 周期哈希保证 wrap 无缝           | 横向 tileable 关键技术 |

### 2.2 本项目采纳的「最佳实践」组合

- **横向 Tileable（水平无缝）**：叶片水平布局用 `hash(x mod w)` 决定，
  跨边界叶片在 `x` 与 `x + w` 位置同时绘制（wrap-around 笔触），后处理读写像素同样按 wrap 坐标处理。
  → 这是 Substance / Photoshop offset 技巧在 GDScript 里的等价实现，**是水平拼接领域公认的必要手段**。
- **纵向分层 + 循环颜色**：贴图纵向不要求 tileable（贴图只在一片多边形内使用），
  但要服从「上 1/3 循环一次颜色组、下 2/3 循环一次」的光照分布。
- **各向异性笔触模糊**：油画感的 Kuwahara 中值滤波沿叶片纵轴方向（局部切线）进行，
  避免横跨叶缝把分层感抹平。
- **Posterize 量化**：把色彩量化到 5 级明度 × 色相格内，模拟油画颜料离散堆叠的笔触感。
- **三块屋顶共用 1 张横向 tileable 贴图**：通过 UV 自然重复——
  `RoofMain` 拉宽时 `uv.x` 重复 `width_cells` 倍，因 tileable 所以左右衔接无缝。
  `RoofLeftGroup1` 与 `RoofRightEnd` 各自的 UV 起点与 `RoofMain` 对齐到 global x 坐标，
  实现整体水平连续。Roof 右端贴图与中央在物理拼合处贴图像素自然衔接。

### 2.3 与「标准做法」的对比

| 维度         | 标准做法                        | 本文采纳的增强                         |
|------------|------------------------------|----------------------------|
| 横向拼缝     | 9-slice 或干脆忽略            周期哈希 + wrap 笔触（更强自然感 |
| 叶片形状     | 矩形 + 噪声                  | 顶端尖 taper + 底端分叉散丝（更接近真草   |
| 颜色处理     | 调色板线性取色               | 颜色稀有度 + 叶内微渐变（更自然           |
| 边缘处理     | 整图描边一圈                 | 叶片自身描边 + 整图外框描边（双层保护        |
| 后处理       | 简单高斯模糊                | 各向异性 Kuwahara + posterize（油画感更      |

## 3. 算法五阶段详解

### 3.1 阶段 1：叶片原型（Blade Primitive）

```
function draw_blade(img, cx, top_y, length, width, angle, color, edge_color):
    # 顶端 20% taper，底部 30% 分叉散丝，中部平直
    # 先以 edge_color 画放大 1px 的轮廓 → 再以主色画内部
    # 沿叶片纵轴画一条中轴高光线（亮 8%）
    # 同一片叶子只用一种主色
```

- 叶片几何坐标按本地空间 (0,0) 为叶尖，(0, length) 为叶根
- 上 1/3 区段叶片 `length ∈ [60, 75]`、`width ∈ [14, 18]`
- 下 2/3 区段叶片 `length ∈ [75, 100]`、`width ∈ [16, 22]`，且密度更稀
- 角度 `angle ∈ [-8°, +8°]`，避免完全竖直的死板感

### 3.2 阶段 2：分层铺设

**区段划分**

- `TOP_BAND = h / 3`（上 1/3）
- `BOT_BAND = h * 2 / 3`（下 2/3）

**颜色循环**

```
PALETTE = [#e5b275, #bf7f41, #aa6a3c, #ac5632 or #794b2c]
# 每个区段内按层数循环遍历调色板，每层对应一排水平叶片
```

**层间布局**

- 一排叶片的 `y` 起点为排顶，叶片纵向覆盖到 `y + length`
- 排与排之间 y 偏移 = 叶片长度 × `OVERLAP_RATIO = 0.6`（一层压一层）
- 同排叶片水平间距 `spacing ∈ [width * 0.7, width * 1.3]`，由 value noise 扰动
- **横向无缝关键**：叶片拓扑位置以 `x mod w` 输入哈希决定，
  跨 `x = 0 / x = w` 边界的叶片在两侧同步绘制（wrap-around）

### 3.3 阶段 3：边缘参差与半透明

- 底排（每个区段最下一排）每片长度按 1D value noise ±35% 扰动
- 叶片之间缝隙像素 Alpha=0，使 Polygon2D 能透出背景
- 左右边界叶片做 taper 收尾（宽度逐渐收为 0），防止硬切
- 贴图顶边与底边的处理：
  - 顶边：叶子尖端可超出贴图顶部（被裁掉），形成自然"看不见叶根"的视觉效果
  - 底边：叶子根部必须完整在贴图内，底排参差下垂直接构成屋顶下沿

### 3.4 阶段 4：描边

两层保护：

1. **叶片层描边**：阶段 1 绘制每片叶子时，先以 `#793c07` 画放大 1px 的轮廓，再覆主色填充，每片叶子周围自然得到深色 1px 边
2. **整图外框描边**：绘制完毕后再沿贴图 4 边各绘 2px 的 `#793c07` 线条，保证屋顶剪影边缘清晰、避免半透明边缘与背景混色时产生白圈

### 3.5 阶段 5：油画感后处理

依次：

1. **Posterize（颜色量化）**：RGB 通道分别映射到 5 级离散值，模拟颜料离散堆叠
2. **各向异性 Kuwahara 中值滤波**：3×3 窗口沿叶片纵轴切线方向读取，取中值，保留笔触边界
   - 简化版：每个像素读其上、下、中心 3 个像素 R/G/B，取中值
   - 沿叶片方向不会跨越叶缝抹平分层
3. **颜料颗粒噪声**：最后加一层强度 ≤2% 的高频噪声，模拟颜料颗粒肌理

## 4. 横向 Tileable 实现要点

### 4.1 核心：周期哈希

```
function blade_x_position(grid_col, seed):
    return wrap(hash(grid_col, seed), 0, w)
```

- 网格列号 `grid_col` 从 `0` 到 `grid_cols - 1`
- 每列的 x 偏移由 `grid_col` 哈希决定，与 `grid_cols` 无关，因而贴图任意宽度都 tileable

### 4.2 核心：wrap-around 笔触

- 当叶片宽度的一半（含 1px 边）超过该叶片 x 到边缘的距离时，在 `x + w` 处补绘同样像素
- 后处理阶段所有像素读写使用 `wrap(x, w)` / `wrap(y, h)` 坐标

### 4.3 三块屋顶拼接

- 三块共用 1 张贴图，UV.x 按「世界 x 坐标 ÷ 贴图像素宽 × scale」映射
- `RoofMain` 拉伸后 UV.x 重复 `width_cells` 次贴图，因 tileable 所以左右完全无缝
- `RoofLeftGroup1` 与 `RoofRightEnd` 的 UV.x 起点对齐到 `RoofMain` 的 global x 坐标
  → 三块在物理拼缝处像素天然衔接

### 4.4 下垂差异的处理

- `RoofRightEnd` 几何下沿比 `RoofMain` 低（由 `RR_BL_Y = -206` vs `RM_BL_Y = -232` 配置保证）
- 贴图自身在下 2/3 区段绘制更长、更稀、更长下垂的叶片（阶段 2 处理）
- 双重贡献：**几何提供更长下沿 + 贴图提供更长叶片**，两者 UV.y 都覆盖完整贴图
  → 下 2/3 的长叶自然显示更多，视觉上下垂更深

## 10. 后续扩展

本文档是本项目程序化材质的基线。本算法框架后续可复用到：

- 墙体石块（调色板换为石灰色系，叶片原型换为石块原型，移除半透明）
- 木纹（叶片原型换为木板，单排无覆盖）
- 金属锈蚀（叶片原型换为锈斑，加腐蚀噪声 mask）

复用思路：

- 把「调色板 / 原型绘制 / 分层策略 / 后处理」抽为独立的配置表 + 策略接口
- 当积累到第二个程序化材质时再考虑重构抽象，避免过早抽象
