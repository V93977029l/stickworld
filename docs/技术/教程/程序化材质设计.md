# 程序化材质设计

> 本文档是程序化材质模块的**设计基线**，专注新方案架构设计。
> 所有历史方案已废弃，不再对比。

---

## 1. 原始诉求（用户口述整理）

### 1.1 核心视觉目标

- **手绘写实感卡通**（Hand-Painted Stylized）：对标 `reference/smithy_lv1_full.png`
- 有明显笔触感和形状轮廓，色彩干净概括，拒绝高频噪点
- 大块明暗关系清晰，层次分明，像手绘一样有"拙朴感"

### 1.2 叶片实体性（不可妥协）

- **叶片是独立实体**：要么完整渲染，要么不渲染
- **绝不"拦腰截断"留半片**——用户反复纠正过无数次
- 叶片之间的空隙 = 透明（alpha=0），Polygon2D 透出背景形成仿真感
- 不是简单的 UV 平铺贴图，而是颗粒分明、可通过透明度蒙版直接显示出仿真感的材质

### 1.3 草坪生长算法范式

- 像植物生成/草坪生长算法：给定大致边界（多边形），在里面"种草"
- 边界是**软约束**，不是硬裁剪
- 下边缘尤其要随机有余量，做旧感，显得更像真实存在的建筑
- 贴图生成函数要接收多边形的大概位置（大致边缘线），以此为边界进行数据生成

### 1.4 倾斜与层叠

- 生成时倾斜 **60°**
- 层层叠叠，一层压一层
- 60°倾斜只是茅草本身倾斜，**茅草的根部坐标不倾斜**（网格不旋转）

### 1.5 两片屋顶

- 当前建筑分为**左侧一片屋顶**和**右侧一片屋顶**
- 左屋顶 -60°，右屋顶 +60°（改个 uniform 参数即可）
- 几何形状：平行四边形（60°扭曲的矩形），最多是多个脊线一致、铺设角度一致的平行四边形拼接体

### 1.6 建筑动态可变

- 建筑长度可随意拉长缩短
- 建筑生成时应该是动态的，材质要跟随变化

### 1.7 材质与几何耦合

- **故意耦合**，不是解耦——这样才能让材质更灵动
- 后续整栋建筑包括木头和石墙之类的也程序化生成
- 贴图生成函数要接收多边形的大概位置

### 1.8 开发策略

- **从单个叶片开始试点**，然后逐渐增多叶片来更细致的调试
- 调试工具：专门做一个游戏场景观察实时渲染画面，右边有一系列拉动条和输入框可微调参数
- AI 迭代工具：颜色比例对比大抵还是有效的（保留），AI 顺手用

## 2. 技术路线：CanvasShader 实时渲染

### 2.1 方案

```
Polygon2D（定义多边形形状，vertex 阶段裁剪外部像素）
    └── ShaderMaterial
            └── thatch.gdshader（fragment shader 实时生成草束 + alpha）
                    ↑ uniform 参数
                    │
            由 @tool 脚本 / 调试面板 / 建筑脚本设置
```

- **多边形形状**由 Polygon2D 顶点定义，Godot 渲染管线自动处理
- **草束分布 + alpha**由 fragment shader 在 UV 空间内计算
- **uniform 参数**：种子、包围盒尺寸、草束参数（角度/密度/长度/粗细/调色板）

### 2.2 为什么用 Shader

- **GPU 并行**，性能无忧，无需预烘焙
- **实时渲染** = 渲染过程本身，建筑变形时 uniform 更新即适应
- **参数化** uniform 天然支持，调试面板实时调
- 随机性用**确定性 hash**（基于像素坐标 + 种子），每帧输出相同，不闪烁

### 2.3 随机性

Shader 内用确定性 hash（基于像素坐标 + 种子），每帧输出相同结果，不闪烁。

## 3. 茅草生成模型

### 3.1 方向与几何

- **根部在脊线（上方）**，叶尖向下垂
- "谁家房顶上铺着茅草是从下往上铺的"——茅草从屋顶脊线向下铺设
- **倾斜60°**：左屋顶 -60°，右屋顶 +60°
- 网格不旋转，只是茅草本身倾斜

```
脊线 ────────────────────────────
 │\  │\  │\  │\  │\  │\  │\  │\    ← 根部在脊线，向下垂
 │ \ │ \ │ \ │ \ │ \ │ \ │ \ │ \
 │  \│  \│  \│  \│  \│  \│  \│  \
 │   │   │   │   │   │   │   │     ← 叶尖（下边缘，参差）
──────────────────────────────────
```

### 3.2 层数驱动（核心）

**不判断"叶片位置是否合法"**——层数已经限制了，每层叶片天然在范围内，不可能出现非法的叶子。

```
层数 = f(渲染范围高度)
  例如: layer_count = round(height / 30)   # 180px → 6层

每层一排叶片：
  - 根部在同一 y 线（整排根部一样）
  - x 方向分布在该层范围内
  - 叶片向下生长（叶尖朝下）
```

**最上层特殊**：根部在脊线，被顶部边界裁剪——这是**唯一允许截断的层**。整排根部一样，只有长度一个参数。上方因为根部被隐藏了。

**其他层**：叶片完整，不截断。

### 3.3 叶片参数化

每个叶片有内在参数（高度随机）：

| 参数 | 说明 |
|------|------|
| 长度 | 基础长度 + 正态随机（几像素小扰动） |
| 宽度 | 基础宽度 + 抖动 |
| 角度 | ±60° + 小抖动（左屋顶-60°，右屋顶+60°） |
| 颜色 | 每层有基调颜色，层内颜色波动较小（正态分布） |
| 弯曲度 | 抖动 |

**颜色规则**：每层一个基调色，层内颜色波动较小（正态分布），不是全局随机。

### 3.4 参考图配色

`thatch_ref.png` 分析结果（OpenCV KMeans）：

| 主色 HEX | 视觉位置 | 用途 |
|----------|----------|------|
| `#e6b578` | 亮区、迎光主色 | 最上层叶片基调 |
| `#d19759` | 次亮区 | 上层/受光面 |
| `#ba783c` | 中间层 | 主体叶片 |
| `#9b5b2d` | 偏暗区 | 中层阴影 |
| `#693a1f` | 深棕/阴影 | 最暗处、缝隙 |

统计信息：
- 平均 BGR: [78, 134, 192]
- 平均 HSV: [14°, 154, 192]
- 非透明像素：205,152

分析脚本输出 swatch：`modules/building_gen/reference/thatch_palette_swatch.png`

**长度参数**：有个大参数也有个小参数——一个大的随机波浪和一个小的针对每个叶片的小正态分布长度修正。所有层叶片都有长度变化，不只是下层。

### 3.4 层叠

- 上层叶尖覆盖下层根部
- 渲染顺序：**从下往上画**（下层先画，上层覆盖）
- 层间 y 偏移 = 叶片长度 × 覆盖比（一层压一层）

### 3.5 下边缘参差

最下层叶片的叶尖形成下边缘：

```
下边界 = 上限（不是硬线，是参考线）

实际最下层叶尖位置:
  base_y = bottom - 20                    # 整体上移20px（留余量，防硬截断）
  大波浪 = 躁动正弦函数                    # 加噪声的正弦，决定下边缘整体起伏
  每叶片修正 = 正态随机 × 15px             # 小正态分布，每叶片长度微调
  最终叶尖 y = base_y + 大波浪 + 每叶片修正

留5px防硬截断 → 多边形稍微比叶片范围大一点
```

**"幸运叶片"** = 正态分布选中较长/较短的叶片，自然形成参差。不是额外机制，是正态分布的自然结果。

**下边缘的大参数线**：加上更多躁动的正弦函数。正弦函数本身不行，要加更多躁动。

### 3.6 多边形信息传递

**最简信息**（理论最小集）：

| uniform | 说明 |
|---------|------|
| `height` | 渲染范围高度（脊线到屋檐） |
| `top_width` | 上边缘线长度（脊线长度） |

**为什么不需要更多**：
- 层数由 height 决定
- 每层叶片数由 top_width（及层宽度）决定
- 平行四边形是60°扭曲的矩形，与草束倾斜角度一致，天然对齐

### 3.7 几何形状限制

**允许的形状**：
- **平行四边形**（60°扭曲的矩形）—— 单个屋顶
- **平行四边形拼接体**—— 复杂屋顶，但需满足：
  - 脊线一致（同一条线）
  - 铺设角度一致（都是60°）

**当前建筑实例**：
- 左屋顶：草束 -60° 倾斜
- 右屋顶：草束 +60° 倾斜
- 两片各自独立生成，共用同一个 Shader，通过 uniform `blade_angle` 区分

## 4. 分层架构

```
┌─────────────────────────────────────────┐
│ 上层：场景集成                            │
│  - 建筑场景挂 Polygon2D + ShaderMaterial │
│  - @tool 脚本设置 uniform                │
│  - 调试面板控制参数                      │
└─────────────────────────────────────────┘
                    ↓ uniform
┌─────────────────────────────────────────┐
│ 中层：材质配方                            │
│  - thatch.gdshader        ← 当前实例     │
│  - wood.gdshader          ← 后续         │
│  - stone_wall.gdshader    ← 后续         │
│  每个配方 = 一个 .gdshader 文件          │
└─────────────────────────────────────────┘
                    ↓ Shader 原语
┌─────────────────────────────────────────┐
│ 底层：Shader 原语（include 或内联）       │
│  - hash(seed, x, y)     确定性随机       │
│  - normal_random(...)   正态分布         │
│  - blade_sdf(...)       叶片形状判定     │
│  - noise_sin(...)       躁动正弦         │
└─────────────────────────────────────────┘
```

**原则**：底层原语与几何无关，中层配方接收 uniform（含几何信息），上层集成负责设置 uniform。

## 5. 开发阶段与优先级

```
阶段1: 单叶片 Shader ✅
  - 固定位置（根部在 UV (0.5, 0)），固定参数
  - 验证：叶片形状、倾斜60°、alpha 边缘、颜色

阶段2: 一排叶片 ✅
  - 网格哈希分布，一排叶片
  - 验证：密度、抖动、排列
  - 行方向垂直于叶片方向

阶段3: 多层 + 层叠 ✅
  - 层数计算（height / layer_height）
  - 层间覆盖，从下往上画
  - 验证：层叠感、覆盖关系

阶段4: 颜色分层 ✅
  - 5 色调色板 + 每片随机选取 + 梢部变暗 + 极细噪点
  - 验证：颜色层次、手绘感

阶段5: 下边缘参差 + 平行四边形边界 ✅
  - 躁动正弦大波浪 + 每叶片小随机
  - 有效范围改为平行四边形，与屋顶 60° 倾斜对齐
  - 整片裁剪：根部或梢部超出范围则整片不渲染
  - 验证：做旧感、自然边缘、屋顶形状匹配

阶段6: 叶片形状优化 ✅
  - 梢部变细 ✅
  - 单片角度随机 ✅
  - 单片弯曲 ✅
  - 楔形笔刷、根部更粗、尖端更细 ✅
  - 油画笔触边缘粗糙度 ✅
  - 层间基调色区分 ✅

阶段7: 集成到建筑 ✅（演示版）
  - 茅草建筑演示场景 `thatch_building_demo.tscn`
  - 动态生成左、右两片屋顶，共用同一份 Shader
  - 验证：建筑长度可变、左右对称、材质与几何耦合
```

**优先级**：阶段1-3 是核心骨架（单叶片 → 一排 → 多层），必须跑通。阶段4-6 是美术打磨。阶段7 是最终集成。

## 7. AI 迭代工具

**保留**：颜色直方图比例对比（`compare.py` 的颜色部分）

**砍掉**：SSIM、MSE、边缘密度、垂直剖面——对风格化手绘纹理是伪指标

**AI 迭代流程**：
1. @tool 导出脚本把当前 Shader 渲染结果截图成 png
2. compare.py 对比生成图与参考图的颜色直方图比例
3. AI 根据颜色比例偏差 + 美术原理调整参数
4. 用户在调试场景中肉眼确认

**依赖**：Python + OpenCV（仅颜色直方图）+ Godot（截图导出）

## 附录：Shader 语法注意事项

- Godot shader 注释用 `//`（行注释）或 `/* */`（块注释），**不能用 `#`**
- `shader_type canvas_item;` 用于 2D
- uniform 颜色用 `: source_color` hint
- UV: `(0,0)` 左上角，`(1,1)` 右下角
