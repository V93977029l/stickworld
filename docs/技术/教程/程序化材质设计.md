# 程序化材质设计

> 本文档是程序化材质模块的**设计基线**，替代旧版《程序化材质生成》和《程序化材质开发工作流》。
> 历史方案（A-J）归档见 §附录。

***

## 1. 设计目标

用 **CanvasShader 实时渲染**生成程序化建筑材质（茅草屋顶为首个实例，后续扩展木头、石墙）。

核心诉求：

- **叶片是独立实体**：要么完整渲染，要么不渲染，绝不"拦腰截断"留半片
- **alpha 透明蒙版**：叶片实体 alpha=1，缝隙 alpha=0，Polygon2D 透出背景形成仿真感
- **材质与几何耦合**：Shader 接收多边形信息，在边界内生成适配的材质
- **建筑动态可变**：建筑长度可拉长缩短，Shader 实时适应，无需重新烘焙
- **手绘写实感卡通**：对标 `reference/smithy_lv1_full.png` 的 stylized hand-painted 风格

***

## 2. 技术路线：CanvasShader 实时渲染

### 2.1 为什么用 Shader 不用 GDScript

| 维度  | GDScript 逐像素合成（旧方案，已废弃）    | CanvasShader 实时渲染（新方案） |
| --- | -------------------------- | ---------------------- |
| 性能  | 解释执行，512×512 要数秒           | GPU 并行，实时              |
| 动态性 | 烘焙成贴图，建筑变形要重新生成            | uniform 更新即适应，无需烘焙     |
| 调试  | 改参数→起 Godot headless→生成→对比 | 改 uniform→实时看效果        |
| 分辨率 | 1024×1024 会卡死              | 无限制（GPU）               |

### 2.2 架构

```
Polygon2D（定义多边形形状，vertex 阶段裁剪外部像素）
    └── ShaderMaterial
            └──茅草.shader（fragment shader 实时生成草束 + alpha）
                    ↑ uniform 参数
                    │
            由 @tool 脚本 / 调试面板 / 建筑脚本设置
```

- **多边形形状**由 Polygon2D 顶点定义，Godot 渲染管线自动裁剪多边形外像素
- **草束分布 + alpha**由 fragment shader 在 UV 空间内计算
- **uniform 参数**：种子、包围盒尺寸、草束参数（角度/密度/长度/粗细/调色板）

### 2.3 随机性

Shader 内用**确定性 hash**（基于像素坐标 + 种子），每帧输出相同结果，不闪烁。

## 3. 茅草生成模型

### 3.1 方向与几何

- **根部在脊线（上方）**，叶尖向下垂
- **倾斜60°**：左屋顶 -60°，右屋顶 +60°（改个 uniform 参数即可）
- 茅草从屋顶脊线向下铺设（符合真实茅草屋顶的铺设方向）

```
脊线 ────────────────────────────
 │\  │\  │\  │\  │\  │\  │\  │\    ← 根部在脊线，向下垂
 │ \ │ \ │ \ │ \ │ \ │ \ │ \ │ \
 │  \│  \│  \│  \│  \│  \│  \│  \
 │   │   │   │   │   │   │   │     ← 叶尖（下边缘，参差）
──────────────────────────────────
```

### 3.2 层数驱动（核心）

**不判断"叶片位置是否合法"**——层数已经限制了，每层叶片天然在范围内。

```
层数 = f(渲染范围高度)
  例如: layer_count = round(height / 30)   # 180px → 6层

每层一排叶片：
  - 根部在同一 y 线（整排根部一样）
  - x 方向分布在该层范围内
  - 叶片向下生长（叶尖朝下）
```

**最上层特殊**：根部在脊线，被顶部边界裁剪——这是**唯一允许截断的层**。整排根部一样，只有长度一个参数。

**其他层**：叶片完整，不截断。

### 3.3 叶片参数化

每个叶片有内在参数（高度随机）：

| 参数  | 说明                  |
| --- | ------------------- |
| 长度  | 基础长度 + 正态随机（几像素小扰动） |
| 宽度  | 基础宽度 + 抖动           |
| 角度  | ±60° + 小抖动          |
| 颜色  | 每层有基调颜色，层内小波动（正态分布） |
| 弯曲度 | 抖动                  |

**颜色规则**：每层一个基调色，层内颜色波动较小（正态分布），不是全局随机。

### 3.4 层叠

- 上层叶尖覆盖下层根部
- 渲染顺序：**从下往上画**（下层先画，上层覆盖）
- 层间 y 偏移 = 叶片长度 × 覆盖比（一层压一层）

### 3.5 下边缘参差

最下层叶片的叶尖形成下边缘：

```
下边界 = 上限（不是硬线，是参考线）

实际最下层叶尖位置:
  base_y = bottom - 20                    # 整体上移20px（留余量，防硬截断）
  大波浪 = 躁动正弦函数                    # 加噪声的正弦，决定下边缘整体起伏
  每叶片修正 = 正态随机 × 15px             # 小正态分布，每叶片长度微调
  最终叶尖 y = base_y + 大波浪 + 每叶片修正

"幸运叶片" = 正态分布选中较长/较短的叶片，自然形成参差
  （不是额外机制，是正态分布的自然结果）
```

**所有层叶片都有长度变化**（大波浪 + 小正态），不只是下层。但下边缘的参差主要由最下层体现。

### 3.6 多边形信息传递

**最简信息**（理论最小集）：

| uniform     | 说明            |
| ----------- | ------------- |
| `height`    | 渲染范围高度（脊线到屋檐） |
| `top_width` | 上边缘线长度（脊线长度）  |

**为什么不需要更多**：

- 层数由 height 决定
- 每层叶片数由 top\_width（及层宽度）决定
- 平行四边形是60°扭曲的矩形，与草束倾斜角度一致，天然对齐

如果实际需要更多参数（如多层拼接体），可扩展 uniform。

## 4. 几何形状限制

### 4.1 允许的形状

- **平行四边形**（60°扭曲的矩形）—— 单个屋顶
- **平行四边形拼接体**—— 复杂屋顶，但需满足：
  - 脊线一致（同一条线）
  - 铺设角度一致（都是60°）

### 4.2 当前建筑实例

当前建筑有两片屋顶：

- **左屋顶**：草束 -60° 倾斜
- **右屋顶**：草束 +60° 倾斜

两片各自独立生成，共用同一个 Shader，通过 uniform `blade_angle` 区分。

## 5. 分层架构

```
┌─────────────────────────────────────────┐
│ 上层：场景集成                            │
│  - 建筑场景挂 Polygon2D + ShaderMaterial │
│  - @tool 脚本设置 uniform                │
│  - 调试面板控制参数                      │
└─────────────────────────────────────────┘
                    ↓ uniform
┌─────────────────────────────────────────┐
│ 中层：材质配方                            │
│  -茅草.shader        ← 当前实例          │
│  - wood.shader          ← 后续           │
│  - stone_wall.shader    ← 后续           │
│  每个配方 = 一个 .shader 文件            │
└─────────────────────────────────────────┘
                    ↓ Shader 原语
┌─────────────────────────────────────────┐
│ 底层：Shader 原语（include 或内联）       │
│  - hash(seed, x, y)     确定性随机       │
│  - normal_random(...)   正态分布         │
│  - blade_sdf(...)       叶片形状判定     │
│  - noise_sin(...)       躁动正弦         │
└─────────────────────────────────────────┘
```

**原则**：底层原语与几何无关，中层配方接收 uniform（含几何信息），上层集成负责设置 uniform。

## 6. 开发阶段

```
阶段1: 单叶片 Shader
  - 固定位置、固定参数，画一个倾斜60°的叶片
  - 验证：叶片形状、颜色、alpha 边缘
  - 产出:thatch.gdshader（单叶片版）

阶段2: 一排叶片
  - 网格哈希分布，一排叶片
  - 验证：密度、抖动、x 方向排列

阶段3: 多层 + 层叠
  - 层数计算（height / layer_height）
  - 层间覆盖，从下往上画
  - 验证：层叠感、覆盖关系

阶段4: 颜色分层
  - 每层基调色，层内正态波动
  - 验证：颜色层次、手绘感

阶段5: 下边缘参差
  - 躁动正弦大波浪 + 每叶片小正态
  - 验证：做旧感、自然边缘

阶段6: 调试场景
  - 游戏场景 + 右侧参数面板（Slider/SpinBox）
  - 实时调整 uniform，观察渲染效果
  - 产出:thatch_debug.tscn

阶段7: 集成到建筑
  - 挂到 pg_smithy_lv1.tscn 的屋顶 Polygon2D
  - @tool 脚本设置 uniform
  - 验证：建筑场景中的实际效果
```

## 7. 调试工具

**用户调试**：专门的调试游戏场景

```
thatch_debug.tscn
├── Polygon2D（屋顶形状）+ ShaderMaterial
├── Camera2D
└── Control（右侧参数面板）
    ├── Slider: blade_angle (0-90)
    ├── Slider: blade_density
    ├── Slider: blade_length
    ├── Slider: blade_width
    ├── SpinBox: seed
    ├── ColorPicker: palette[0..4]
    └── ... 实时修改 uniform
```

参数面板的 `value_changed` 信号 → 设置 `ShaderMaterial.set_shader_parameter()`，实时看到效果。

## 8. AI 迭代工具

**保留**：颜色直方图比例对比（compare.py 的颜色部分）

**砍掉**：SSIM、MSE、边缘密度、垂直剖面——这些对风格化手绘纹理是伪指标

**AI 迭代流程**：

1. @tool 导出脚本把当前 Shader 渲染结果截图成 png
2. compare.py 对比生成图与参考图的颜色直方图比例
3. AI 根据颜色比例偏差 + 美术原理调整参数
4. 用户在调试场景中肉眼确认

**依赖**：Python + OpenCV（仅颜色直方图）+ Godot（截图导出）

## 9. 后续扩展

本框架后续可复用到其他材质：

| 材质   | shader 文件            | 草束原型替换为       |
| ---- | -------------------- | ------------- |
| 茅草屋顶 | thatch.gdshader      | 倾斜草束（当前）      |
| 木板墙  | wood.gdshader        | 木板（水平铺设）      |
| 石墙   | stone\_wall.gdshader | 石块（矩形拼接）      |
| 金属锈  | metal\_rust.gdshader | 锈斑（腐蚀噪声 mask） |

每个材质一个独立 .shader 文件，共享底层 Shader 原语。
