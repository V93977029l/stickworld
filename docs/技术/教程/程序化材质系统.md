# 程序化材质系统

> 本文档是 `modules/building_gen` 的**系统级设计基线**。
> 目标：支持多种建筑材质（茅草、木头、石墙等）并行迭代开发，共享底层 Shader 原语与工具链。
> 首个实现的材质是**茅草屋顶**（`materials/thatch/`），作为整个系统的参考实例。

---

## 1. 设计目标

### 1.1 核心视觉目标

- **手绘写实感卡通**（Hand-Painted Stylized）：对标 `materials/thatch/reference/smithy_lv1_full.png`
- 有明显笔触感和形状轮廓，色彩干净概括，拒绝高频噪点
- 大块明暗关系清晰，层次分明，像手绘一样有"拙朴感"

### 1.2 叶片/笔触实体性（不可妥协）

- **笔触是独立实体**：要么完整渲染，要么不渲染
- **绝不"拦腰截断"留半片**
- 笔触之间的空隙 = 透明（alpha=0），透出背景形成仿真感
- 不是简单的 UV 平铺贴图，而是颗粒分明、可通过透明度蒙版直接显示出仿真感的材质

### 1.3 材质与几何耦合

- **故意耦合**，不是解耦——这样才能让材质更灵动
- 后续整栋建筑包括木头和石墙之类的也程序化生成
- 贴图生成函数接收多边形的大概位置（大致边缘线），以此为边界进行数据生成

### 1.4 并行迭代策略

- 每种材质拥有独立的目录、调试场景、参考图、测试
- 共享底层 Shader 原语（hash、noise、笔触 SDF 等）
- 共享截图/对比工具链
- AI 可针对单个材质 24 小时自主迭代，不影响其他材质

## 3. 技术路线：CanvasShader 实时渲染

### 3.1 方案

```
Sprite2D / Polygon2D（定义形状，传递 UV）
    └── ShaderMaterial
            └── <name>.gdshader（fragment shader 实时生成笔触 + alpha）
                    ↑ uniform 参数
                    │
            由 @tool 脚本 / 调试面板 / 建筑脚本设置
```

- **形状**由 Sprite2D/Polygon2D 定义，Godot 渲染管线自动处理
- **笔触分布 + alpha**由 fragment shader 在 UV 空间内计算
- **uniform 参数**：种子、包围盒尺寸、笔触参数（角度/密度/长度/粗细/调色板）

### 3.2 为什么用 Shader

- **GPU 并行**，性能无忧，无需预烘焙
- **实时渲染** = 渲染过程本身，建筑变形时 uniform 更新即适应
- **参数化** uniform 天然支持，调试面板实时调
- 随机性用**确定性 hash**（基于像素坐标 + 种子），每帧输出相同，不闪烁

### 3.3 随机性

Shader 内用确定性 hash（基于像素坐标 + 种子），每帧输出相同结果，不闪烁。
共享原语见 [modules/building_gen/shaders/lib/hash.gdshaderinc](file:///f:/VSCode/game-2/stick-world/modules/building_gen/shaders/lib/hash.gdshaderinc)。

## 4. 分层架构

```
┌─────────────────────────────────────────┐
│ 上层：场景集成                            │
│  - 建筑场景挂 Sprite2D/Polygon2D + ShaderMaterial
│  - @tool 脚本 / 建筑生成器设置 uniform     │
│  - 调试面板控制参数                      │
└─────────────────────────────────────────┘
                    ↓ uniform
┌─────────────────────────────────────────┐
│ 中层：材质配方                            │
│  - materials/thatch/shaders/thatch.gdshader  ← 当前实例
│  - materials/wood/shaders/wood.gdshader      ← 后续
│  - materials/stone_wall/shaders/...          ← 后续
│  每个配方 = 一个 .gdshader 文件            │
└─────────────────────────────────────────┘
                    ↓ #include
┌─────────────────────────────────────────┐
│ 底层：Shader 原语                         │
│  - shaders/lib/hash.gdshaderinc          │
│    - hash21 / hash31 确定性随机           │
│  - （后续扩展）brush.gdshaderinc         │
│    - 笔触 SDF、油画边缘粗糙               │
└─────────────────────────────────────────┘
```

**原则**：底层原语与几何无关，中层配方接收 uniform（含几何信息），上层集成负责设置 uniform。

## 5. 调试与截图工作流

### 5.1 调试场景

每个材质都有独立的 `_debug.tscn`：

```
<name>_debug.tscn
├── Sprite2D + ShaderMaterial
├── Camera2D
├── CaptureHelper（Node，挂载 capture_in_game.gd）
└── DebugUI（CanvasLayer，由 <name>_debug.gd 动态创建）
    └── Panel（右侧参数面板）
```

### 5.2 自动化截图

通用截图脚本：

```powershell
# 默认捕获 thatch 调试场景
.\modules\building_gen\tools\capture_standard.ps1

# 按材质名捕获
.\modules\building_gen\tools\capture_standard.ps1 -Material thatch

# 显式指定场景与输出路径
.\modules\building_gen\tools\capture_standard.ps1 `
  -ScenePath "res://modules/building_gen/materials/thatch/scenes/thatch_debug.tscn" `
  -OutputFrame "modules/building_gen/materials/thatch/reference/thatch_debug_capture.png"
```

关键注意点：
1. **不能用 `--headless`**。无窗口模式下 GPU 不会真正渲染 CanvasItem Shader。
2. `--position 10000,10000` 把窗口移到屏幕外，显示驱动仍在正常工作。
3. `capture_in_game.gd` 等待若干帧确保 Shader 编译完成后再截图。

## 6. AI 迭代工具

**保留**：颜色直方图比例对比（`tools/compare/compare.py` 的颜色部分，待实现）

**砍掉**：SSIM、MSE、边缘密度、垂直剖面——对风格化手绘纹理是伪指标

**AI 迭代流程**：
1. 运行 `capture_standard.ps1 -Material <name>` 导出当前 Shader 渲染结果
2. `compare.py` 对比生成图与 `materials/<name>/reference/<name>_ref.png` 的颜色直方图
3. AI 根据颜色比例偏差 + 美术原理调整参数
4. 用户在调试场景中肉眼确认

**依赖**：Python + OpenCV（仅颜色直方图）+ Godot（截图导出）

## 附录：Shader 语法注意事项

- Godot shader 注释用 `//`（行注释）或 `/* */`（块注释），**不能用 `#`**（`#include` 除外）
- `shader_type canvas_item;` 用于 2D
- uniform 颜色用 `: source_color` hint
- UV: `(0,0)` 左上角，`(1,1)` 右下角
- 共享原语通过 `#include "res://.../xxx.gdshaderinc"` 引入
