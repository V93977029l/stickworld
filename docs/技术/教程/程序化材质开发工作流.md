# 程序化材质开发工作流（通用引擎）

> 改代码 → 注册材质 → 跑脚本 → 迭代对比。

每种材质在引擎中注册一条配置即可复用全部工具链。

## 1. 一次迭代

```powershell
cd f:\VSCode\game-2\stick-world
python modules/building_gen/tools/py/iterate.py --material thatch --stats
```

归档旧版 → 生成 `reference/preview_<材质>.png` → 多维度对比。

| 参数 | 说明 |
|------|------|
| `--material X` | 切换材质（默认 thatch），见 §2 注册表 |
| `--stats` | 生成后运行多维度对比（MSE + 垂直剖面 + 直方图 + 边缘密度） |
| `--open` | 生成后打开文件 |
| `--compare-only` | 跳过生成和归档，仅对已有文件运行对比 |
| `--pair A.png B.png` | A/B 对比两份生成图（单变量改动归因） |
| `--list` | 列出所有已注册材质 |

## 2. 材质注册表

在 [iterate.py](file:///f:/VSCode/game-2/stick-world/modules/building_gen/tools/py/iterate.py) 的 `MATERIALS` 字典中维护：

| 材质 key | 说明 | texture_type |
|----------|------|-------------|
| `thatch` | 茅草屋顶 | `structured`（水平 tileable / 垂直结构化） |
| `stone_wall` | 石墙（预留） | `seamless`（全向 tileable） |
| `wood_plank` | 木板墙（预留） | `seamless` |
| `crenellation` | 城垛屋顶（预留） | `structured` |

**`texture_type` 含义**：
- `structured`：类似植被/植物生成器，水平无限重复，垂直有顶底边界。**SSIM 不适合，优先看垂直剖面**。
- `seamless`：传统无缝平铺纹理（砖墙、地砖），SSIM 有效。

**扩展新材质只需两步**：
1. 在 `MATERIALS` 字典中追加一条配置（材质名、输出路径、参考图路径、分区数、类型）
2. 在 [dmp.gd](file:///f:/VSCode/game-2/stick-world/modules/building_gen/tools/dmp.gd) 的 `match` 中追加一个分支（调用 `procedural_materials.gd` 的对应函数）

## 5. 对比指标说明

`--stats` 输出由 [compare.py](file:///f:/VSCode/game-2/stick-world/modules/building_gen/tools/py/compare.py) 提供。

| 指标 | 适用 | 说明 | 理想方向 |
|------|------|------|----------|
| **MSE** | 通用 | 像素均方误差 | 越小越好 |
| **垂直剖面** | `structured` 专用 | 逐行亮度对比（剖面 MSE + 相关系数），衡量分层节奏 | 剖面 MSE↓、相关→1 |
| **边缘密度** | 通用 | Canny 边缘占比，衡量纹理细节清晰度 | 越接近参考图越好 |
| **直方图相关** | 通用 | BGR 颜色分布相关性 | 越接近 1 越好 |
| **区域统计** | 通用 | 水平分区逐区域 MSE/SSIM/颜色均值 | 暴露局部差异 |
| **SSIM** | `seamless` 专用 | 结构相似度，`structured` 不适用（标注"参考"） | 越接近 1 越好 |
| **A/B 综合评分** | 通用 | MSE/垂直剖面/边缘/直方图投票 | 单变量归因 |

## 7. 迭代决策循环

一次完整的迭代闭环：

```
读取 iteration_log.md（了解历史迭代和当前瓶颈）
        ↓
打开 preview_xxx.png 和 xxx_ref.png 并排对比（视觉判断）
        ↓
阅读 --stats 输出（量化指标）
        ↓
决定：改哪个参数、往哪个方向改（每次只改一个变量）
        ↓
修改 procedural_materials.gd 中对应函数
        ↓
python iterate.py --material xxx --stats  （生成 + 对比）
        ↓
对比新旧指标，判断是否改善
        ↓
在 iteration_log.md 末尾追加本轮记录（改了啥、效果怎样、反思）
        ↓
回到第一步继续
```

**核心文件索引**：

| 文件 | 什么时候读 |
|------|-----------|
| [iteration_log.md](file:///f:/VSCode/game-2/stick-world/modules/building_gen/reference/iteration_log.md) | 每轮开始前——了解历史、避免重复踩坑 |
| `preview_xxx.png` + `xxx_ref.png` | 每轮——视觉对比 |
| [procedural_materials.gd](file:///f:/VSCode/game-2/stick-world/modules/building_gen/scripts/materials/procedural_materials.gd) | 改代码前——读当前参数 |
| `--stats` 分区统计 | 每轮——定位哪个区域最差 |

**决策原则**：

1. **每次只改一个变量** — 否则无法归因
2. **稳定代码结构 > 激进重构** — 迭代 #5 双循环无声失败
3. **先大结构后细节** — 层高/角度/密度对了再调纤维纹理
4. **`structured` 看垂直剖面，`seamless` 看 SSIM** — 不同纹理用不同指标

## 10. 风格化渲染迭代诉求（茅草材质）

### 10.1 核心目标

让程序化生成的建筑材质呈现**手绘写实感卡通（Hand-Painted Stylized）**的风格化渲染效果，对标 `reference/smithy_lv1_full.png` 的美术品质。这不是照片写实，也不是纯像素风，而是：
- 有明显笔触感和形状轮廓
- 色彩干净、概括，拒绝高频噪点
- 大块明暗关系清晰，层次分明
- 像手绘一样有"拙朴感"和"呼吸感"

### 10.2 用户完整诉求（迭代过程中反复强调）

1. **核心是形状和轮廓，不是颜色**——茅草要"一绺绺像铺在房梁上的"，颜色只是增加层次感，形状和位置才是关键。
2. **场景对标**——让 `smithy_preview.tscn` 渲染出的画面和 `smithy_lv1_full.png` 尽可能相似。不是看单张贴图，而是看**整张渲染场景**是否像手绘卡通建筑。
3. **风格定义**——手绘写实感卡通的游戏风格化渲染，对标《原神》《魔兽世界》《风之旅人》这类 stylized PBR / hand-painted 流派。
4. **持续迭代**——通过不断迭代让程序化生成的建筑**越来越像**参考图，不是一次到位。
5. **场景贴图说明**——`smithy_preview.tscn` 场景文件本身不存储屋顶贴图，贴图由 `smithy_preview.gd` 的 `_build()` 在运行时调用 `PM.make_thatch_layered()` 生成并通过 `_poly()` 设置到 `Polygon2D.texture`。所以"修改材质"= 修改 `procedural_materials.gd` 中 `make_thatch_layered()` 的算法，重渲场景即可看到效果。
6. **废弃方案要归档**——每个迭代方案（A-J）在确定废弃后必须移到 `tools/py/archive/`（脚本）和 `reference/history/`（preview png），保持工作区清爽。

### 参考图分析方法

用 `tools/py/analyze_ref.py` 深度分析参考图的视觉特征：

| 分析维度 | 方法 | 用途 |
|----------|------|------|
| 主色调 | K-Means 聚类 | 提取真实色板 |
| 边缘方向 | 霍夫变换统计 | 判断纹理主方向（水平/垂直/斜向） |
| 垂直亮度剖面 | 逐行均值 | 理解分层结构和明暗节奏 |
| 边缘密度 | Canny | 量化纹理细节量 |
| 连通域 | connectedComponents | 理解"草束"形状和尺寸 |

### 关键发现（thatch_ref.png 分析）

1. **纹理方向以水平为主**：霍夫变换检测到 1024 条水平线 vs 13 条垂直线——茅草是水平分层铺设的，不是垂直纤维
2. **色板（BGR）**：亮区 [122,182,231] → 主色 [93,156,212] → 中间 [66,127,192] → 暗区 [48,99,165] → 阴影 [35,65,114]
3. **垂直亮度剖面**：顶部亮（Y10-30%最亮）→ 底部逐渐变暗 → 最底部接近全黑（草尖垂挂外是背景）
4. **参差底部**：茅草底部不是整齐的直线，而是参差不齐的草尖垂挂

### 风格化材质最佳实践（来自行业调研）

来源：[Pixel Art Roof Texture Guide](https://frontierfoam.com/pixel-art-roof-texture-retro-game-graphics/)、[Hand-Painted Textures Production Guide](https://www.artstation.com/blogs/retrostyle-games1/837Ea/hand-painted-textures-for-games-production-economics)、[What are Stylized Textures?](https://stylizedtextures.com/blog/what-are-stylized-textures)、[Stylized Pastry Bakery 流程](https://80.lv/articles/designing-a-stylized-pastry-bakery-using-substance-3d-unreal-engine-zbrush)。

1. **大块、干净、概括**——风格化的核心是"选择性"地使用信息，拒绝高频噪点（"脏"）
2. **AO → 色彩渐变**——环境光遮蔽不是阴影，而是色彩渐变的驱动器
3. **Curvature → 概括高光**——只在最凸起的边缘画一条锐利、干净的高光线
4. **层叠结构**——茅草用层叠的草束簇表现，每簇有明确的形状轮廓
5. **干刷技法**——底色铺好后，向上干刷亮色高光（阳光漂白效果）
6. **2-3 种暖棕黄 + 冷色阴影**——色板要精简（4-8 色以内）
7. **笔触感（Painterly）**——可见的笔触是 hand-painted 风格的灵魂，不要过度混合（over-blending）
8. **形状语言（Shape Language）**——stylized = systemic simplification，避免过长的直线，注入圆润形状打破方盒子感
9. **Early In-Game Testing**——贴图在工具里好看不算数，必须 early & frequently 在实际场景里测试（Substance Designer 文档同样强调）
10. **支持 silhouette**——贴图应当增强几何 silhouette 而不是和它竞争

### 10.3 迭代原则（修正版）

1. **形状 > 颜色**——核心是形状和轮廓像"一绺绺铺在房梁上的茅草"，颜色只是增加层次感
2. **大块 > 细节**——先保证大结构对了，再调细节
3. **斜向草束 > 水平条纹**——参考图茅草是"一绺绺斜向草束"，不是水平条纹。霍夫变换检测到的"水平线"其实是草束底部排列的水平线，不是纹理本身的方向
4. **场景验证**——最终用 `render_preview.gd` 渲染 `smithy_preview.tscn`，与 `smithy_lv1_full.png` 并排对比；单张贴图好看不算数

### 10.5 方案 J v9 核心算法（油画手绘感）

**核心突破（v6→v9）**：把草束从"矩形笔触"改为"楔形笔触"（根部粗、尖端细），模拟油画笔刷的压感。

- **5 层从下往上绘制**，每层有：薄底色 → 一排斜向楔形草束 → 参差底部
- **草束形状**：楔形（root 粗 10-15px → tip 细 1-3px），像油画笔刷
- **草束方向**：-65° 到 -35°（明确斜向，避免"横平竖直"）
- **草束长度**：层高 × 1.0~1.4（跨入上层，形成层叠遮盖）
- **草束根部**：从层底到层中分布（不是只在层底）
- **草束颜色**：root 亮一档（受光）→ tip 暗一档（阴影），渐变到 80%
- **色彩抖动**：每像素有 jitter + warm_jitter，模拟油画色彩混合
- **薄底色**：只覆盖本层 + 下层 8%（让下层草束大量露出，避免"水平色块"感）
- **水平 tileable**：草束 x 坐标用模运算 wrap，让贴图左右无缝
- **参差底部**：草尖垂挂模拟自然边缘

### 10.6 渲染管线关键点

`smithy_preview.gd` 的 `_poly()` 函数 + `render_preview.gd` 的 `_replace_thatch_texture()` 共同负责把贴图贴到屋顶 Polygon2D 上：

1. **场景贴图来源**：`smithy_preview.gd._build()` 在编辑器模式下调用 `PM.make_thatch_layered()` 生成贴图并设到 Polygon2D，但 `.tscn` 文件不保存运行时生成的 `ImageTexture`，所以加载场景时 texture 为 null
2. **运行时替换**：`render_preview.gd._replace_thatch_texture()` 在加载场景后重新生成贴图并替换到所有名称含 "Roof" 的 Polygon2D
3. **UV 重复**：`repeat_x = max(1.0, poly_w / 256.0)`，让贴图按 256px 一次重复
4. **texture_filter**：用 LINEAR（油画手绘感需要平滑过渡）
5. **texture_repeat**：启用 TEXTURE_REPEAT_ENABLED，让 repeat_x > 1 时贴图真正重复

### 10.7 关键陷阱与解决

| 陷阱 | 现象 | 解决 |
|------|------|------|
| `--headless` 无法截图 | SubViewport.get_texture().get_image() 返回 null | 改用 `--display-driver windows --audio-driver Dummy` |
| 贴图被压缩稀释 | 贴图 std=28，渲染后 ROI std=24，edge_density 从 12% 降到 2.7% | 草束画得更粗（10-15px）+ 楔形让边缘清晰 |
| "横平竖直"色块感 | 草束太粗变色块 + 水平底色带太厚 | 改楔形笔触 + 薄底色（只覆盖下层 8%） |
| GDScript int 按值传递 | `_replace_recursive` 中 count 不累加 | 用 `Array` 作可变引用（`counter := [0]`） |
| 颜色过暗 | 渲染图 mean=88，参考图 mean=119 | 色板整体调亮一档（root/bg 都提升） |

### 10.8 后续迭代方向

当前 v9 已实现油画手绘感的斜向楔形草束，但渲染图屋顶草束细节"能隐约看到"但"不如贴图本身清晰"。后续可探索：

1. **增大贴图分辨率**：从 512x512 提升到 1024x1024，让单根草束占更多像素
2. **降低 UV 重复**：让贴图按 1:1 显示（不压缩），草束大小不变
3. **多贴图变体**：为不同屋顶 polygon 生成专属贴图（RoofMain/RoofRightEnd/RoofLeftGroup1 各一张）
4. **加入 normal map**：用伪法线让草束有立体感（油画厚涂感）
5. **手绘笔触叠加**：在楔形草束上再叠一层细笔触（油画的"扫笔"技法）

### 10.9 几何修正：非拼接范围贴图 + 透明草穗边缘（2026-07-21）

**用户反馈触发**：

- 屋顶边缘像"裁剪的平行四边形"，不像手绘茅草
- 右侧两个屋顶多边形（RoofMain + RoofRightEnd）其实是同一片屋顶，应渲染成连续的加长感
- 后景稻草墙没有处理，只是前景材质颜色加深即可

**实现路线选择**：先快速修几何，再评估 Shader。

**核心改动**：

1. **屋顶多边形合并**：将 RoofMain 与 RoofRightEnd 合并为单一连续多边形 `RoofMainRight`，消除接缝。
2. **范围专属贴图**：为每个屋顶多边形按其实际包围盒尺寸生成专属茅草贴图，不再使用全局 tileable 贴图无限拼接。
   - 贴图宽度 ≈ 多边形底边宽度（可略大以允许边缘流苏）
   - 贴图高度 ≈ 多边形高度（屋顶斜高）
   - 顶部对齐屋顶脊线，底部对齐屋檐，左右边缘对齐屋顶侧边界
3. **透明草穗边缘**：贴图边缘不再是硬边，而是用 alpha 通道模拟参差不齐的草穗垂挂。
   - 底部生成随机长度垂挂流苏，alpha 渐变淡出
   - 左右侧边生成短草穗，alpha 不规则衰减
   - 透明区域 RGB 预乘 alpha，避免边缘黑边
4. **后景墙深色版**：后景 BackWall / BackWallTop 使用同一套茅草生成器，整体色调乘以 0.75~0.80 的暗化系数，模拟后景稻草受光较弱。
5. **UV 调整**：范围贴图不再重复，UV 直接映射到整张贴图像素：`uv = [ (0,0), (tw,0), (tw,th), (0,th) ]`。
