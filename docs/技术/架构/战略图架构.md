## 〇、本文档的目的

现有 `modules/world_map/` 是早期毛坯房，单一粒度的 `RegionDefinition` + 1024×512 渲染管线，与程序化世界生成（8192×8192、三级层级）、GDD 五层级管理深度、其他模块的最新约定都脱节。本文档**重新设计战略图模块的架构**，不写业务代码，只定义：

- 模块定位与边界
- 三级粒度的数据模型
- 渲染架构（含 Shader 升级路径）
- API 契约（对齐其他模块）
- 与现有代码的迁移路径

## 一、模块定位

### 1.1 战略图 vs 场景图（术语基线）

详见 [`世界地图数据流.md`](世界地图数据流.md) §1.1。本文档涉及的「战略图」定义：

- **战略图（Strategic Map）**：鸟瞰多边形领土的「看地图」，玩家不在其中，用于指挥、缩放、外交、查看版图。即「右上角小地图」放大版。
- **场景图（Scene Map）**：玩家实际所在的卷轴地图（GDD §6.7 六类），火柴人在其中真实移动。

战略图模块 `modules/world_map/` 只负责战略图。场景图由 `modules/world/` 等模块负责。

### 1.2 战略图的核心职责

1. **三级缩放渲染**：L3 大世界 / L2 地区 / L1 地块，按缩放级别切换数据粒度和渲染策略
2. **地块查询与选中**：鼠标点击查询 region_id / tile_id / settlement_id，支持框选
3. **地图模式切换**：政治/地形/资源/人口/战线 五种模式（沿用现有 MapModeManager）
4. **与其他模块的单向通信**：通过 api.gd 暴露查询接口，通过 EventBus 接收归属变化/战斗事件
5. **与场景图的切换**：玩家按 Tab 打开战略图（ModalOverlay），点击聚落触发 SceneLoader.load_map()

### 1.3 战略图不负责的事

- ❌ 火柴人渲染、AI、移动（场景图的事）
- ❌ 战斗逻辑、伤害计算（combat 模块的事）
- ❌ 建筑放置、占地网格（场景图 + construction 模块的事）
- ❌ 程序化生成（开发期 Python 工具的事，战略图只消费产物）
- ❌ 存档/读档（SaveManager 的事，战略图数据是只读静态资源 + 运行时缓存）

### 1.4 与 GDD 五层级管理深度的对应

| GDD §4.2 层级 | 战略图覆盖？ | 说明 |
|--------------|------------|------|
| 第5层 帝国级 | ✅ L3 大世界 | 看全大陆，地区色块 + 政权边界 |
| 第4层 行省级 | ✅ L2 地区 | 看一个地区，地块多边形 + Q版图标 |
| 第3层 城市/郡县级 | ✅ L1 地块 | 看一个地块，聚落点位 + 道路 + 资源 |
| 第2层 镇/村级 | ❌ 切到场景图 | 进入聚落 = `SceneLoader.load_map()` |
| 第1层 个体/班组 | ❌ 场景图内附身 | `PossessionInterface` |

> **关键设计原则**：战略图的「缩放」不是连续的相机 zoom，而是**离散的粒度切换**。每个粒度加载不同的数据集和纹理，渲染策略也不同。从 L1 地块再往下缩放，就跳出战略图，进入场景图。

## 三、数据模型

### 3.1 StrategicMapData（运行时容器）

```gdscript
class_name StrategicMapData extends Resource

# L3 大世界（启动时必加载）
var continent: ContinentData = null

# L2 地区（按 region_id 懒加载）
# key: region_id (String), value: RegionData
var regions: Dictionary = {}

# L1 地块（按 tile_id 懒加载）
# key: tile_id (String), value: MapTileData
var tiles: Dictionary = {}

# 政权数据（启动时必加载，小）
var political: PoliticalData = null

# 当前粒度（0=L3, 1=L2, 2=L1）
var current_granularity: int = 0

# 当前聚焦的父级 ID
# L3 时为 ""，L2 时为 region_id，L1 时为 tile_id
var focused_parent_id: String = ""

# 势力颜色表（key: owner_id String, value: Color）
var owner_colors: Dictionary = {}


# ===== 加载 =====

## 加载 L3（启动时调用）
func load_continent(manifest: Resource) -> Result
## [P] manifest 是 config/strategic_map/manifest.tres
## [Q] continent 字段填充，current_granularity = 0

## 加载指定 L2 地区（玩家点击或缩放到 L2 时调用）
func load_region(region_id: String) -> Result
## [P] continent 已加载，region_id 在 continent.regions 内
## [Q] regions[region_id] 填充，current_granularity = 1, focused_parent_id = region_id

## 加载指定 L1 地块
func load_tile(tile_id: String) -> Result
## [P] 该 tile 所属 region 已加载
## [Q] tiles[tile_id] 填充，current_granularity = 2, focused_parent_id = tile_id

## 卸载非相邻粒度的数据（内存控制）
func unload_distant(target_granularity: int, keep_ids: Array[String]) -> void


# ===== 查询 =====

## 根据当前粒度和屏幕坐标查询最细粒度的 ID
func query_id_at_screen(screen_pos: Vector2) -> Dictionary
## 返回 {"granularity": int, "region_id": String, "tile_id": String, "settlement_id": String}
## 未命中字段为 ""

## 获取当前粒度下所有可见多边形
func get_visible_polygons() -> Array[Dictionary]
## 返回 [{"id": String, "polygon": PackedVector2Array, "owner_id": String, "biome": int}, ...]

## 获取聚落引用（L1 粒度下）
func get_settlement(settlement_id: String) -> SettlementRef
```

### 3.2 ContinentData（L3 大世界）

```gdscript
class_name ContinentData extends Resource

var continent_id: String = "continent_main"

# 海洋掩码（用于河流裁剪、海岸线绘制）
var ocean_mask: Image = null

# L3 级别的资源（启动时加载）
# 风格化底图（Python 工具预渲染，含草坪/火山/山脉/河流）
var style_base_texture: Texture2D = null   # 4096×4096 或 8192×8192
# 边界索引图（每地区一色，NEAREST 采样）
```

## 四、渲染架构

### 4.1 核心原则：分层级差异化渲染

三个层级的渲染策略**不同**，越往下越依赖运行时程序化生成：

| 层级 | 底图 | 运行时叠加层 | 程序化程度 | 为什么 |
|------|------|------------|-----------|--------|
| **L3 大世界** | 预渲染风格化底图（Python 工具生成，美术精修） | 边界描边、政治色、地区名、政权边界、河流 | **低**（底图静态） | 大世界视图相对稳定，地区边界不常变，预渲染质量最高 |
| **L2 地区** | 预渲染底图（该地区风格化） | 边界描边、地块着色、Q版地标图标、聚落规模指示 | **中**（地标动态放置） | 地块边界不变，但地标/聚落规模会变 |
| **L1 地块** | 预渲染底图（该地块地形纹理） | **聚落建筑群（动态生成）**、道路、资源图标、战线 | **高**（聚落建筑群实时生成） | 地块是实际游玩的地图，聚落规模随玩家建设实时变化，必须动态 |

> **关键**：L3 是"相对静态的鸟瞰图"，L1 是"活的战术地图"。L1 上的聚落不是静态图标，而是运行时根据聚落数据生成的建筑群——小村落是一小团建筑，大城市是范围很大的建筑群，玩家建设会实时改变地图上的聚落表示。

### 4.2 数据层：多层级独立地图包

8192×8192 原始数据（掩码 + 高度场 + 群系 + 河流矢量）**不直接用于运行时**，由开发期 Python 工具裁剪 + 派生成多个独立的"地图包"，每层级一份或多份：

```
原始数据（8192，开发期 Python 生成）
  │
  │ tools/worldgen/stylize_renderer.py 裁剪 + 风格化底图生成
  ▼
┌──────────────────┬──────────────────────┬──────────────────────┐
│ L3 大世界包（1份）│ L2 地区包（N份）     │ L1 地块包（M份）     │
│ 整个大陆         │ 每地区一份           │ 每地块一份           │
│                  │                      │                      │
│ 含：             │ 含：                 │ 含：                 │
│  · 风格化底图    │  · 风格化底图        │  · 风格化底图        │
│    (4096-8192)   │    (1024-2048)       │    (512-1024)        │
│  · 边界索引图    │  · 边界索引图        │  · 边界索引图        │
│  · 矢量多边形    │  · 矢量多边形        │  · 矢量多边形        │
│  · 河流矢量      │  · 河流矢量(裁剪)    │  · 聚落数据          │
│  · 群系图        │  · 群系图(裁剪)      │  · 道路矢量          │
│                  │  · 地标列表          │  · 资源点列表        │
└──────────────────┴──────────────────────┴──────────────────────┘
```

**每个地图包是独立的 Godot 资源集合**，运行时按当前粒度加载对应包，粒度切换 = 卸载旧包 + 加载新包。

### 4.3 L3 大世界渲染（预渲染为主）

```
1. 绘制 continent_style_base.png（风格化底图，全屏）
   · 草原区已铺好草坪纹理（开发期合成）
   · 火山区已画好火山图标（开发期合成）
   · 山脉区已画好山脉符号（开发期合成）
   · 河流已画好蓝色折线（开发期合成）
2. Shader 叠加层（运行时轻量）：
   · region_overlay.shader 读取边界索引图，按地图模式着色
   · border_outline.shader 描边地区边界
3. 矢量叠加层（运行时）：
   · 地区名文字
   · 政权边界（粗线，区别于地区边界）
   · 战线标记（战斗进行中时）
```

**L3 的特点**：底图是"画好的地图"，运行时只叠加边界和政治信息。玩家看到的草坪/火山/山脉都是预渲染的，质量高、性能好。

### 4.4 L2 地区渲染（预渲染 + 中量程序化）

```
1. 绘制 region_<id>_style_base.png（该地区风格化底图）
   · 该地区的草坪/火山/山脉已预渲染
   · 河流已预渲染（裁剪自 L3）
2. Shader 叠加层：
   · 地块边界索引图 + 按地图模式着色
   · 地块边界描边
3. 程序化叠加层（运行时动态）：
   · Q版地标图标（矿坑/魔力泉/港口/要塞）—— 从地标列表放置
   · 聚落规模指示（每个地块的中心聚落用不同大小图标）
   · 地块名文字
```

**L2 的特点**：底图预渲染，但地标和聚落规模指示是动态的（聚落规模会变）。

### 4.5 L1 地块渲染（重度程序化，"活的地图"）

L1 是最特殊的——地块是实际游玩的地图，聚落规模随玩家建设实时变化，必须重度程序化：

```
1. 绘制 tile_<id>_style_base.png（该地块地形纹理底图）
   · 仅地形纹理（草坪/沙地/雪原的底色和纹理）
   · 不含聚落、道路、资源（这些全部运行时生成）
2. 程序化叠加层（运行时动态生成）：
   · 聚落建筑群（核心，见 §4.6）
   · 聚落间道路（从道路矢量绘制）
   · 资源点图标（矿/沥青/魔力泉）
   · 河流（从矢量绘制，带流动效果）
3. Shader 叠加层：
   · 聚落领地边界（淡色填充）
4. 矢量叠加层：
   · 聚落名文字
   · 战线标记
```

**L1 的特点**：底图只是"地形画布"，聚落/道路/资源全部运行时生成。玩家建设改变聚落数据 → 地图上的聚落表示实时变化。

### 4.6 动态聚落建筑群表示（L1 核心特色）

L1 地块图上的聚落不是静态图标，是**运行时根据聚落数据生成的建筑群节点**：

| 聚落级别 | 地图表示 | 建筑数量 | 占地范围 | 生成依据 |
|---------|---------|---------|---------|---------|
| T1 村落 | 一小团建筑 | 3-5 个小建筑 sprite | ~32×32 px | level=1 + population_score |
| T2 镇/大部落 | 大一点的建筑群 | 6-10 个建筑 | ~64×64 px | level=2 + population_score |
| T3 城市 | 范围较大的建筑群 | 15-30 个建筑 + 城墙 | ~128×128 px | level=3 + population_score |
| T4 中心城市 | 大范围建筑群 + 地标 | 40-80 个建筑 + 城墙 + 地标 | ~192×192 px | level=4 + population_score |
| T5 帝国首都 | 超大建筑群 + 宫殿 | 100+ 建筑 + 宫殿 | ~256×256 px | level=5 + 特殊建筑集 |

**动态生成逻辑**：

```gdscript
# SettlementRenderer.gd（新组件）
func render_settlement(settlement: SettlementRef) -> Node2D:
    var group = Node2D.new()
    var base_count = _get_building_count(settlement.level, settlement.population_score)
    var base_radius = _get_footprint_radius(settlement.level, settlement.population_score)
    
    # 按 industry 选择建筑 sprite 子集
    var building_sprites = _select_building_sprites(settlement.industry, settlement.level)
    
    # 在 base_radius 范围内随机但确定性地放置建筑（用 settlement.seed 保证确定性）
    var rng = RandomNumberGenerator.new()
    rng.seed = hash(settlement.settlement_id)
    for i in base_count:
        var sprite = building_sprites[rng.randi() % building_sprites.size()].instantiate()
        var angle = rng.randf() * TAU
        var dist = rng.randf() * base_radius
        sprite.position = Vector2(cos(angle), sin(angle)) * dist
        group.add_child(sprite)
    
    # 城墙（T3+）
    if settlement.level >= 3:
        group.add_child(_create_wall_ring(base_radius))
    
    # 地标建筑（T4+ 或特殊产业）
    if settlement.level >= 4 or settlement.industry.has("military"):
        group.add_child(_create_landmark(settlement))
    
    return group
```

**关键特性**：
- **确定性**：同一聚落每次渲染都一样（用 settlement_id 作种子），不会闪烁
- **动态**：population_score 变化 → 建筑数量变化 → 地图上聚落大小实时变化
- **响应玩家建设**：玩家在场景图里建农场 → 触发 EventBus.settlement_updated → 战略图 L1 重新渲染该聚落
- **产业差异化**：采矿聚落有矿坑图标，贸易聚落有市场图标，军事要塞有堡垒图标

### 4.7 地图拼接预览模式（跨地区作战）

**需求**：跨地区作战时，玩家需要在地图查看界面看到**相邻地块/地区**的预览和详情，把多个地图包拼接显示，而不是切回大世界图。

**设计方案**：新增"拼接预览模式"（Stitched Preview Mode），与普通粒度模式正交：

```
普通模式：
  L3 大世界 → L2 地区 → L1 地块（单一地图包）

拼接预览模式（跨地区作战时启用）：
  以当前地块/地区为中心，加载相邻 N 个地块/地区的地图包
  按地理邻接关系拼接显示，边缘对齐
  中心包完整渲染，相邻包降级预览（简化叠加层）
```

**拼接渲染策略**：

| 包的角色 | 渲染内容 | 性能策略 |
|---------|---------|---------|
| **中心包**（玩家当前关注） | 完整渲染（底图 + 全部叠加层 + 动态聚落） | 全分辨率 |
| **相邻包**（上下左右4-8个） | 底图 + 边界 + 简化聚落指示（仅级别图标，不生成建筑群） | 降采样底图，跳过程序化生成 |
| **远邻包**（更外围） | 仅边界轮廓 + 政治色 | 矢量多边形描边，不加载底图 |

**边缘对齐约定**：

地图包之间的边缘必须对齐，拼接时无缝。约定：
- 每个地图包记录其在世界坐标系中的**地理范围**（`world_bounds: Rect2`，0-8192 坐标系）
- 拼接时按 `world_bounds` 计算偏移，相邻包的边缘像素天然连续（因为都源自同一张 8192 原始数据裁剪）
- 边界索引图在包边缘有 1-2 像素的重叠区，避免拼接缝隙

**拼接预览的交互**：
- 玩家在 L1 地块图按"显示周围"按钮 → 启用拼接预览
- 相邻地块的简化预览出现在当前地块周围
- 鼠标悬停相邻地块 → 显示该地块详情面板（聚落数、归属、资源）
- 点击相邻地块 → 切换中心包到该地块（拼接预览保持，新中心完整渲染）
- 行军路径叠加：玩家规划跨地块行军时，路径在拼接图上显示，经过的所有地块高亮

### 4.8 Shader 设计要点

#### region_overlay.shader（边界着色，L3/L2 共用）

```glsl
uniform sampler2D region_mask;    // 边界索引图（NEAREST 采样）
uniform sampler2D base_map;       // 风格化底图（BILINEAR 采样）
uniform sampler2D owner_color_lut;  // 1D LUT 纹理
uniform float overlay_alpha;

void fragment() {
    vec4 mask_px = textureLod(region_mask, UV, 0.0);  // NEAREST，不用 mipmap
    int region_id = (int(mask_px.r * 255.0) << 16)
                  | (int(mask_px.g * 255.0) << 8)
                  | int(mask_px.b * 255.0);
    
    vec4 base = texture(base_map, UV);
    vec4 overlay = texelFetch(owner_color_lut, ivec2(region_id % 256, 0), 0);
    overlay.a *= overlay_alpha;
    COLOR = mix(base, overlay, overlay.a);
}
```

> **关键**：边界索引图用 **NEAREST 采样**（不能用 BILINEAR/mipmap，否则相邻 ID 颜色混合导致解码错误）。这是 P社 Clausewitz 引擎省份位图的同款做法，保持边界锐利。

#### border_outline.shader（边界描边）

```glsl
uniform sampler2D region_mask;
uniform vec2 texel_size;  // 1/纹理尺寸
uniform vec4 border_color;

void fragment() {
    int center = decode_id(region_mask, UV);
    int right = decode_id(region_mask, UV + vec2(texel_size.x, 0));
    int down = decode_id(region_mask, UV + vec2(0, texel_size.y));
    
    if (center != right || center != down) {
        COLOR = border_color;  // 边界像素
    } else {
        COLOR = vec4(0);  // 透明，不覆盖底图
    }
}
```

### 4.9 分辨率策略

每个层级的地图包独立分辨率，按"全屏观看时不出现像素块"为标准：

| 层级 | 底图分辨率 | 边界索引图分辨率 | 理由 |
|------|-----------|---------------|------|
| L3 大世界 | 4096×4096 或 8192×8192 | 同底图 | 整大陆要支持缩放看细节 |
| L2 地区 | 1024×1024 ~ 2048×2048（按地区大小） | 同底图 | 单地区范围小，1024 够清晰 |
| L1 地块 | 512×512 ~ 1024×1024（按地块大小） | 同底图 | 单地块更小，512 够清晰 |

> **不降采样到 1024 一刀切**：每层级按实际地理范围选合适分辨率。L3 最大（整大陆），L1 最小（单地块），各自的全屏观看都清晰。

### 4.10 P0 简版 vs P1 完整版

| 阶段 | L3 渲染 | L2 渲染 | L1 渲染 | 拼接预览 |
|------|---------|---------|---------|---------|
| **P0 简版** | 预渲染底图 + GDScript `_draw` 叠加边界 | 同 L3 | 底图 + 静态聚落图标（不生成建筑群） | ❌ 不实现 |
| **P1 完整版** | 预渲染底图 + Shader 叠加 | 同 L3 | 底图 + **动态聚落建筑群** + Shader | ✅ 实现 |
| **P2 优化版** | + LOD 分块加载 | + LOD | + 聚落 LOD（远距离简化） | + 远邻包矢量描边 |

> P0 先跑通三层级切换 + 基础渲染，P1 再上 Shader 和动态聚落建筑群，P2 优化拼接预览和 LOD。**避免在毛坯房阶段过度工程化**。

## 五、地图拼接预览模式（详细设计）

### 5.1 触发场景

- **跨地块行军**：玩家在 L1 地块图规划行军，路径延伸到相邻地块 → 自动启用拼接预览
- **跨地区作战**：战斗涉及多个地块 → 战略图自动切到拼接预览显示战场全貌
- **手动浏览**：玩家按"显示周围"按钮手动启用

### 5.2 拼接预览的渲染层级

```
┌─────────────────────────────────────────────────┐
│ 远邻包（矢量描边层）                              │
│ · 仅绘制多边形轮廓 + 政治色填充                   │
│ · 不加载底图纹理                                 │
├─────────────────────────────────────────────────┤
│ 相邻包（简化预览层）                              │
│ · 加载降采样底图（512×512）                       │
│ · 绘制边界 + 简化聚落图标（仅级别，无建筑群）      │
│ · 跳过道路、资源、战线等细节                       │
├─────────────────────────────────────────────────┤
│ 中心包（完整渲染层）                              │
│ · 全分辨率底图 + 全部叠加层                       │
│ · 动态聚落建筑群                                  │
│ · 道路、资源、战线                                │
└─────────────────────────────────────────────────┘
```

### 5.3 拼接预览的数据需求

每个地图包需要额外记录：

```gdscript
# 在 ContinentData / RegionData / TileData 中增加
var world_bounds: Rect2  # 该包在世界坐标系（0-8192）中的地理范围
var neighbors: Array[String]  # 邻接包的 ID 列表（同粒度）
var edge_overlap: int = 2  # 边缘重叠像素数（避免拼接缝隙）
```

### 5.4 拼接预览的性能控制

- **同时加载的包数量上限**：9（中心 + 8 邻居），超过则远邻降级为矢量描边
- **相邻包底图降采样**：用 512×512 版本，不用全分辨率
- **相邻包跳过程序化生成**：不调用 SettlementRenderer，只画级别图标
- **帧率保护**：拼接预览启用时，中心包保持 60fps，相邻包允许降到 30fps

### 5.5 拼接预览的交互

| 操作 | 行为 |
|------|------|
| 启用拼接预览 | 加载相邻包，渲染三层（远邻/相邻/中心） |
| 鼠标悬停相邻包 | 显示该包详情面板（聚落数、归属、资源摘要） |
| 点击相邻包 | 切换为中心包（旧中心降级为相邻包） |
| 行军路径规划 | 路径在拼接图上绘制，经过的包高亮 |
| 关闭拼接预览 | 卸载相邻包，回到单一包渲染 |

## 七、相机与缩放控制

### 6.1 三级粒度的相机行为

现有 `MapCamera` 是单一粒度的拖拽+滚轮缩放。新架构按粒度区分：

| 粒度 | 相机行为 | 缩放范围 |
|------|---------|---------|
| L3 大世界 | 平移整个大陆，滚轮缩放视觉细节（不切粒度） | 0.5x ~ 2x |
| L2 地区 | 平移该地区，滚轮缩放视觉细节 | 0.5x ~ 2x |
| L1 地块 | 平移该地块，滚轮缩放视觉细节 | 0.5x ~ 2x |

**粒度切换的触发**：
- 双击地区 → 切到 L2（聚焦该地区）
- 双击地块 → 切到 L1（聚焦该地块）
- ESC / 后退按钮 → 返回上一粒度
- **滚轮不触发粒度切换**（避免误操作），只在当前粒度内缩放视觉细节

### 6.2 边界约束

每个粒度的相机视野不超出当前粒度的数据范围：
- L3：视野不超出大陆轮廓
- L2：视野不超出该地区多边形
- L1：视野不超出该地块多边形

### 6.3 与场景图相机的隔离

战略图相机是独立的 `Camera2D`，挂在 `strategic_map.tscn` 根节点下。场景图相机（`CameraRig`）在战略图打开时暂停，关闭时恢复。**两者不共享状态**。

```
  → InputDispatcher 检测当前模式（战斗中禁止）
  → UIRoot 显示 strategic_map ModalOverlay
  → StrategicMapController._ready()（首次打开时 initialize，后续直接显示）
  → 暂停场景图的 CameraRig 和 InputDispatcher
  → 记录 return_map_id = SceneLoader.current_map_id
```

### 8.2 战略图内操作

```
玩家在战略图：
  - 拖拽/滚轮：当前粒度内浏览
  - 双击地区/地块：切换粒度
  - 单击聚落（L1 粒度）：选中
  - 双击聚落（L1 粒度）：enter_settlement()
    → EventBus.travel_requested(settlement.map_id)
    → SceneLoader.load_map(settlement.map_id)
    → 关闭战略图 ModalOverlay
    → 恢复场景图的 CameraRig 和 InputDispatcher
  - ESC：close_strategic_map()，返回 return_map_id
```

### 8.3 场景图内事件同步到战略图

战略图关闭期间，场景图内可能发生归属变化、战斗等。这些事件通过 EventBus 缓存，战略图下次打开时刷新：

```
战略图关闭时：
  - expansion.region_owner_changed → 加入 _pending_refresh 队列
  - combat.battle_started → 加入 _pending_battles
  - combat.battle_ended → 从 _pending_battles 移除

战略图打开时：
  - 处理 _pending_refresh，刷新受影响地区的 PoliticalData
  - 处理 _pending_battles，叠加战线标记
```
