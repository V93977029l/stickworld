# 建筑模块化设计

> 技术层单一真相源：建筑 = 数据驱动 + 程序化生成。
> 关联：[场景与战斗架构.md](场景与战斗架构.md) §4.3 §5、[核心实体与状态机.md](核心实体与状态机.md)、[../../设计/系统/03-定居点与建筑.md](../../设计/系统/03-定居点与建筑.md)
> **实战教程**：[../../教程/建筑系统教程.md](../../教程/建筑系统教程.md)（行业最佳实践 + Day 1 避坑 + 参考→参数化完整流程）

---

## 一、核心思想

建筑不再是一组预制场景（`bld_smithy_lv1.tscn` 等），而是：

```
BuildingDef (数据配方)  ──→  BuildingGen.generate()  ──→  BuildingEntity (运行时节点树)
```

- **数据层**：`BuildingDef` / `ModuleDef` / `MaterialDef` / `RoofShape` 全部 `.tres`
- **生成层**：吃配方产出节点树，不持久化（每次实例化重建）
- **兼容层**：保留大建筑（宫殿、地下城）的预制场景方式

---

## 二、坐标系

- 建筑根原点 = 最底层最左格的左下角
- 1 格 = `CELL_SIZE = 32` 像素（沿用 `PlacementGrid.CELL_SIZE`）
- +X 向右，+Y 向上
- 模块原点 = 模块占第一格的左下角，模块内部贴图/工作位相对此点声明

---

## 三、BuildingEntity 节点结构

```
BuildingEntity (Node2D)
├── Layers[0..N-1]                         # 第 0 层 = 最底
│   └── BuildingLayer (Node2D)
│       ├── BackLayer (Node2D)            z_index=0
│       │   ├── Pillars_Back[]            # 仅 thatch 有
│       │   └── Walls_Back[]              # 仅 wood/stone/brick 有
│       ├── Floor (Node2D)                z_index=4   # 始终可见
│       ├── Modules_Slot (Node2D)         z_index=5   # 本层模块挂载点
│       │   └── Module[*]
│       └── FrontLayer (Node2D)           z_index=10
│           ├── Pillars_Front[]           # 仅 thatch 有，避开模块格
│           ├── Walls_Front[]             # 仅 wood/stone/brick，模块格处跳过
│           └── Openings_Front[]          # 装饰门/窗（B1 后）
├── Roof (Node2D)                         z_index=11  # 整个建筑一个屋顶（最顶层之上）
│   ├── Roof_Back_Strip[]                 # 无缝平铺
│   ├── Roof_End_Left / Roof_End_Right    # 两端山墙
│   └── Roof_Front_Strip[]
├── InteractionZone (Area2D)             # 透明化触发区（按 size 自动生成）
├── EnterTrigger (Area2D)                 # 可选，传送触发（大建筑）
├── PassageBarrier (Area2D)               # 由所有层的墙/柱叠加生成
└── Footprint (CollisionShape2D)
```

### 3.1 z_index 规则

| 层 | z_index | 玩家位置 |
|----|---------|---------|
| BackLayer | 0 | 玩家在外时墙在玩家后；玩家在内时墙也在玩家后 |
| Floor | 4 | 始终可见 |
| Modules_Slot | 5 | 玩家在外（建筑外）时模块在玩家前 → 外面能看见 |
| FrontLayer | 10 | 玩家在外时墙在最前 → 但模块格处挖洞，所以能透出模块；玩家在内时墙在玩家前或后（看玩家相对墙的位置） |
| Roof | 11 | 永远最上层 |

**关键**：玩家位置由物理空间决定，建筑 z_index 不切换。"模块外可见"通过**前景墙挖洞**自然实现。

### 3.2 模块挖洞规则

模块占地集合记为 `exposed_cells[layer_idx]`：
- 该层的 `FrontLayer.Walls_Front` **不生成** 落在 `exposed_cells` 中的格子的墙 Sprite
- 模块自身的 `interior_scene` 挂到 `Modules_Slot`，按模块原点对齐到格左下角
- 模块占用的前景柱也跳过（柱在模块外露可见）

## 四、四种材质结构差异

| `material_id` | 层数上限 | 墙体 | 柱体 | 屋顶 |
|--------------|---------|------|------|------|
| `thatch` | 1 | 无 | 前后景柱 | 人字草顶 |
| `wood` | 2 | 前后木板 + 纵梁（背景隐藏） | 无 | 人字木板顶 |
| `stone` | 3 | 前后石墙 + 角石 | 无 | 平顶城垛 / 人字石顶 |
| `brick` | 4 | 前后砖墙 | 无 | 人字砖顶 / 平顶 |

### 4.1 稻草棚柱布置规则

- 两端必有：`x = 0` 和 `x = width - 1`
- 中间每隔 6~10 格 1 根（种子用 `def.pillar_seed` 或 `hash(def_id + str(layer_idx))` 稳态随机）
- 前景柱生成时检查 `exposed_cells`，落在模块占位格的柱**跳过**（让模块外露）
- 背景柱不避开模块
- 每根柱从该材质的 `pillar_back_textures` / `pillar_front_textures` 数组随机选一张（同建筑稳定哈希种子）

### 4.2 屋顶拼合

```
for x in range(0, width):
  平铺 roof_back_seamless 在 (x*32, roof_y_back)
  平铺 roof_front_seamless 在 (x*32, roof_y_front)
draw roof_end_left 在 (0, roof_y)
draw roof_end_right 在 ((width-1)*32, roof_y)
```

具体高度、offset 由 `RoofShape.total_height` + `front_y_offset` / `endcap_width` 指定（架构不管具体像素值，材质资源里填）。

## 五、数据结构（Resource 类）

### 5.1 BuildingDef

```gdscript
class BuildingDef extends Resource:
    var def_id: String
    var display_name: String
    var size: Vector2i                          # (width_cells, layer_count)
    var default_material: String                 # 当前统一材质，后期混合用
    var layers: Array[LayerSpec]                 # 可空，空则全部用 default_material
    var pillar_seed: int                         # 0 = 用 hash(def_id) 
    var decoration_openings: Array[OpeningSpec]  # B1 后才用，玩家手动加的装饰
    var interior_mode: int                       # NONE / TRANSPARENT / TELEPORT（沿用 §5）
    var mega_interior_map_id: String             # TELEPORT 模式才用
    # 派生：base_hp、base_work 由 size + material 自动算（生成时查 MaterialDef.stats）

class LayerSpec extends Resource:
    var material_id: String                      # 后期混合材质用
    var module_specs: Array[ModulePlacement]

class ModulePlacement extends Resource:
    var module_id: String
    var cell_pos: Vector2i                       # 模块在该层的格坐标
    var auto_position: bool                      # true = 自动布局，cell_pos 忽略
```

### 5.2 ModuleDef

```gdscript
class ModuleDef extends Resource:
    var module_id: String
    var display_name: String
    var footprint: Vector2i                      # 默认 (1,1)
    var interior_scene: PackedScene              # 含装饰 + 工作位 Marker2D + 行为锚点
    var work_slots_local: Array[Vector2i]        # 相对模块原点（左下角）
    var work_slot_count: int                     # 从 work_slots_local 派生
    var placement_constraints: Dictionary        # {min_layer:0, max_layer:99, needs_ground_layer:0, allowed_materials:[]}
    var exposed_to_outside: bool                 # 模块是否需要前景挖洞（默认 true）
    var opening_required: int                    # NONE / DOOR / WINDOW / GAP（B1 装饰开口约束）
```

### 5.3 MaterialDef

```gdscript
class MaterialDef extends Resource:
    var material_id: String                     # thatch / wood / stone / brick
    var display_name: String
    var max_layers: int                         # 1 / 2 / 3 / 4
    var has_walls: bool                         # thatch=false
    var has_pillars: bool                       # thatch=true
    var pillar_back_textures: Array[Texture2D]  # 多张贴图供随机
    var pillar_front_textures: Array[Texture2D]
    var pillar_spacing_min: int                 # 默认 6
    var pillar_spacing_max: int                 # 默认 10
    var wall_back_texture: Texture2D            # wood/stone/brick 才填
    var wall_front_texture: Texture2D
    var roof_shape: Resource                    # 指向 RoofShape 资源
    var color_tint: Color = Color.WHITE
    var stats: Dictionary                       # {hp_per_cell, work_per_cell, build_cost}
```

### 5.4 RoofShape

```gdscript
class RoofShape extends Resource:
    var shape_id: String                        # "thatch_gable" / "wood_gable" / "stone_flat" / "brick_crenel"
    var roof_back_seamless: Texture2D           # 可平铺贴图（沿 X 无缝）
    var roof_front_seamless: Texture2D
    var roof_end_left: Texture2D                # 两端山墙装饰
    var roof_end_right: Texture2D
    var endcap_width: int                       # 端点贴图宽度（像素）
    var total_height: int                       # 屋顶总高（像素）
    var front_y_offset: int                     # 前景屋顶相对后景的 Y 偏移
```

## 六、生成器主流程

```
modules/building_gen/
├── building_gen.gd              # 入口：generate(def) -> BuildingEntity
├── layer_builder.gd             # 单层拼合
├── roof_builder.gd              # 屋顶生成
├── pillar_placer.gd             # 柱位置计算 + 避开模块
├── module_placer.gd             # 模块自动/手动布局
├── collision_assembler.gd       # 由墙/柱叠加生成碰撞
├── interaction_zone_builder.gd  # 按建筑 size 生成 InteractionArea
└── api.gd                       # 公共接口
```

### 6.1 主入口

```gdscript
# building_gen.gd
static func generate(def: BuildingDef) -> Node2D:
    var root = BuildingEntity.new()
    root.def = def
    var occupied_per_layer = {}  # layer_idx -> Set[Vector2i]

    for layer_idx in range(def.size.y):
        var layer = BuildingLayer.new()
        var mat = _layer_material(def, layer_idx)

        # 1. 摆模块（先摆再挖墙，因为要先知道模块占位）
        var exposed_cells = module_placer.place_modules(layer, def.layers[layer_idx], mat)
        occupied_per_layer[layer_idx] = exposed_cells

        # 2. 后景
        layer_builder.build_back(layer, mat, def.size.x, layer_idx, def.pillar_seed)
        # 3. 地板（每层都有）
        layer_builder.build_floor(layer, mat, def.size.x, layer_idx)
        # 4. 前景（挖洞：跳过模块格）
        layer_builder.build_front(layer, mat, def.size.x, layer_idx, def.pillar_seed, exposed_cells)

        root.add_child(layer)

    # 5. 屋顶（最顶层材质）
    var top_mat = _layer_material(def, def.size.y - 1)
    roof_builder.build_roof(root, top_mat.roof_shape, def.size.x, root_top_y)

    # 6. 碰撞由所有层的墙/柱生成
    collision_assembler.assemble(root)
    # 7. 交互区按建筑整体 size 生成
    interaction_zone_builder.build(root, def.size)

    return root
```

### 6.2 柱位置计算

```gdscript
# pillar_placer.gd
static func compute_pillar_xs(def: BuildingDef, layer_idx: int, width: int, avoid_set) -> Array:
    var rng = RandomNumberGenerator.new()
    rng.seed = def.pillar_seed if def.pillar_seed != 0 else hash(def.def_id + str(layer_idx))
    var result = [0]                           # 左端必有
    var x = 1
    while x < width - 1:
        var step = rng.randi_range(pillar_spacing_min, pillar_spacing_max)
        x += step
        if x >= width - 1: break
        if avoid_set and _any_cell_in_col(avoid_set, x):
            continue                           # 避开模块
        result.append(x)
    result.append(width - 1)                   # 右端必有
    return result

static func _pick_texture(arr: Array, seed: int) -> Texture2D:
    if arr.is_empty(): return null
    var rng = RandomNumberGenerator.new()
    rng.seed = seed
    return arr[rng.randi() % arr.size()]
```

### 6.3 前景墙挖洞

```gdscript
# layer_builder.gd
static func build_front(layer, mat, width, layer_idx, seed, exposed_cells):
    if mat.has_walls:
        for x in range(width):
            if Vector2i(x, 0) in exposed_cells:
                continue                      # 模块占位，不生成前景墙
            var sprite = Sprite2D.new()
            sprite.texture = mat.wall_front_texture
            sprite.position = Vector2(x * CELL_SIZE + CELL_SIZE/2, 0)
            layer.front_layer.add_child(sprite)
    if mat.has_pillars:
        var pillar_xs = pillar_placer.compute_pillar_xs(..., exposed_cells)
        for px in pillar_xs:
            var tex = pillar_placer._pick_texture(mat.pillar_front_textures, hash(seed + str(px)))
            var sprite = Sprite2D.new()
            sprite.texture = tex
            sprite.position = Vector2(px * CELL_SIZE + CELL_SIZE/2, 0)
            layer.front_layer.add_child(sprite)
```

## 七、模块挖洞流程

```
ModulePlacer.place_modules(layer, layer_spec, mat) -> Set[Vector2i]:
    exposed_cells = Set()
    for mod_spec in layer_spec.module_specs:
        mod_def = ModuleRegistry.get(mod_spec.module_id)
        pos = mod_spec.auto_position ? _autoplace(layer, mod_def, exposed_cells) : mod_spec.cell_pos
        instance = mod_def.interior_scene.instantiate()
        instance.position = Vector2(pos.x * CELL_SIZE, pos.y * CELL_SIZE)
        layer.modules_slot.add_child(instance)
        # 标记占位
        for dx in range(mod_def.footprint.x):
            for dy in range(mod_def.footprint.y):
                exposed_cells.add(Vector2i(pos.x + dx, pos.y + dy))
    return exposed_cells
```

## 八、Resource 注册表

```gdscript
# modules/building_gen/api.gd
class_name BuildingGenApi
extends Node

var _materials: Dictionary = {}    # material_id -> MaterialDef
var _modules: Dictionary = {}       # module_id -> ModuleDef
var _roofs: Dictionary = {}         # shape_id -> RoofShape

func register_material(m: MaterialDef): _materials[m.material_id] = m
func register_module(m: ModuleDef): _modules[m.module_id] = m
func register_roof(r: RoofShape): _roofs[r.shape_id] = r

func get_material(id: String) -> MaterialDef: return _materials.get(id)
func get_module(id: String) -> ModuleDef: return _modules.get(id)
func get_roof(id: String) -> RoofShape: return _roofs.get(id)

func generate_building(def: BuildingDef) -> Node2D:
    return BuildingGen.generate(def)
```

## 九、与现有系统的兼容映射

| 现有概念 | 重构后映射 |
|---------|-----------|
| `Building.Exterior` | `Building.Layers[*].BackLayer + FrontLayer + Roof` |
| `Building.Interior` | `Building.Layers[*].Modules_Slot + Floor` |
| `Building.WallFront`（透明化操作对象） | 遍历所有层的 `FrontLayer.Walls_Front`，批量 alpha→0.3 + `Pillars_Front` 半透明 |
| `Building.WorkSlots` | 从全部 Module 的 `work_slots_local` 反弹构建的聚合列表 |
| `Building.InteractionZone` | 由 `size = (width, layer_count)` 自动生成 |
| `InitialBuildingsList.building_defs` | 改为存 `BuildingDef.tres` 引用 |
| `ConstructionManager.spawn_operational_building(def_id, cell_x, width)` | 改为 `spawn_operational_building(building_def, cell_x)` |
| `bld_smithy_lv1.tscn` 等静态场景 | 保留给大建筑（宫殿、地下城）作预制件方案 |

## 十三、架构边界

**架构不管**：
- 具体贴图像素值（高度、宽度、offset）—— 全部在 `.tres` 资源里填
- 柱子具体几根、几像素高 —— 通过材质表的 spacing 规则 + 贴图尺寸决定
- 模块内部装饰的摆放 —— 在模块的 `interior_scene.tscn` 里自由布置
- 材质升级流程 —— 由经济/科技系统决定
- 建造进度、工人派工 —— 由 Construction 模块负责

**架构负责**：
- 数据结构定义（Resource 类）
- 生成流程编排（BuildingGen 主循环 + 各子生成器）
- 挖洞逻辑（前景墙跳过模块格）
- 柱位置算法（避模块 + 稳态随机）

## 十四、核心设计决策

> 以下是在开发过程中沉淀的关键架构决策，解释"为什么这样做"而非"做了什么"。

### 14.1 为什么用程序化纹理而非 PNG

- 一个人开发，没有像素美术 → 调参数比画像素快 10 倍
- 建筑宽度可变（3~16 格），一张 64×64 纹理通过 UV 映射适配所有宽度
- 避免 tscn 嵌入纹理二进制 → 参考场景从 966KB 降到 6 行
- 木纹/稻草/石头/砖块四套纹理用同一套 `procedural_materials.gd` 代码生成

### 14.2 为什么"建筑 = 材质 × 模块"（正交设计）

传统做法：铁匠铺是一个预制场景，民居是另一个预制场景。这导致每新增一种功能都要做 N×M 个场景。

本项目做法：

```
铁匠铺   = 稻草棚(thatch) + 锻造作坊(forge_workshop)
高级铁匠铺 = 砖瓦房(brick)  + 锻造作坊(forge_workshop)
民居     = 木板房(wood)   + 住所(residence)
```

- **材质**决定外观（层数上限、墙体/柱体/屋顶形状、建造资源）
- **模块**决定功能（工作位、产能、内部装饰、开口需求）
- 两者自由组合 → 4 种材质 × 5 种模块 = 20 种可能，只需维护 4 + 5 = 9 个定义

### 14.3 为什么"首次填充后退出"而非代码持续拥有

Godot 编辑器和代码都对 `.tscn` 进行写操作，没有合并逻辑 → 后写者覆盖前者。

解决：代码在 `_ready` 中检测 `has_node("L1_BackWall")`，已有节点则跳过。生成的场景保存（Ctrl+S）后成为纯编辑器资产。

### 14.4 行业对标

| 项目 | 方法 | 借鉴点 |
|------|------|-------|
| THE FINALS | Blockout → Feature Nodes | 粗模先行，程序填细节 |
| Townscaper | Wave Function Collapse | 纯约束驱动，无预制场景 |
| Houdini PDG | 节点图参数化 | 一套参数 → 无数变体 |

> 详细行业实践和方法论见 [建筑系统教程](../../教程/建筑系统教程.md) §一。
