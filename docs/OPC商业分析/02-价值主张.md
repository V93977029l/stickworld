# 价值主张分析

> 方法论：OPC一人企业方法论 · 阶段03
> **说明**：以下为基于创始人设计描述的竞品分析与定位分析。打⭐的是创始人原话。
> 分析时间：2026-06-20

---

## 一、核心概念

> ⭐ "管理学游戏 / 组织机构搭建模拟器"

从村民拉帮结派起步，逐步搭建组织机构，体验管理学概念的演进。前期手操一切，后期通过制度和市场机制让帝国自行运转。

**核心钩子**：宏观上在指挥帝国机构，底层仍然是一个一个火柴人在按照你设计的规则交互（meta包装机制）。科技系统不是抽象按钮，而是对现实科研机构的映射——真的要搭建科学院、分配学者、设定成果交付链条。

---

## 二、竞品分析

### 2.1 直接竞品

> ⭐ "黑色战争"、"玩具城堡"（4399）、"合战忍者村"、"火柴人战争遗产"

Steam上没有完全相同的"管理学模拟器"。用户提到的参考游戏中：

| 游戏 | 像什么 | 不像什么 |
|------|--------|----------|
| 《火柴人战争遗产》 | 2D侧视角战斗、出兵机制、火柴人风格、征服解锁兵种 | 纯战斗，没有经营深度 |
| 《合战忍者村》 | 经营+出兵、村庄建设 | 战斗是自动/抽象，没有2D侧视角实操 |
| 《冰汽时代》 | 部落存亡张力、艰难决策 | 没有战斗、没有组织架构设计 |
| 《全面战争》 | 大地图+战斗双层结构 | 聚焦战斗，不是管理学模拟 |
| 《文明》 | 长期策略、科技树 | 非实时、无组织管理 |
| 《小兵步枪》 | 选中排长指挥手下 | 纯战斗，没有经营层 |

**结论**：没有直接竞品。但这也可能是"做出来太难"的信号。

### 2.2 差异化空间

与现有游戏的本质区别：

1. **管理学的系统化呈现**——不是"有个科技树就叫有管理"，而是把组织搭建、指挥链设计、成果交付流程本身就做成玩法
2. **从手操到自动化的转变过程**——不是二选一，是一个渐变的过程，从纯手操→小队指挥→部门管理→帝国调度，贯穿全程
3. **meta包装机制**——宏观帝国管理 + 底层火柴人交互，双重视角

## 三、核心钩子评估

| 钩子要素 | 状态 |
|----------|------|
| 概念独特性 | 高——管理学模拟器，无直接竞品 |
| 传播性 | 中—高——"微型帝国"、"一个人的文明"、"管理学模拟器"有天然话题性 |
| 可演示性 | 需要验证——2D侧视角战斗可演示，但"组织架构设计"是否好看需要验证 |
| UGC传播潜力 | 有——可复制/分享组织结构配置 |
| 重玩价值 | 有——不同文明不同最优组织结构 |

**最大风险**：管理学概念→好玩的游戏，这个转换是否成立？原型测试是关键。
