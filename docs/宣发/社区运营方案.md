# 社区运营方案

> 火柴人大战略——独立游戏的核心资产不是 Wishlist，是种子用户群。
> B站 为主战场，QQ群 为私域阵地。

---

## 一、QQ群运营

### 1.1 建群时机

**现在就可以建。** 群不用等游戏做好了——提前建好，放那儿养着。偶尔发火柴人相关梗图保持活跃。等预热期开始后把群号挂在视频末尾/B站简介。

### 1.2 群配置

| 项目 | 建议 |
|------|------|
| 群名 | `[待定游戏名] 官方交流群` |
| 群规模 | 先建 500 人群（免费），不够再升 2000 人 |
| 群头像 | Logo 纯图案版 |
| 入群验证 | 关掉——独立游戏没必要设门槛 |
| 管理员 | 创始人 + 1-2 个活跃群友（上线后再招募） |

### 1.4 群运营节奏

| 阶段 | 频率 | 内容 |

| 预热期（发布前 4 周） | 每 2-3 天 | 新视频/新消息同步 + 收集反馈 |
| 发布期 | 每天 | Bug 收集 + 感谢 + 维持热度 |
| 长尾期 | 每周 2-3 次 | 更新公告 + 玩家 UGC 分享 + 偶尔活动 |

## 二、B站社区运营

### 2.1 B站账号养号

在发布第一条视频之前，B站账号不要是"空的"：

- **关注同类型 UP 主**：P社游戏 UP、自动化游戏 UP、独立游戏 UP
- **点赞/收藏/评论他们的视频**：让 B站 算法知道你是"策略游戏区的活人"
- **偶尔发动态**：转发他们的视频 + 一句话评论
- **不要求互推**：你没粉丝，互推没意义。先做好自己的内容。

### 2.2 评论区运营

B站 视频发布后，**前 1 小时的评论互动直接影响算法推荐**。

- 发布后立即自己发一条置顶评论（挂 Steam 链接 + "喜欢的朋友点个赞支持一下独立游戏！"）
- 每条评论尽量回复——尤其是提问和反馈
- 如果有人发"这游戏看起来不错"——回复感谢 + 引导关注
- 不要和喷子对线——效率为零，只会让评论区变战场

### 2.3 动态运营

| 类型 | 频率 | 内容示例 |
|------|------|---------|
| 进度播报 | 1-2 次/周（预热期） | "今天修了 47 个 bug，感觉人快没了但游戏更好了" |
| 投票互动 | 偶尔 | "你们觉得这个 Logo 方向 A 还是 B 好？" |
| 转发玩家 UGC | 上线后 | 转发群友分享的指挥链设计 |
| 整活 | 偶尔 | 火柴人梗图、离谱 bug 截图 |

## 三、UGC 传播策略

### 3.1 核心 UGC 点：组织架构分享

你的游戏有一个天然 UGC 优势——**指挥链/组织架构可以导出和分享**。

这不是一个"截图发朋友圈"级别的 UGC。这是一个功能性 UGC——"我设计了最强的军事编制，拿去用"。

**如何设计这个传播闭环**：

1. 游戏内：组织架构面板支持"导出配置"（生成一段文本/JSON/代码）和"导入配置"
2. 传播途径：B站 评论区、QQ群 文件、Steam 讨论区、可能的创意工坊
3. 话题标签：统一用 `#[游戏名]指挥链` 方便搜索
4. 创始人带头：自己先分享几个精心设计的架构，示范"好的指挥链长什么样"
5. 视频化：把玩家分享的优秀架构做成视频——"这届玩家的指挥链有多离谱"

### 3.2 其他 UGC 方向

| UGC 类型 | 传播潜力 | 引导方式 |
|----------|---------|---------|
| 指挥链分享 | ⭐⭐⭐⭐⭐ | 导出/导入功能 + 社区话题标签 |
| 离谱操作 | ⭐⭐⭐⭐ | "翻车实录"合集 → 引导玩家分享自己的翻车经历 |
| 帝国叙事 | ⭐⭐⭐ | "你的帝国最传奇的一战是什么？"——引导写小作文 |
| 整活挑战 | ⭐⭐⭐⭐ | "只用游牧打赢平原文明"——设定挑战让玩家尝试 |

## 四、小黑盒运营

### 4.1 内容策略

小黑盒用户画像：Steam 深度玩家，信息获取以"帖子"和"评测"为主。

- 帖子内容：从 B站 视频简介 + 截图直接复制。不需要单独创作。
- 节奏：和 B站 同步。预热期每周 1 篇，发布周每天 1 篇。
- 帖子格式：图文 + Steam 链接 + 关键信息（定价/折扣/日期）
- 标题风格：比 B站 更直白——小黑盒标题不需要"玩梗"，直接说清楚是什么游戏。

### 4.2 送 Key 活动

小黑盒是做送 key 活动的最佳平台（Steam 高纯度用户 + 评测文化）。

**流程**：
1. Steamworks 后台生成 20-50 个 key
2. 发帖："[游戏名] 即将上线，送 10 个 key！评论说说你最喜欢的策略游戏即可参与"
3. 抽奖方式：随机数抽楼层，公开透明
4. 发 key 时私聊留言："希望玩完后去 Steam 写个评测 🙏"
5. 送 key 的人中有 30-50% 会写评测——这是新游戏早期好评的基础

**不要做的**：
- 不要要求"好评才给 key"（违反 Steam 规则，可能被 ban）
- 不要一次性送太多（20 个以内的 key 效果最好，多了显得廉价）

## 五、Steam 社区运营

### 5.1 讨论区规划

| 分区 | 用途 |
|------|------|
| 综合讨论 | 闲聊、安利、新手提问 |
| Bug 反馈 | 严重 bug → 这里发。小问题 → QQ群 |
| 建议 | 玩家建议收集 |
| 攻略 | 指挥链分享、最优开局等 |

### 5.2 评测管理

- **前 10 条评测至关重要**——Steam 会突显给所有浏览商店页的人
- 如果有差评，**公开回复**而不是删除。独立游戏开发者真诚回复差评比删评更能建立信任
- 回复差评的模板："感谢反馈！你提到的问题我们正在修，预计下一版修复。如果有具体细节欢迎补充 🙏"
- 早期差评里如果有合理建议，采纳后更新 → 再回复那人："你之前提的建议我们已经实装了，感谢！"
- 这会让围观者看到"这个开发者在认真做游戏"——比任何广告都有用

## 六、发布后社区维护

### 6.1 第一周是最关键的

- 每天至少看一次 QQ群 + Steam 讨论区 + B站评论区
- Bug 快速修，修完立刻公告
- 每一条差评都回复（真诚，不杠）
- 不要因为销量不达预期就消失——社区需要看到你在

### 6.2 后续更新节奏

| 更新类型 | 频率 | 内容 |
|----------|------|------|
| 热修 | 发布后 1-2 周，随时 | 严重 bug、崩溃 |
| 小更新 | 2-4 周一次 | QoL 改进、新内容、平衡调整 |
| 大更新 | 1-3 月一次 | 新文明、新系统、新地图 |
| DLC | 6-12 月后 | 大型内容扩展 |

每次更新发一条 B站动态 / QQ群公告，**大更新可以再发一条视频**——这是长尾期重新拉流量的机会。

## 七、危机应对

| 危机 | 应对 |
|------|------|
| 发布日服务器崩了（如果有联机） | 立刻公告 + 预计恢复时间 + 持续更新进展——沉默最致命 |
| 差评如潮（< 60% 好评） | 停止推广。分析差评共性，决定是修还是调整定位。不要花钱推一个差评如潮的游戏 |
| "抄袭 XX 游戏"争议 | 不杠。诚实说明设计参考了哪些游戏，说明差异化在哪。火柴人大战略的缝合广度就是最好的辩护 |
| 群内吵架/节奏 | 管理员及时介入，警告无效直接踢。不要让一个群毁了你的心态 |
