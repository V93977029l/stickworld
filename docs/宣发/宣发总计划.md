# 宣发总计划

> 火柴人大战略（Stick World）—— Steam PC 独立游戏。
> 单人开发，预算 1 万 RMB 上限，不买量，靠内容传播。

---

## ★ 当前执行方案：试水精简版

> **方针**：不指望卖爆，不给宣发太大心理负担。用最轻量的方式让可能喜欢的人发现它就够。
> **心态**：卖几千就赚了，视频播放量低是正常的，当作品集也不亏。

### 总览

```
发布前 2 周 ──────── 发布日 ──── 发布后
  B站 5-6 条整活视频     上线
  小黑盒 1-2 篇用心文章
  抖音 直接搬 B站 视频
  其他平台 听天由命
```

### B站（核心，所有火力集中在这里）

**视频数量**：5-6 条，2 周内发完。

**视频类型**：全部是几分钟的**整活视频**。不搞系统介绍、不做教程、不录旁白解说。

**内容公式**：
- 纯录屏自己玩的时候最抓眼的片段
- 不需要配音，不需要字幕，配个背景音乐就行
- 片段本身足够有趣——火柴人翻车、指挥链崩盘、离谱操作、缩放震撼画面

**标题策略（雷霆式）**：
- 精髓：**乍一看意味不明，但能立刻激起好奇**——玩家会点进去想"这标题啥意思？"
- 反例（不要这样写）："火柴人大战略 steam 独立游戏预告"
- 正例方向：
  - 「我这个师长要是正常人，这把已经赢了」（配图：指挥链某个离谱节点）
  - 「拉了个表格，前线全军覆没」（配图：组织架构面板 + 战场爆炸）
  - 「他说自己能打，我就让他上了」（配图：一个弱鸡火柴人被派去前线）
  - 「我不该任命他当财政部长的」（配图：经济报表炸了）
  - 「这游戏玩了 20 小时，我的帝国还是一坨」（配图：混乱的帝国版图）

**为什么这样做**：
1. 不需要后期剪辑功力——纯录屏片段，你的游戏内容够有趣自然有人看
2. 雷霆标题天然高点击率——比"游戏介绍"的标题更有可能被算法推荐
3. 整活视频对播放量的心理预期低——百位数播放不丢人，千位数就算赚了
4. 如果你花了大量时间做系统介绍视频结果播放量惨淡，心态会很崩；整活视频你本来就没期待，发出去随缘

**发布时间线**：

| 时间 | 动作 |
|------|------|
| 发布前 14 天 | 第 1 条视频 |
| 发布前 11 天 | 第 2 条视频 |
| 发布前 8 天 | 第 3 条视频 |
| 发布前 5 天 | 第 4 条视频 |
| 发布前 3 天 | 第 5 条视频 |
| 发布前 1 天 | 第 6 条视频（可选，如果素材够） |
| **发布日** | **Steam 上线。B站发"正式上线！"动态。** |

### 小黑盒（用心写 1-2 篇）

不复制 B站 标题。小黑盒用户需要的是**信息密度**——告诉他们这游戏是什么、多少钱、为什么值得试。

**第 1 篇**（发布前 1 周）：游戏介绍 + Steam 商店页链接。
- 写清楚这是一个怎样的游戏（不用玩梗，小黑盒用户吃直球）
- 重点写组织架构玩法和无缝缩放——这是别人没有的
- 放几张截图
- 比起没用的宣传写几句自己开发的心路历程——独立游戏开发者在小黑盒有天然好感加成

**第 2 篇**（发布日）：正式上线帖。
- 定价 + 首周折扣 + Steam 链接
- 也可以顺便送几个 key（5-10 个够了，评论抽奖最简单）
- 重点：挂 Steam 链接，把路人转化成购买

### 抖音

从 B站 视频中剪 30 秒竖版。不需要单独创作。发完就完了，别盯数据。

### 其他平台

听天由命——意思是：不用专门为它们准备内容。如果有人从其他地方看到你的游戏来问，再酌情回应。

### 表情包 / Steam 社区物品

这是不用花钱不用拍视频的免费曝光渠道。

**Steam 点数商店**：上传聊天表情 + 动态头像 + 个人资料背景。玩家用 Steam 点数兑换后，每次在 Steam 聊天/评测/个人资料中出现，都是对你游戏的免费曝光。独立游戏在这方面竞争极少——大多数开发者忽视了这个渠道。

素材来源：火柴人角色、战斗名场面、Logo、蓝色史莱姆吉祥物——自由搭配。

**QQ群表情包**：发布时在群文件里丢一套火柴人梗图表情包，玩家自取。群友用熟了就会往别的群扩散——零成本病毒传播。

> 详见 `美术素材清单.md` §七「表情包与 Steam 社区物品」。

### 这个方案的优点

1. **沉没成本低**：如果游戏卖得不好，你在宣发上花的精力加起来不超过 2 周 + 十几小时
2. **心态可控**：每个视频你本来就对播放量没有高期待，不会有落差
3. **足够用了**：真正会被你游戏吸引的人——P社玩家、自动化党——哪怕你只发一条视频他们也能找到你
4. **主战场在 Steam 算法**：商店页写好了，标签选对了，Steam 免费帮你推的人比你 B站 视频能拉来的人多得多

### 什么情况下升级到完整方案？

如果原型验证后，你自己玩了都觉得"卧槽这游戏真的好玩，这不止卖几千的量"，或者前两条 B站 视频意外爆了（播放量 > 2万），再考虑投入更多时间按下面的完整方案走。

## 一、宣发原则

### 1.1 三条铁律

1. **集中宣发，不做长跑**：独立游戏没有品牌积淀，宣发必须在发布前 2-4 周集中引爆。提前半年发 devlog 是浪费子弹——现在没人记得住你。
2. **B站是主战场**：长视频 + 社区互动 + 算法推荐 → 独立游戏最有效的获客渠道。其他平台的内容从 B站 素材直接剪裁复制。
3. **玩梗 > 正经**：火柴人天然带荒诞感。正式预告片反而打不过"整活视频"。参考：《戴森球计划》的"从手搓到自动化"、《太吾绘卷》的"代码级复杂"梗。

### 1.2 宣发的本质不是"告诉别人有这个东西"

是让目标玩家**在 3 秒内产生"我要试试这个"的冲动**。对于火柴人大战略，最强的钩子是：

- "你亲手设计的指挥链自己跑起来了" → 工厂自动化玩家的高潮体验
- "从一个人到整个帝国，你滚轮滚到底" → 尺度跃迁的视觉震撼
- "放大看你手下的小火柴人真的在打仗" → 微观可观察的反差萌

### 1.3 独立游戏宣发的"发布即宣发"逻辑

你说得对：**独立游戏宣发后必须立即能买**，否则热度在发布前就散掉了。

这意味着：
- Steam 商店页面可以在原型可玩后提前上（攒 Wishlist），但**不主动推**
- 真正的内容营销（B站视频、小黑盒帖子）集中在**发布前 2-4 周 + 发布当周**
- 上线后 1-2 周是黄金窗口，持续更新/互动维持热度
- 全程大约 **4-8 周**的密集宣发期，之前全部是准备期

### 阶段 0：商店页上线（原型验证通过后）

**触发条件**：原型核心循环（定居点 + 小队战斗 + 基础资源管理）可玩，创始人觉得好玩。

**动作**：

1. Steam 商店页面上线（不上线游戏，只上页面）
   - 填入商店文案（`Steam商店文案框架.md`，此时填充第一版）
   - 放 5 张最有代表性的截图（或概念图 + 标注"开发中截图"）
   - 放至少 1 个 GIF 展示核心玩法
2. **不主动推**，纯被动攒 Wishlist
3. 可发一条 B站动态："商店页上了"——轻量，不花钱推

**目标**：自然积累几百个 Wishlist，为正式发布时 Steam 算法启动提供基础。

### 阶段 1：预热期（发布前 4 周）

**触发条件**：游戏内容量够剪出 3-5 条视频，且确定发布日期。

| 周 | B站 动作 | 小黑盒 | 抖音 |
|----|---------|--------|------|
| W-4 | 第一条视频：「这游戏让你亲手设计帝国的指挥链」 | — | — |
| W-3 | 第二条视频：「从一个人到整个帝国，无缝缩放演示」 | 第一条帖子（复制 B站 视频简介版） | 第一条短视频（从 B站 视频剪 30s 竖版） |
| W-2 | 第三条视频：「看我把指挥链搞崩了——翻车实录」 | 第二条帖子 | 第二条短视频 |
| W-1 | 第四条视频：「正式发布日期公布！+ Demo 试玩」 | 第三条帖子 + 送 key 活动预热 | 第三条短视频 |

**B站视频策略**：
- 第一条：**系统介绍型**——"这游戏到底是什么"。用清晰的旁白 + 界面演示讲清楚核心玩法。目标：让 P社玩家/自动化玩家说"卧槽这就是给我做的"。
- 第二条：**视觉震撼型**——无缝缩放的纯演示，从火柴人的脸一直缩到帝国全景。不需要太多解说，音乐 + 画面就够了。目标：让路人好奇"啥游戏这么酷"。
- 第三条：**整活型**——你故意把指挥链设计得很烂，然后看军队崩盘。娱乐向，走"翻车"流量。目标：高播放量，破圈。
- 第四条：**信息型**——发布日期 + Steam链接 + Demo试玩。目标：转化 Wishlist 为购买。

**关键判断点**：如果第四条视频发布后 Wishlist 增长不理想，考虑**推迟发布**。独立游戏没有"必须按时发"的压力——发错了就没有第二次机会。

### 阶段 2：发布周（D-Day + 0~7天）

| 天 | 动作 |
|----|------|
| D-1 | 所有平台发布"明天上线"提醒 |
| D-Day | Steam 正式上线。B站发"正式上线！"视频。小黑盒发上线帖。QQ群全员通知。 |
| D+1~3 | 每日盯着评论/B站弹幕/QQ群，快速修严重 bug。发一条"上线首日感谢 + 已知问题修复"动态。 |
| D+4~7 | 如果数据不错（好评 > 80%），发"感谢+后续计划"视频。如果数据差，冷静分析差评原因，优先修。 |

**发布日必须就绪的**：
- 定价确定
- 首周折扣（建议 10-15%，独立游戏标配）
- QQ群置顶公告（购买链接 + 反馈渠道 + 群规）
- Steam 讨论区已开好分区（Bug反馈 / 建议 / 攻略）

### 阶段 3：长尾期（发布后 2-8 周）

| 时间 | 动作 |
|------|------|
| W+2 | 根据玩家反馈发"后续更新计划" |
| W+3~4 | 如果社区有优质 UGC（指挥链分享、搞笑片段），整理发视频 |
| W+5~8 | 节奏放缓。有内容更新 → 发动态。没内容 → 潜水开发 |

**不要做的**：发布后疯狂发宣传视频试图再冲一波。Steam 算法对发布后第二个月的曝光量骤降，存量时期做好社区维护比拉新更重要。

## 三、渠道策略

### 3.1 B站——核心攻坚战场

**为什么 B站 是主战场而不是抖音/小红书**：
- B站 用户 = Steam 用户高度重合（PC玩家、策略游戏受众）
- 长视频 = 能讲清楚复杂玩法（策略游戏在抖音 15 秒说不清）
- 算法推荐 = 内容好自然有流量，不需要买
- 专栏/动态 = 持续触达粉丝

**内容类型配比**：

| 类型 | 频率 | 目的 | 示例 |
|------|------|------|------|
| 系统介绍/教程 | 2-3 条 | 让目标玩家理解玩法 | "这游戏的指挥链设计到底有多深" |
| 纯演示/视觉展示 | 1-2 条 | 震撼拉新 | 无缝缩放纯享 + 配乐 |
| 整活/玩梗 | 2-3 条 | 破圈流量 | 翻车实录、离谱战术挑战 |
| 开发者动态 | 随意 | 维持社区温度 | "今晚修了 47 个 bug" |

**封面标题方法论**：独立游戏在 B站 的封面 = 70% 的点击率。

- ✅ 好标题：「我设计了一个指挥链，然后它自己把敌军打崩了」
- ✅ 好标题：「当你的科研院所有三层管理架构——这游戏疯了」
- ❌ 烂标题：「火柴人大战略游戏介绍」
- ❌ 烂标题：「Stick World Gameplay Trailer」
- 封面：大字 + 游戏内最抓眼的画面（战场/缩放/GIF 裁一帧）

### 3.2 小黑盒——Steam 精准触达

**为什么做**：小黑盒用户 100% 是 Steam 玩家。转化率极高。

**怎么用**：
- 帖子内容从 B站 视频简介复制即可，不需要单独制作
- 重点是**挂 Steam 链接**和**送 key 活动**
- "送 key 换评测"是独立游戏标准操作——提前准备好 20-50 个 key
- 节奏：和 B站 同步，发布前 2 周开始

### 3.3 抖音——短视频复制分发

**为什么做**：用户基数大，但转化率低。战略游戏在抖音天然吃亏（太复杂）。

**怎么用**：
- 从 B站 视频素材剪**竖版 30-60 秒**版本
- 重点放视觉冲击力强的片段（缩放、战斗、翻车）
- 不要试图在抖音讲清楚玩法——做不到。目标是"这啥？去看看"
- 挂 Steam 链接（抖音现在支持）

### 3.4 QQ群——私域转化池

**为什么重要**：独立游戏的核心资产不是 Wishlist 数量，是**愿意帮你测试、反馈、传播的种子用户群**。

**群定位**：
- 发布前：养群，不发游戏内容，偶尔发火柴人相关梗图保持活跃
- 预热期：拉入从 B站 来的早期关注者
- 发布期：Bug反馈第一入口 + 口碑发酵起点
- 长尾期：核心玩家社群，UGC 来源

**群配置**：
- 群名：[待定游戏名] 官方群
- 群规：禁广告/禁人身攻击/鼓励反馈
- 置顶公告模板 → 见 `社区运营方案.md`

## 五、风险与应对

| 风险 | 应对 |
|------|------|
| 第一条视频播放量低（< 5000） | 不是游戏问题，是标题/封面问题。快速迭代，换封面重发或做第二条 |
| Wishlist 增长缓慢 | 检查：商店页截图够吸引吗？描述够清晰吗？标签选对了吗？ |
| 发布日 bug 炸了 | 提前准备热修流程。B站/小黑盒发公告说明已知问题+修复时间。速度决定口碑。 |
| 首周销量远低于预期 | 分析差评。如果定价问题 → 永久降价。如果内容不足问题 → 快速更新。如果玩法问题 → 基本没救，当作品集。 |
| 核心假设（管理学→好玩）被证伪 | 这就是为什么要先做原型。原型觉得不好玩 → 调整设计 → 再测 → 还是不好玩 → 止损。 |
